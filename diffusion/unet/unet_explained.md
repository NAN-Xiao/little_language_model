# UNet 详解 —— 扩散模型的噪声预测骨干

> UNet 是扩散模型（DDPM / Stable Diffusion）的"大脑"，负责看一张加了噪声的图，猜出里面混了多少噪声。

---

## 一、UNet 是干什么的？

### 输入输出

```
输入:
  x_t      —— 加了噪声的图像  (B, 3, 64, 64)
  t        —— 时间步（第几步） (B,)
  context  —— 文本嵌入（可选） (B, 77, 768)

输出:
  ε_pred   —— 预测的噪声      (B, 3, 64, 64)
```

**一句话**：UNet 是个"噪声探测器"，看一张脏图，告诉你"这里面有多少噪声、噪声长什么样"。

### 为什么不用普通 CNN？

普通 CNN（如 ResNet）只做"下采样"（图像越压越小），会丢失细节。UNet 做了两件事：
1. **下采样**（编码器）：压扁图像，提取语义（"这是猫还是狗"）
2. **上采样**（解码器）：放大图像，恢复分辨率
3. **跳跃连接**：把编码器的细节直接传给解码器，防止信息丢失

就像侦探破案：
- 编码器 = "先站远看全貌"（下采样，看整体）
- 瓶颈 = "分析核心线索"（最低分辨率，语义最丰富）
- 解码器 = "走近看细节"（上采样，恢复精细纹理）
- 跳跃连接 = "把远处看到的细节记下来，近处分析时对照"

## 二、UNet 整体结构

```
输入 x_t (B, 3, 64, 64)  +  时间步 t (B,)
         │
         ▼
┌─────────────────────────────┐
│   时间嵌入 SinusoidalEmb     │   t → (B, 256)
└─────────────┬───────────────┘
              │ time_emb
┌─────────────▼──────────────────────────────────────▼─────────┐
│  输入卷积: (B, 3, 64, 64) → (B, 64, 64, 64)                  │
└───────────────────────────────────────────────────────────────┘
              │
┌─────────────▼──────────────────────────────────────▼─────────┐
│  编码器 (Encoder) —— 越压越小                                  │
│                                                                │
│  DownBlock_0:  (B,  64, 64, 64) → skip_0 ─────────────────┐   │
│  DownBlock_1:  (B, 128, 32, 32) → skip_1 ───────────────┐ │   │
│  DownBlock_2:  (B, 256, 16, 16) → skip_2 ─────────────┐ │ │   │
│  DownBlock_3:  (B, 512,  8,  8) → skip_3 ───────────┐ │ │ │   │
└───────────────────────────────────────────────────────┘ │ │ │   │
              │                                            │ │ │   │
┌─────────────▼────────────────────────────────────────────┘ │ │   │
│  瓶颈 (Bottleneck): (B, 512, 8, 8) ← 语义最浓缩的地方      │ │   │
└────────────────────────────────────────────────────────────┘ │   │
              │                                                │   │
┌─────────────▼────────────────────────────────────────────────┘   │
│  解码器 (Decoder) —— 越放越大，拼接跳跃连接                      │
│                                                                │
│  UpBlock_3:  concat(skip_3) → (B, 256, 16, 16) ◄─────────────┘   │
│  UpBlock_2:  concat(skip_2) → (B, 128, 32, 32) ◄───────────────┘   │
│  UpBlock_1:  concat(skip_1) → (B,  64, 64, 64) ◄─────────────────┘   │
│  UpBlock_0:  concat(skip_0) → (B,  64, 64, 64)                     │
└───────────────────────────────────────────────────────────────────┘
              │
         输出头 (1×1 卷积)
              │
         ε_pred (B, 3, 64, 64)
```

**形状变化全览**（以 64×64 输入为例）：

| 阶段 | 模块 | 空间尺寸 | 通道数 | 作用 |
|---|---|---|---|---|
| 输入 | - | 64×64 | 3 | 原始加噪图像 |
| 输入卷积 | Conv3×3 | 64×64 | 64 | 映射到特征空间 |
| 编码器0 | DownBlock | 32×32 | 128 | 下采样2×，提取底层特征 |
| 编码器1 | DownBlock | 16×16 | 256 | 下采样2×，提取中层特征 |
| 编码器2 | DownBlock | 8×8 | 512 | 下采样2×，提取高层语义 |
| 瓶颈 | Bottleneck | 8×8 | 512 | 全局注意力，语义最浓缩 |
| 解码器2 | UpBlock | 16×16 | 256 | 上采样2×，恢复中层 |
| 解码器1 | UpBlock | 32×32 | 128 | 上采样2×，恢复底层 |
| 解码器0 | UpBlock | 64×64 | 64 | 上采样2×，恢复全分辨率 |
| 输出头 | Conv1×1 | 64×64 | 3 | 映射回图像空间 |

## 三、核心组件详解

### 1. 时间嵌入（SinusoidalTimeEmbedding）

**问题**：时间步 `t` 是一个整数（如 500），直接传进神经网络效果很差。神经网络不懂"500"和"501"有什么区别。

**解决**：用正弦/余弦函数把 `t` 编码成高维向量，类似于 Transformer 的位置编码：

```
emb[2i]   = sin(t / 10000^(2i/dim))
emb[2i+1] = cos(t / 10000^(2i/dim))
```

**形象理解**：

```
t = 0:    [sin(0), cos(0), sin(0), cos(0), ...]  = [0, 1, 0, 1, ...]
t = 10:   [sin(10), cos(10), sin(0.1), cos(0.1), ...]
t = 500:  [sin(500), cos(500), sin(5), cos(5), ...]
t = 999:  [sin(999), cos(999), sin(9.99), cos(9.99), ...]

每个 t 产生一个独特的"指纹向量"，让网络能区分不同的噪声水平。
```

然后再过两层 MLP，增强表达能力：

```
SinEmb(t) → Linear(dim, 256) → SiLU → Linear(256, 256) → time_emb
```

**用法**：`time_emb` 会被加到每个 ResBlock 的特征图上，告诉网络"现在是第几步"。

---

### 2. 残差块（ResBlock）—— UNet 的积木

ResBlock 是 UNet 中最基本的单元，结构如下：

```
输入 x (B, in_ch, H, W)
  │
  ├──→ GroupNorm → SiLU → Conv3×3 ──┬──→ GroupNorm → SiLU → Dropout → Conv3×3 ──┐
  │                                 │                                              │
  │         time_emb                │                                              │
  │   (B, 256) → Linear → (B, out_ch)                                              │
  │           ↓ reshape (B, out_ch, 1, 1)                                          │
  │            加到特征图上 ──────────┘                                              │
  │                                                                                │
  └──→ 残差映射（若通道不同，用 1×1 卷积对齐）──────────────────────────────────────┘
                                │
                                ↓
                           输出 (B, out_ch, H, W)
```

**时间注入方式**：

```python
# time_emb: (B, 256) → 经 Linear → (B, out_ch)
t = self.time_proj(time_emb)          # (B, out_ch)

# reshape 成 (B, out_ch, 1, 1)，广播加到 (B, out_ch, H, W)
h = h + t[:, :, None, None]           # (B, out_ch, H, W)
```

**形象理解**：`time_emb` 就像做饭时的"火候参数"。同样的食材（特征图），大火和小火做出来的菜完全不同。时间嵌入告诉网络"现在是文火慢炖（早期，噪声少）还是猛火爆炒（晚期，噪声多）"。

---

### 3. 自注意力（AttentionBlock）—— 全局视野

**问题**：卷积只能看局部（3×3 窗口），看不到远处。比如图像左上角和右下角的关系，卷积要堆很多层才能建立联系。

**解决**：在分辨率最低的地方（如 8×8）用自注意力，让每个位置都能"看到"所有其他位置。

```
输入: (B, C, 8, 8)
  ↓
reshape: (B, C, 8, 8) → (B, 64, C)   ← 把 8×8=64 个空间位置变成序列
  ↓
MultiHeadAttention(Q=K=V=x): (B, 64, C)
  ↓
reshape 回: (B, C, 8, 8)
  ↓
残差连接: + 输入
```

**形象理解**：

```
一张 8×8 的特征图，每个格子是一个"像素代表":

  ┌───┬───┬───┬───┬───┬───┬───┬───┐
  │ A │ B │ C │ D │ E │ F │ G │ H │   ← 每个字母代表一个位置的
  ├───┼───┼───┼───┼───┼───┼───┼───┤       特征向量
  │ I │ J │ K │ L │ M │ N │ O │ P │
  └───┴───┴───┴───┴───┴───┴───┴───┘

自注意力让每个代表都能和其他所有代表"开会讨论":
  "A 说：我和 K 很像，注意一下"
  "M 说：我和 D 有关联"
  
卷积做不到这一点，因为 3×3 窗口只能看到邻居。
```

**为什么只在低分辨率用？**

因为注意力计算量和序列长度的平方成正比（`O(N²)`）：
- 8×8 = 64 个位置 → 64² = 4096，很快
- 64×64 = 4096 个位置 → 4096² = 16,777,216，太慢

所以只在编码器末端和瓶颈用注意力，那里分辨率最低。

---

### 4. 交叉注意力（CrossAttentionBlock）—— 文本控制图像

这是 Stable Diffusion 实现"文字生成图像"的核心机制。

**原理**：

```
Q（查询）= 图像特征    ← 图像问："文本里有什么信息对我有用？"
K（键）  = 文本嵌入    ← 文本提供"检索索引"
V（值）  = 文本嵌入    ← 文本提供"实际内容"
```

**形象比喻**：

```
图像特征 = 一个学生
文本嵌入 = 一本教材

交叉注意力就是：
  学生根据自己的疑问（Q），
  在教材目录里找到相关章节（K 匹配），
  然后阅读那章内容（V）。

例如 prompt = "一只橙色的猫":
  图像中"橙色区域"的特征 → Q
  "橙色"这个词的嵌入      → K, V
  注意力权重高 → 橙色区域被"染"成橙色

  图像中"猫咪形状"的特征 → Q
  "猫"这个词的嵌入        → K, V
  注意力权重高 → 猫咪形状被"塑造"出来
```

**数学流程**：

```python
# 图像特征: (B, H*W, C)     ← Q 的来源
# 文本嵌入: (B, L, D_ctx)   ← K, V 的来源，L=77（CLIP token 数）

Q = h @ W_q    → (B, H*W, C)
K = c @ W_k    → (B, L, C)     ← 注意：K、V 来自文本！
V = c @ W_v    → (B, L, C)

Attn = softmax(Q @ K^T / sqrt(d)) @ V   → (B, H*W, C)
```

**与自注意力的区别**：
- 自注意力：Q = K = V = 图像（图像"问自己"）
- 交叉注意力：Q = 图像，K = V = 文本（图像"问文本"）

---

### 5. 下采样块（DownBlock）—— 编码器

```
输入: (B, in_ch, H, W)
  ↓
ResBlock × 2（+ time_emb）
  ↓
[可选] Self-Attention
  ↓
[可选] Cross-Attention（+ context）
  ↓
Conv3×3 stride=2（空间减半）
  ↓
输出: (B, out_ch, H/2, W/2)  +  skip (B, out_ch, H, W)
```

**skip（跳跃连接）**：在下采样**之前**保存特征图，稍后传给解码器。

**为什么保存下采样前的特征？**

```
下采样前的特征图（32×32）保留了精细的边缘、纹理信息。
下采样后的特征图（16×16）丢失了这些细节，只有模糊的语义。

跳跃连接 = "拍照存档"：
  编码器在 32×32 时拍了一张"高清照"
  解码器恢复到 32×32 时，把这张"高清照"拼接过来对照
  → 解码器既有语义（大局观），又有细节（高清照）
```

---

### 6. 上采样块（UpBlock）—— 解码器

```
输入: (B, in_ch, H, W)
  ↓
Upsample（双线性插值 ×2）→ (B, in_ch, H*2, W*2)
  ↓
concat([上采样结果, skip]) → (B, in_ch+skip_ch, H*2, W*2)
  ↓
ResBlock × 2（+ time_emb）
  ↓
[可选] Self-Attention
  ↓
[可选] Cross-Attention（+ context）
  ↓
输出: (B, out_ch, H*2, W*2)
```

**拼接跳跃连接**：

```python
# 上采样后的特征: (B, 256, 16, 16)
# 跳跃连接:       (B, 256, 16, 16)  ← 来自编码器同阶段的"存档照"
# 拼接后:         (B, 512, 16, 16)  ← 通道翻倍！

h = torch.cat([h, skip], dim=1)   # 在通道维度拼接
```

**形象理解**：

```
解码器在恢复图像时：
  上采样结果 = "我猜测这里应该是猫"（模糊，只有语义）
  skip       = "编码器说这里有胡须纹理"（清晰，有细节）
  拼接后     = "哦，猫 + 胡须纹理 = 有胡须的猫"（语义+细节）
```

---

### 7. 瓶颈（Bottleneck）—— 语义最浓缩

```
输入: (B, channels, H, W)   ← 最低分辨率，如 8×8
  ↓
ResBlock → Self-Attention → Cross-Attention → ResBlock
  ↓
输出: (B, channels, H, W)
```

瓶颈是 UNet 的"大脑皮层"：
- 分辨率最低（8×8），计算注意力最快
- 语义最浓缩（"这是猫、橙色、在草地上"）
- 文本注入效果最强（交叉注意力在这里作用最大）

## 四、完整数据流（带具体数字）

假设条件：
- Batch size: 2
- 图像: 64×64 RGB
- 文本: "一只橙色的猫"（CLIP 编码后 77×768）

```python
import torch
import torch.nn.functional as F

# ═══════════════════════════════════════════════════════════════
# 输入
# ═══════════════════════════════════════════════════════════════
x_t = torch.randn(2, 3, 64, 64)       # 加噪图像 (B, C, H, W)
t = torch.tensor([500, 750])           # 时间步 (B,)
context = torch.randn(2, 77, 768)      # 文本嵌入 (B, L, D_ctx)
                                       # 来自 CLIP: 77个token，每个768维

# ═══════════════════════════════════════════════════════════════
# Step 1: 时间嵌入
# ═══════════════════════════════════════════════════════════════
# t = [500, 750] → 正弦编码 → MLP → time_emb
time_emb = time_embedding(t)           # (2, 256)
# 内部过程:
#   t=500 → sin/cos 编码 → [sin(500), cos(500), sin(5), cos(5), ...]
#   → Linear(256, 256) → SiLU → Linear(256, 256)
#   结果: 每个时间步一个 256 维向量

# ═══════════════════════════════════════════════════════════════
# Step 2: 输入卷积
# ═══════════════════════════════════════════════════════════════
h = input_conv(x_t)                    # (2, 3, 64, 64) → (2, 64, 64, 64)
# Conv3×3(3→64): 3通道图像映射到64维特征空间

# ═══════════════════════════════════════════════════════════════
# Step 3: 编码器（下采样 + 收集跳跃连接）
# ═══════════════════════════════════════════════════════════════
# DownBlock_0: (2, 64, 64, 64) → (2, 128, 32, 32) + skip_0
h, skip_0 = down_block_0(h, time_emb, context)
# 内部:
#   ResBlock(64→128): 通道翻倍，空间不变
#   time_emb 注入: (2, 256) → Linear → (2, 128) → reshape → 加到特征图
#   [无注意力，分辨率64太大]
#   Conv stride=2: (2, 128, 64, 64) → (2, 128, 32, 32)
#   skip_0 = 下采样前的特征 (2, 128, 64, 64) ← "存档照"

# DownBlock_1: (2, 128, 32, 32) → (2, 256, 16, 16) + skip_1
h, skip_1 = down_block_1(h, time_emb, context)
# skip_1 = (2, 256, 32, 32)

# DownBlock_2: (2, 256, 16, 16) → (2, 512, 8, 8) + skip_2
h, skip_2 = down_block_2(h, time_emb, context)
# skip_2 = (2, 512, 16, 16)

# DownBlock_3: (2, 512, 8, 8) → (2, 512, 4, 4) + skip_3
h, skip_3 = down_block_3(h, time_emb, context)
# skip_3 = (2, 512, 8, 8)

# 编码器总结:
#   空间: 64×64 → 32×32 → 16×16 → 8×8 → 4×4（压了16倍）
#   通道: 64 → 128 → 256 → 512 → 512
#   收集: skip_0, skip_1, skip_2, skip_3（4张"存档照"）

# ═══════════════════════════════════════════════════════════════
# Step 4: 瓶颈
# ═══════════════════════════════════════════════════════════════
h = bottleneck(h, time_emb, context)   # (2, 512, 4, 4)
# 内部:
#   ResBlock → Self-Attention(Q=K=V=图像) → Cross-Attention(Q=图像, K/V=文本) → ResBlock
#   自注意力: 把 4×4=16 个位置展平成序列，互相看
#   交叉注意力: 图像问文本"我该生成什么？"，文本回答"橙色、猫"

# ═══════════════════════════════════════════════════════════════
# Step 5: 解码器（上采样 + 拼接跳跃连接）
# ═══════════════════════════════════════════════════════════════
# UpBlock_3: (2, 512, 4, 4) + skip_3(2, 512, 8, 8) → (2, 256, 8, 8)
h = up_block_3(h, skip_3, time_emb, context)
# 内部:
#   Upsample×2: (2, 512, 4, 4) → (2, 512, 8, 8)
#   concat([上采样结果, skip_3]): (2, 512+512, 8, 8) = (2, 1024, 8, 8)
#   ResBlock(1024→256): 压缩通道

# UpBlock_2: (2, 256, 8, 8) + skip_2(2, 512, 16, 16) → (2, 128, 16, 16)
h = up_block_2(h, skip_2, time_emb, context)

# UpBlock_1: (2, 128, 16, 16) + skip_1(2, 256, 32, 32) → (2, 64, 32, 32)
h = up_block_1(h, skip_1, time_emb, context)

# UpBlock_0: (2, 64, 32, 32) + skip_0(2, 128, 64, 64) → (2, 64, 64, 64)
h = up_block_0(h, skip_0, time_emb, context)

# 解码器总结:
#   空间: 4×4 → 8×8 → 16×16 → 32×32 → 64×64（恢复原始大小）
#   每步都拼接了编码器的"存档照"，细节得以保留

# ═══════════════════════════════════════════════════════════════
# Step 6: 输出头
# ═══════════════════════════════════════════════════════════════
eps_pred = output_head(h)              # (2, 64, 64, 64) → (2, 3, 64, 64)
# GroupNorm → SiLU → Conv1×1(64→3)
# 把64维特征映射回3通道（RGB），就是预测的噪声

# eps_pred: (2, 3, 64, 64)  ← "我猜测这张图里混了这些噪声"
```

## 五、数学公式速查

### 时间嵌入（正弦编码）

```
emb[2i]   = sin(t / 10000^(2i/d))
emb[2i+1] = cos(t / 10000^(2i/d))

其中 d = 基础维度（如 256），i = 0, 1, ..., d/2-1
```

### 自注意力（Scaled Dot-Product Attention）

```
Attention(Q, K, V) = softmax(Q @ K^T / sqrt(d_k)) @ V

Q = h @ W_q    (B, N, C)
K = h @ W_k    (B, N, C)
V = h @ W_v    (B, N, C)

N = H×W（序列长度），d_k = C / num_heads
```

### 交叉注意力

```
Q = h_img @ W_q     (B, H*W, C)     ← 图像特征
K = c_text @ W_k    (B, L,   C)     ← 文本嵌入
V = c_text @ W_v    (B, L,   C)     ← 文本嵌入

Attn = softmax(Q @ K^T / sqrt(d_k)) @ V   → (B, H*W, C)
```

### 残差连接

```
out = F(x) + x

如果通道不匹配: out = F(x) + Conv1×1(x)
```

## 六、模拟训练代码

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

# ═══════════════════════════════════════════════════════════════
# 假设条件
# ═══════════════════════════════════════════════════════════════
B = 2                      # batch size
C = 3                      # RGB 通道
H, W = 64, 64              # 图像尺寸
T = 1000                   # DDPM 总步数

# ═══════════════════════════════════════════════════════════════
# Step 1: 准备输入
# ═══════════════════════════════════════════════════════════════
# 干净图像
x_0 = torch.randn(B, C, H, W)        # (2, 3, 64, 64)

# 文本提示词（CLIP 编码后）
context = torch.randn(B, 77, 768)    # (2, 77, 768)

# ═══════════════════════════════════════════════════════════════
# Step 2: DDPM 加噪（查表）
# ═══════════════════════════════════════════════════════════════
# 随机采样时间步
t = torch.randint(0, T, (B,))        # → [500, 750]

# 采样噪声
noise = torch.randn_like(x_0)        # (2, 3, 64, 64)，randn_like: 和 x_0 同形状

# 查表得系数（从预计算的 1000 元素数组中索引）
# sqrt_alphas_cumprod:      t=500→0.71,  t=750→0.50
# sqrt_one_minus_alphas_cumprod: t=500→0.70,  t=750→0.87
sqrt_alpha_bar = scheduler.sqrt_alphas_cumprod[t].reshape(B, 1, 1, 1)
sqrt_one_minus = scheduler.sqrt_one_minus_alphas_cumprod[t].reshape(B, 1, 1, 1)

# 加噪: x_t = √ᾱ_t·x_0 + √(1-ᾱ_t)·ε
x_t = sqrt_alpha_bar * x_0 + sqrt_one_minus * noise
#     [0.71, 0.50] * x_0  +  [0.70, 0.87] * noise
#     图1: 71%原图 + 70%噪声
#     图2: 50%原图 + 87%噪声

# ═══════════════════════════════════════════════════════════════
# Step 3: UNet 预测噪声
# ═══════════════════════════════════════════════════════════════
# 创建 UNet（配置为小型网络，适合学习）
config = UNetConfig(
    in_channels=3,
    out_channels=3,
    base_channels=64,
    channel_mults=(1, 2, 4),      # 3级下采样（64→32→16→8）
    num_res_blocks=2,
    time_emb_dim=256,
    context_dim=768,               # 使用文本条件化
)
unet = UNet(config)

# 前向传播
# 输入: x_t(2,3,64,64) + t(2,) + context(2,77,768)
# 输出: 预测的噪声
eps_pred = unet(x_t, t, context=context)   # → (2, 3, 64, 64)

# UNet 内部做了什么（简化版）:
#   t=[500,750] → TimeEmbedding → (2, 256)
#   x_t → Conv3×3 → (2, 64, 64, 64)
#   编码器: 64×64→32×32→16×16→8×8（收集跳跃连接）
#   瓶颈: Self-Attn + Cross-Attn(文本)
#   解码器: 8×8→16×16→32×32→64×64（拼接跳跃连接）
#   输出头: (2, 64, 64, 64) → (2, 3, 64, 64)

# ═══════════════════════════════════════════════════════════════
# Step 4: 计算损失
# ═══════════════════════════════════════════════════════════════
# 损失 = 预测噪声 vs 真实噪声 的均方误差
loss = F.mse_loss(eps_pred, noise)
# loss = mean((eps_pred - noise)²)
#      在 (B, C, H, W) = (2, 3, 64, 64) 的所有像素上求平均

# ═══════════════════════════════════════════════════════════════
# Step 5: 反向传播（只更新 UNet！）
# ═══════════════════════════════════════════════════════════════
loss.backward()
optimizer.step()

# 训练完成！UNet 学到了：
# "给定一张加噪图 x_t、时间步 t、文本描述，我能猜出里面有多少噪声"
```

## 七、UNet 的关键设计哲学

| 设计 | 目的 | 如果去掉会怎样？ |
|---|---|---|
| **编码器-解码器** | 先压缩语义，再恢复细节 | 只剩编码器 → 无法恢复高分辨率 |
| **跳跃连接** | 保留细节信息 | 去掉 → 生成图像模糊、丢失纹理 |
| **时间嵌入** | 告诉网络当前噪声水平 | 去掉 → 网络不知道"第几步"，预测混乱 |
| **自注意力** | 建立全局依赖 | 去掉 → 左眼看不着右眼，生成不协调 |
| **交叉注意力** | 文本控制图像生成 | 去掉 → 变成无条件生成，不听文字 |
| **残差连接** | 缓解梯度消失，加速训练 | 去掉 → 深层网络训练困难 |
| **GroupNorm** | 稳定小 batch 训练 | 用 BatchNorm → batch=1 时崩溃 |

## 八、一句话总结

```
UNet = 卷积编码器（压扁看全局）
     + 卷积解码器（放大恢复细节）
     + 跳跃连接（保留高清细节）
     + 时间嵌入（知道第几步）
     + 交叉注意力（听文字的话）
     
输入: "一张加了噪声的图 + 时间步 + 文本描述"
输出: "这张图里有多少噪声"
```

> UNet 不是生成图像，而是**预测噪声**。真正的"去噪"在采样器里做（DDPMNoiseScheduler.remove_noise_step）。UNet 只是提供"噪声预测"这个关键信息。

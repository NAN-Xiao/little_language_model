# 注意力与位置编码详解：从数学原理到维度流转

---

## 0. 核心问题：位置编码是加到 token 上，还是旋转 token？

```
两种都有！取决于用哪种方式:

  正弦位置编码 → 加法: embedding = embedding + PE(pos)
    直接加在 embedding 向量上, 在进入 Transformer 层之前只加一次
    就像查完词表得到768维向量后, 再叠加一个768维的位置向量

  RoPE 旋转编码 → 旋转: q' = R(θ) · q
    旋转的是注意力里拆完多头之后的 q 和 k, 不是 embedding
    每层 Transformer block 里都旋转一次

  ┌──────────────────────────────────────────────────────────────┐
  │  加法 (正弦编码): 作用于 embedding                           │
  │                                                              │
  │  流程:                                                       │
  │    token_ids → Embedding查表 → (B, seq, 768)                │
  │              → × √768                                       │
  │              → + PE(pos)        ← 加法! 加在 embedding 上   │
  │              → 进入 Transformer 层 (不再加)                  │
  │                                                              │
  │  多个 token 怎么加? 每个 token 各自加不同位置的 PE:         │
  │                                                              │
  │    "猫" 在位置0:  embedding_猫 + PE[0]  ← PE[0]=[0,1,0,1,.]│
  │    "吃" 在位置1:  embedding_吃 + PE[1]  ← PE[1]=[0.84,...]  │
  │    "鱼" 在位置2:  embedding_鱼 + PE[2]  ← PE[2]=[0.91,...]  │
  │                                                              │
  │    每个 token 的 768 维向量都加上各自位置的 768 维 PE       │
  │    → 位置不同, 加的偏移不同 → 即使内容相同也能区分         │
  │                                                              │
  │  embedding向量:  ──→  长度约1, 方向0°                       │
  │  + PE(pos):     ──↗  叠加位置偏移                            │
  │  结果:          ──→↗  长度变了, 方向也变了                   │
  │                                                              │
  │  ────────────────────────────────────────────────────────    │
  │                                                              │
  │  旋转 (RoPE): 作用于注意力中的 q 和 k                        │
  │                                                              │
  │  流程:                                                       │
  │    token_ids → Embedding查表 → (B, seq, 768)                │
  │              → × √768                                       │
  │              → 进入 Transformer 层 (不加位置编码!)           │
  │              → 每层内部:                                     │
  │                  x → w_q → 拆多头 → q (B, 12, seq, 64)      │
  │                  q' = rotate(q, θ_pos) ← 旋转! 作用在q,k上 │
  │                  k' = rotate(k, θ_pos)                      │
  │                  v 不旋转                                    │
  │                                                              │
  │  多个 token 怎么旋转? 每个 token 独立旋转, 角度不同:       │
  │                                                              │
  │    "猫" 在位置0:  q_猫 旋转 0×θ → 不转                     │
  │    "吃" 在位置1:  q_吃 旋转 1×θ → 转θ角度                  │
  │    "鱼" 在位置2:  q_鱼 旋转 2×θ → 转2θ角度                 │
  │                                                              │
  │    768维拆成12头×64维, 每64维再拆32对×2维                   │
  │    每对各自在2D平面上旋转, 角度 = 位置号 × 该对的频率θⱼ    │
  │    位置0 → 旋转0°, 位置1 → 旋转57°, 位置2 → 旋转115°       │
  │    → 不同位置的 token 方向不同 → 点积时能区分位置           │
  │                                                              │
  │  q向量:  ──→  长度3, 方向0°                                 │
  │  旋转θ:  ↗─   长度还是3! 只有方向变了                       │
  │  结果:   ↗─   长度不变                                      │
  │                                                              │
  │  关键区别:                                                   │
  │    加法: 加在 embedding 上, 值和方向都变, 只加一次, 深层稀释│
  │    旋转: 旋转 q/k, 只变方向, 每层都做, 永不稀释             │
  └──────────────────────────────────────────────────────────────┘
```

## 1. 为什么需要位置编码？

```
Transformer 的注意力本身是"位置无关"的:

  q · k 只看内容，不看位置

  输入:  ["猫", "吃", "鱼"]  →  q·k 点积
  输入:  ["鱼", "吃", "猫"]  →  q·k 点积

  如果 "猫" 和 "鱼" 的词向量相同，两种输入的注意力结果一样！

  但语义完全不同: "猫吃鱼" vs "鱼吃猫"

  → 必须额外告诉模型: 每个 token 在第几个位置
```

## 3. 正弦位置编码详解 (SinusoidalPE)

### 3.1 函数原型

```python
class SinusoidalPositionalEncoding(nn.Module):
    def __init__(self, d_model: int, max_len: int = 5000, dropout: float = 0.1)
    def forward(self, x: torch.Tensor) -> torch.Tensor
        # x: (B, seq_len, d_model) → (B, seq_len, d_model)
```

### 3.2 数学公式

```
PE(pos, 2i)   = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))

  pos: token 的位置 (0, 1, 2, ...)
  i:   维度索引 (0, 1, 2, ..., d_model/2-1)
  偶数维用 sin，奇数维用 cos，成对编码同一频率
```

### 3.3 用模拟数值理解

```
d_model=8 (简化，实际768), 3 个位置:

  频率分母 (div_term):
    i=0: 10000^(0/8) = 1.0       → 频率最高
    i=1: 10000^(2/8) = 21.5
    i=2: 10000^(4/8) = 100.0
    i=3: 10000^(6/8) = 464.2     → 频率最低

  位置0:
    PE[0] = [sin(0/1.0),  cos(0/1.0),  sin(0/21.5),  cos(0/21.5),  sin(0/100),  cos(0/100),  sin(0/464),  cos(0/464)]
          = [0,            1,            0,             1,             0,            1,             0,            1]

  位置1:
    PE[1] = [sin(1/1.0),  cos(1/1.0),  sin(1/21.5),  cos(1/21.5),  sin(1/100),  cos(1/100),  sin(1/464),  cos(1/464)]
          = [0.84,         0.54,         0.047,         0.999,         0.010,        1.000,         0.002,        1.000]

  位置2:
    PE[2] = [sin(2/1.0),  cos(2/1.0),  sin(2/21.5),  cos(2/21.5),  sin(2/100),  cos(2/100),  sin(2/464),  cos(2/464)]
          = [0.91,        -0.42,         0.093,         0.996,         0.020,        1.000,         0.004,        1.000]

  观察:
    高频维 (i=0): 位置0→1→2 变化剧烈 (0→0.84→0.91) → 区分相邻位置
    低频维 (i=3): 位置0→1→2 几乎不变 (1→1→1)        → 感受远距离关系
```

### 3.4 多个 token 时怎么加？每个 token 各自加不同位置的 PE

```
  输入句子: "猫 吃 鱼"  → 3 个 token

  PE 是一个预计算好的矩阵, 形状 (max_len, d_model):
    PE[0]: 位置0 的 768 维向量 = [0, 1, 0, 1, 0, 1, ...]
    PE[1]: 位置1 的 768 维向量 = [0.84, 0.54, 0.05, 1.0, ...]
    PE[2]: 位置2 的 768 维向量 = [0.91, -0.42, 0.09, 1.0, ...]
    ...

  每个 token 的 embedding 加上对应位置的 PE:

    "猫" 位置0:  [0.12, -0.03, 0.87, ...] + [0,    1,     0,    1,    ...]
                                          = [0.12, 0.97,  0.87, 1.87, ...]

    "吃" 位置1:  [0.44, 0.91, -0.22, ...] + [0.84, 0.54,  0.05, 1.0,  ...]
                                          = [1.28, 1.45, -0.17, 1.78, ...]

    "鱼" 位置2:  [-0.55, 0.33, 0.11, ...] + [0.91, -0.42, 0.09, 1.0,  ...]
                                          = [0.36, -0.09, 0.20, 1.11, ...]

  关键: 每个 token 加的偏移不同!
    → 即使两个 token 内容相同 (embedding 相同)
    → 加了不同位置的 PE 后, 768 维向量就不同了
    → 注意力计算时能区分 "第2位的猫" 和 "第5位的猫"

  代码怎么实现的?
    x: (B, seq, 768)       ← seq 个 token, 每个 768 维
    pe[:, :seq]: (1, seq, 768)  ← seq 个位置的 PE

    x = x + pe[:, :seq]    ← 逐 token 对应相加
                            ← token0 加 PE[0], token1 加 PE[1], ...

    不需要循环! PyTorch 广播自动完成逐位置相加
```

### 3.5 完整维度流转

```
  token_ids: (2, 5)                    ← 2个样本, 5个token
       │
       ▼  Embedding(68, 768) 查表
  x: (2, 5, 768)                       ← 每个 token 变成 768 维向量
       │
       ▼  × √768 ≈ 27.7               ← 缩放! 为什么?
  x: (2, 5, 768)
       │                                  Embedding 初始化: 值约 ±0.04
       │                                  PE 值: 约 ±1
       │                                  不缩放: 位置编码 >> 语义 → 语义被淹没
       │                                  缩放后: embedding 值约 ±1, 和 PE 同量级
       │
       ▼  + PE[:, :5]                   ← 加法! 不是旋转!
  x': (2, 5, 768)                       ← PE 形状 (1, 5, 768), 广播相加
       │                                  token0 + PE[0], token1 + PE[1], ...
       │
       ▼  进入 10 层 DecoderBlock       ← 后面不再加位置编码
       │
  ...每层内部 attention.py 里:           ← 不做任何位置相关操作!
    q = w_q(x)                          ← q 里已经"带着"位置信息 (来自入口的 PE)
    k = w_k(x)                          ← k 里也是
    scores = q · k^T                    ← 点积时位置信息参与计算
```

### 3.5 正弦编码的问题：深层稀释

```
  入口:  x + PE(pos)                    ← 位置信息清晰

  第1层: x₁ = x + Attn(norm(x))        ← 残差连接保留了部分原始 x
         x₁ 里还有位置信息的"影子"

  第5层: x₅ = x₄ + Attn(norm(x₄))      ← 每层都在 x 上叠加新变换
         原始 PE 的影响被层层稀释

  第10层: x₁₀ 里位置信息已经很弱        ← 为什么? 因为每层经过:
         - LayerNorm: 归一化可能削弱 PE 的影响
         - Attention: 聚合了其他 token 的信息
         - FFN: 非线性变换进一步混合

  → 深层模型 (100层) 可能"忘掉"位置信息
  → 这就是 RoPE 被发明的原因
```

## 4. RoPE 旋转位置编码详解

### 4.1 函数原型

```python
class RotaryPositionEmbedding(nn.Module):
    def __init__(self, d_k: int, max_len: int = 8192, base: float = 10000.0)
    def forward(self, q: torch.Tensor, k: torch.Tensor, offset: int = 0)
        -> tuple[torch.Tensor, torch.Tensor]
        # q: (B, n_heads, seq_len, d_k) → (B, n_heads, seq_len, d_k)
        # k: (B, n_heads, seq_len, d_k) → (B, n_heads, seq_len, d_k)
        # 返回旋转后的 q 和 k，形状不变
```

### 4.2 核心数学原理

```
┌──────────────────────────────────────────────────────────────────┐
│  2D 旋转矩阵:                                                   │
│                                                                  │
│    [cos(θ)  -sin(θ)] [x₀]   [x₀·cos(θ) - x₁·sin(θ)]          │
│    [sin(θ)   cos(θ)] [x₁] = [x₀·sin(θ) + x₁·cos(θ)]          │
│                                                                  │
│  旋转不改变向量长度, 只改变方向!                                 │
│    旋转前: ||[x₀, x₁]|| = √(x₀² + x₁²)                       │
│    旋转后: ||[x₀', x₁']|| = √(x₀'² + x₁'²) = 同上             │
│    → 只注入位置信息, 不破坏原有的语义强度                        │
│                                                                  │
│  关键性质: 两个旋转后的向量点积, 只取决于旋转角度差:            │
│                                                                  │
│    旋转了 m·θ 的 q  ·  旋转了 n·θ 的 k                          │
│    = f(q, k, (m-n)·θ)                                           │
│    只和 (m-n) 有关, 和 m, n 的具体值无关!                        │
│    → 这就是"相对位置"编码的数学保证                             │
│                                                                  │
│  证明 (2D 情况):                                                │
│    q' = R(m·θ)·q = [q₀·cos(mθ) - q₁·sin(mθ),                  │
│                      q₀·sin(mθ) + q₁·cos(mθ)]                   │
│    k' = R(n·θ)·k = [k₀·cos(nθ) - k₁·sin(nθ),                  │
│                      k₀·sin(nθ) + k₁·cos(nθ)]                   │
│                                                                  │
│    q'·k' = q₀k₀·cos(mθ)cos(nθ) + q₀k₀·sin(mθ)sin(nθ)         │
│          + ... (展开后用三角恒等式)                              │
│          = q₀k₀·cos((m-n)θ) + q₁k₁·cos((m-n)θ)                │
│          + (q₀k₁ + q₁k₀)·sin((m-n)θ)                           │
│          = f(q, k, (m-n)θ)                                       │
│                                                                  │
│    → 注意力分数只取决于两个 token 的距离 (m-n)                  │
│    → 不管它们在序列的哪个位置, 只要距离相同, 分数就相同        │
│    → 这符合语言规律: "猫吃鱼"中 "猫"→"吃" 的关系              │
│      不管出现在第 0~2 位还是第 100~102 位, 都应该一样          │
└──────────────────────────────────────────────────────────────────┘
```

### 4.3 多维扩展：64 维向量怎么做旋转？

```
d_k = 64 维的向量, 每 2 维看成 1 个 2D 平面, 共 32 个平面:

  平面0:  (q₀,  q₁)  →  旋转角度 m·θ₀
  平面1:  (q₂,  q₃)  →  旋转角度 m·θ₁
  平面2:  (q₄,  q₅)  →  旋转角度 m·θ₂
  ...
  平面31: (q₆₂, q₆₃) →  旋转角度 m·θ₃₁

  频率 θ_j = 1 / 10000^(2j/64):
    θ₀  = 1 / 10000^(0/64)   = 1.0       → 旋转最快, 区分相邻位置
    θ₁  = 1 / 10000^(2/64)   = 0.749
    θ₂  = 1 / 10000^(4/64)   = 0.562
    ...
    θ₃₁ = 1 / 10000^(62/64)  = 0.0001    → 旋转最慢, 感受远距离关系

  ┌──────────────────────────────────────────────────────┐
  │  为什么频率不同?                                      │
  │                                                      │
  │  位置差 = 1:                                          │
  │    θ₀=1.0:   旋转 1.0 弧度 (57°)   → 变化大        │
  │    θ₃₁=0.0001: 旋转 0.0001 弧度 (0.006°) → 几乎没变│
  │                                                      │
  │  位置差 = 1000:                                       │
  │    θ₀=1.0:   旋转 1000 弧度 → 已转很多圈, 混乱     │
  │    θ₃₁=0.0001: 旋转 0.1 弧度 (5.7°) → 仍有区分度   │
  │                                                      │
  │  结论:                                                │
  │    高频 → 精确编码近距离 (1~10 步)                   │
  │    低频 → 粗略编码远距离 (100~1000 步)               │
  │    多尺度频率 = 同时感知近处和远处                    │
  └──────────────────────────────────────────────────────┘
```

### 4.4 多个 token 时怎么旋转？每个 token 独立旋转，角度不同

```
  输入句子: "猫 吃 鱼"  → 3 个 token

  核心思路: 不是旋转整个 (3, 768) 矩阵, 而是每个 token 的向量独立旋转
  旋转角度 = 该 token 的位置号 × 频率θ

  ┌──────────────────────────────────────────────────────────────┐
  │  Step 1: embedding → 拆多头 → 每个 token 有 (12头, 64维)    │
  │                                                              │
  │    "猫" 位置0:  embedding_猫 → w_q → (12, 64)              │
  │    "吃" 位置1:  embedding_吃 → w_q → (12, 64)              │
  │    "鱼" 位置2:  embedding_鱼 → w_q → (12, 64)              │
  │                                                              │
  │  Step 2: 每个头的 64 维拆成 32 对 × 2维                     │
  │                                                              │
  │    "猫" 的头0: [(q₀,q₁), (q₂,q₃), ..., (q₆₂,q₆₃)]         │
  │    "吃" 的头0: [(q₀,q₁), (q₂,q₃), ..., (q₆₂,q₆₃)]         │
  │    "鱼" 的头0: [(q₀,q₁), (q₂,q₃), ..., (q₆₂,q₆₃)]         │
  │                                                              │
  │  Step 3: 每对 2D 坐标独立旋转, 角度 = 位置号 × 该对频率θⱼ  │
  │                                                              │
  │    "猫" 位置0: 每对旋转 0×θⱼ = 0     → 不转                │
  │      (q₀,q₁) → 还是 (q₀,q₁)                                │
  │      (q₂,q₃) → 还是 (q₂,q₃)                                │
  │                                                              │
  │    "吃" 位置1: 每对旋转 1×θⱼ                                 │
  │      (q₀,q₁) 旋转 θ₀=1.0 弧度 → (q₀', q₁')  方向变了      │
  │      (q₂,q₃) 旋转 θ₁=0.749 弧度 → (q₂', q₃') 方向变了     │
  │                                                              │
  │    "鱼" 位置2: 每对旋转 2×θⱼ                                 │
  │      (q₀,q₁) 旋转 2θ₀=2.0 弧度 → (q₀'', q₁'')  转更多     │
  │      (q₂,q₃) 旋转 2θ₁=1.498 弧度 → (q₂'', q₃'') 转更多    │
  │                                                              │
  │  32 对都用不同频率θⱼ, 所以每对旋转角度不同:                 │
  │    高频对 (θ₀=1.0):   位置差1 → 角度差1.0 → 区分度高       │
  │    低频对 (θ₃₁=0.0001): 位置差1 → 角度差0.0001 → 区分度低  │
  │    → 高频编码近距离, 低频编码远距离                          │
  │                                                              │
  │  结果:                                                       │
  │    3 个 token 的 64 维向量方向各不相同                       │
  │    → 点积时能区分位置                                       │
  │    → 且点积只取决于位置差 (相对位置), 不取决于绝对位置       │
  │                                                              │
  │  代码怎么实现的?                                             │
  │    cos_cached: (max_len, 64) ← 预计算的 cos 值              │
  │    sin_cached: (max_len, 64) ← 预计算的 sin 值              │
  │                                                              │
  │    q: (B, 12, 3, 64)         ← 3个token, 64维              │
  │    cos_cached[0:3]: (3, 64)  ← 3个位置的cos值               │
  │    unsqueeze → (1, 1, 3, 64) ← 广播到所有batch和head        │
  │                                                              │
  │    q_rotated = q * cos + rotate_half(q) * sin                │
  │    → 逐元素乘, 形状不变 (B, 12, 3, 64)                      │
  │    → 不需要循环! PyTorch 广播自动完成逐 token 旋转           │
  └──────────────────────────────────────────────────────────────┘
```

### 4.5 用模拟数值走一遍旋转

```
假设 d_k=4 (简化), 3 个 token, 只看平面0:

  θ₀ = 1.0

  原始 q (未经旋转):
    位置0: q[0] = [0.5, 0.3, ...]     ← (q₀, q₁) = (0.5, 0.3)
    位置1: q[1] = [0.2, -0.1, ...]
    位置2: q[2] = [-0.4, 0.6, ...]

  旋转后 q' (对平面0 旋转 m·θ₀):
    位置0: m=0, 旋转 0 弧度:
      q'₀ = 0.5·cos(0) - 0.3·sin(0) = 0.5·1 - 0.3·0 = 0.5
      q'₁ = 0.5·sin(0) + 0.3·cos(0) = 0.5·0 + 0.3·1 = 0.3
      → 没变! 位置0 不旋转

    位置1: m=1, 旋转 1.0 弧度 (≈57°):
      cos(1.0) ≈ 0.540,  sin(1.0) ≈ 0.841
      q'₀ = 0.2·0.540 - (-0.1)·0.841 = 0.108 + 0.084 = 0.192
      q'₁ = 0.2·0.841 + (-0.1)·0.540 = 0.168 - 0.054 = 0.114
      → 旋转了! 向量方向变了

    位置2: m=2, 旋转 2.0 弧度 (≈115°):
      cos(2.0) ≈ -0.416,  sin(2.0) ≈ 0.909
      q'₀ = (-0.4)·(-0.416) - 0.6·0.909 = 0.166 - 0.545 = -0.379
      q'₁ = (-0.4)·0.909 + 0.6·(-0.416) = -0.364 - 0.250 = -0.614
      → 旋转更多!

  同样旋转 k:

  现在算注意力分数 q'·k'^T:
    位置2 的 q' · 位置1 的 k'  (距离 = 2-1 = 1)
    位置1 的 q' · 位置0 的 k'  (距离 = 1-0 = 1)

    两个距离都是 1, 旋转角度差都是 1·θ₀
    → 点积结果只取决于距离，不取决于绝对位置
    → 这就是相对位置编码!
```

### 4.6 代码中的高效实现：_rotate_half

```
逐对旋转太慢 (要循环 32 次), 代码用了一种向量化技巧:

  数学上的旋转 (对第 j 对):
    q'_{2j}   = q_{2j}·cos(mθ_j) - q_{2j+1}·sin(mθ_j)
    q'_{2j+1} = q_{2j}·sin(mθ_j) + q_{2j+1}·cos(mθ_j)

  代码把 64 维分成前半和后半:
    前半: [q₀, q₁, q₂, q₃, ..., q₃₁]  (32个, 对应各对的第1个)
    后半: [q₃₂, q₃₃, ..., q₆₃]          (32个, 对应各对的第2个)

  _rotate_half 做什么?
    x = [q₀, q₁, q₂, ..., q₃₁, q₃₂, q₃₃, ..., q₆₃]
                      前半               后半
    返回: [-q₃₂, -q₃₃, ..., -q₆₃, q₀, q₁, ..., q₃₁]
                       -后半                   前半

  然后:
    q' = q * cos + rotate_half(q) * sin

    对前半的 qⱼ:
      q'ⱼ = qⱼ·cosⱼ + (-qⱼ₊₃₂)·sinⱼ = qⱼ·cos(mθⱼ) - qⱼ₊₃₂·sin(mθⱼ)
      → 就是旋转公式!

    对后半的 qⱼ₊₃₂:
      q'ⱼ₊₃₂ = qⱼ₊₃₂·cosⱼ₊₃₂ + qⱼ·sinⱼ₊₃₂ = qⱼ₊₃₂·cos(mθⱼ) + qⱼ·sin(mθⱼ)
      → 也是旋转公式!

  数值示例 (d_k=8):
    q = [a, b, c, d, e, f, g, h]
    前半: [a, b, c, d]    后半: [e, f, g, h]
    rotate_half(q) = [-e, -f, -g, -h, a, b, c, d]

    q * cos:
      [a·cos₀, b·cos₁, c·cos₂, d·cos₃, e·cos₀, f·cos₁, g·cos₂, h·cos₃]
    rotate_half(q) * sin:
      [-e·sin₀, -f·sin₁, -g·sin₂, -h·sin₃, a·sin₀, b·sin₁, c·sin₂, d·sin₃]

    q' = 上面两行相加:
      前半: [a·cos₀ - e·sin₀, b·cos₁ - f·sin₁, c·cos₂ - g·sin₂, d·cos₃ - h·sin₃]
      后半: [e·cos₀ + a·sin₀, f·cos₁ + b·sin₁, g·cos₂ + c·sin₂, h·cos₃ + d·sin₃]

    即每对 (a,e) (b,f) (c,g) (d,h) 都做了 2D 旋转! ✓
```

### 4.7 完整维度流转 (RoPE 模式)

```
  token_ids: (2, 5)                         ← 2个样本, 5个token
       │
       ▼  Embedding(68, 768) 查表
  x: (2, 5, 768)
       │
       ▼  × √768 ≈ 27.7                    ← 缩放 (和正弦模式一样)
  x: (2, 5, 768)
       │
       ▼  Dropout (注意: 不加位置编码!)      ← 和正弦模式的区别!
  x: (2, 5, 768)                            ← 此刻没有位置信息
       │
       ▼  进入第 1 层 DecoderBlock
       │    ┌──────────────────────────────────────────────────┐
       │    │  norm1 = LayerNorm(x)     → (2, 5, 768)         │
       │    │                                                │
       │    │  线性投影:                                     │
       │    │    w_q(normed): (2, 5, 768) → (2, 5, 768)      │
       │    │    拆多头: .view(2, 5, 12, 64)                  │
       │    │           .transpose(1,2) → (2, 12, 5, 64)      │
       │    │    q: (2, 12, 5, 64)  ← 此时还没有位置信息!     │
       │    │    k: (2, 12, 5, 64)                            │
       │    │    v: (2, 12, 5, 64)                            │
       │    │                                                │
       │    │  ★ RoPE 旋转 ★ ← 在这里注入位置信息!           │
       │    │    q, k = self.rope(q, k)                       │
       │    │    q_rotated: (2, 12, 5, 64)  ← 形状不变       │
       │    │    k_rotated: (2, 12, 5, 64)  ← 形状不变       │
       │    │    v: (2, 12, 5, 64) ← 不旋转! v 不需要位置    │
       │    │                                                │
       │    │  注意力计算:                                   │
       │    │    scores = q_rotated @ k_rotated^T / √64       │
       │    │      (2, 12, 5, 64) @ (2, 12, 64, 5)           │
       │    │      → (2, 12, 5, 5)                           │
       │    │      此时分数里已经编码了相对位置!              │
       │    │    ... softmax, @ v, 拼头, w_o ...              │
       │    └──────────────────────────────────────────────────┘
       │
       ▼  进入第 2 层 DecoderBlock           ← 同样有 RoPE!
       │    ★ 再次旋转 Q 和 K ★
       │    每层都重新注入位置信息 → 永远不会稀释!
       │
       ▼  ... 第 3~10 层, 每层都有 RoPE
       │
  (2, 5, 768)
```

## 6. 为什么 RoPE 只旋转 q 和 k，不旋转 v？

```
注意力公式:
  out = softmax(q'·k'^T / √d) × v

  q'·k'^T:  两个旋转向量的点积 → 编码了相对位置信息
            → "token m 和 token n 的距离是 m-n"

  × v:      用位置感知的权重去加权 v 的内容
            → v 本身只是"内容载体"，不需要位置信息
            → 位置信息已经通过 softmax 权重注入了

  如果也旋转 v 会怎样?
    → out = softmax(q'·k'^T) × v'
    → v' 被旋转后，内容向量的方向变了
    → 但 out 的语义是"加权求和的内容"，旋转 v 没有额外好处
    → 反而可能破坏内容信息

  类比:
    q·k 点积 = "我要找什么" × "你有什么" → 匹配度 (需要位置)
    × v      = "按匹配度取内容"          → 取内容 (不需要位置)
    就像在图书馆找书:
      q·k: 根据书名和位置找到书 (需要知道在哪)
      × v: 把书的内容读出来 (内容本身不需要位置标签)
```

## 7. 频率公式为什么相同？

```
正弦编码:
  PE(pos, 2i) = sin(pos / 10000^(2i/d_model))    i=0..d_model/2-1

RoPE:
  θ_j = 1 / 10000^(2j/d_k)                         j=0..d_k/2-1

  都是 1/10000^(2·索引/维度), 只是索引范围不同

  为什么用 10000 作为基数?
    10000^(0/d) = 1        → 最高频率, 周期 = 2π ≈ 6.28 个位置
    10000^(1/d) ≈ 1.01     → 略低
    ...
    10000^(d/2/d) = 100    → 最低频率, 周期 = 2π×100 ≈ 628 个位置

    这个范围让:
      - 相邻位置在高频维有明显区别
      - 远距离位置在低频维仍有区分
      - 10000 是经验值, 大部分模型都用这个

  为什么维度不同但公式一样?
    正弦编码: 对整个 d_model=768 编码, i 从 0 到 383
    RoPE: 对每个头的 d_k=64 编码, j 从 0 到 31
    但每个头独立旋转, 所以只需要对自己头的维度编码
    → 768/12=64, 每个头 64 维, 32 对频率, 够用了
```

## 8. 推理时的 offset 参数

```
训练时: 一次看完整序列, offset=0
  q: (B, n_heads, seq_len, d_k)
  cos_cached[0:seq_len] → 从位置0开始旋转

推理时: 增量解码, 每次 only 生成 1 个新 token
  已有 token: [t₀, t₁, t₂, t₃]  → 想生成 t₄
  新的 q 只有 1 个 token (t₄), 但它在位置 4

  如果 offset=0, cos_cached[0:1] → 位置0的旋转角度 → 错!
  需要 offset=4, cos_cached[4:5] → 位置4的旋转角度 → 对!

  代码:
    # 训练
    q, k = self.rope(q, k, offset=0)

    # 推理 (transformer.py 中)
    q, k = self.rope(q, k, offset=已生成token数)
```

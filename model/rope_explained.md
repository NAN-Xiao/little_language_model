# RoPE (旋转位置编码) —— 不用复数的详细解释

> 本文用**纯 2D 旋转矩阵**解释 RoPE，不涉及复数。只要懂矩阵乘法就能看懂。

---

## 一、为什么要位置编码?

Transformer 的注意力计算 `q · k` 只比较**内容相似度**，完全不知道 token 在哪个位置。

```
句子 A: "猫 抓 鼠"
句子 B: "鼠 抓 猫"

如果不给位置信息，"猫"在位置0和位置2的向量完全一样。
注意力会认为两个句子没有区别 —— 但语义完全相反!
```

**位置编码**就是给每个位置一个"签名"，让模型知道谁在前谁在后。

---

## 二、核心思想: 用"旋转"给位置签名

### 2.1 2D 旋转矩阵 (初中数学)

把一个 2D 点 `(x, y)` 绕原点旋转 θ 角度:

```
[x']   [cosθ  -sinθ] [x]
[y'] = [sinθ   cosθ] [y]

展开:
x' = x·cosθ - y·sinθ
y' = x·sinθ + y·cosθ
```

**例子**: 把点 `(1, 0)` 旋转 45° (π/4 弧度):

```
cos(45°) = 0.707,  sin(45°) = 0.707

x' = 1×0.707 - 0×0.707 = 0.707
y' = 1×0.707 + 0×0.707 = 0.707

结果: (0.707, 0.707) ← 确实是45度方向
```

### 2.2 关键发现: 旋转后的点积只和"角度差"有关

假设有两个向量 `a` 和 `b`，先把 `a` 旋转 θ_a，把 `b` 旋转 θ_b，然后算点积:

```
(旋转后的 a) · (旋转后的 b) = |a||b| · cos(θ_a - θ_b)
```

**重要**: 结果只取决于**角度差** (θ_a - θ_b)，和各自的绝对角度无关!

**具体例子**:

```
情况1: a 旋转 30°, b 旋转 10° → 角度差 = 20°
情况2: a 旋转 50°, b 旋转 30° → 角度差 = 20°

两种情况点积完全相同!
```

这就是 RoPE 的灵魂:**编码的是"相对位置差"，不是"绝对位置"**。

## 三、如何用旋转编码 token 位置?

### 3.1 把 d_k 维拆成多个 2D 对

d_k=64 的向量太长，不容易想象。先看 d_k=4:

```
向量 q = [q₀, q₁, q₂, q₃]

拆成两对2D:
  对0: (q₀, q₁)
  对1: (q₂, q₃)
```

每对独立旋转，各自有自己的旋转角度。

### 3.2 位置 m 的 token 怎么旋转?

对位置 m 的 token，第 j 对旋转 `m × θ_j` 角度。

```
θ_j 是第 j 对的"基础角度"(也叫频率):
  θ₀ = 1.0       (旋转最快)
  θ₁ = 0.749
  θ₂ = 0.562
  ...
  θ₃₁ = 0.0001   (旋转最慢)

位置 m=2 时:
  对0 旋转: 2 × 1.0 = 2 弧度 (约115°)
  对1 旋转: 2 × 0.749 = 1.498 弧度 (约86°)
```

### 3.3 为什么不同对用不同角度?

**高频 (θ 大)**: 位置差1就转很多 → 能精确区分相邻 token
**低频 (θ 小)**: 位置差100也才转一点点 → 能感受远距离关系

就像钟表:
- 秒针转得快 → 精确到秒
- 分针转得慢 → 看的是分钟级变化
- 时针转得更慢 → 看的是小时级变化

**32 个"钟表"同时工作，编码不同尺度的位置关系**。

## 四、完整数值例子 (d_k=4)

### 4.1 原始向量

```
q (位置2的 token) = [1.0, 0.5, 0.3, 0.8]

拆对:
  对0: (1.0, 0.5)
  对1: (0.3, 0.8)
```

### 4.2 查 cos/sin 表

位置 m=2:

```
对0 (θ₀=1.0):  旋转角度 = 2×1.0 = 2.0 弧度
  cos(2.0) = -0.416,  sin(2.0) = 0.909

对1 (θ₁=0.749): 旋转角度 = 2×0.749 = 1.498 弧度
  cos(1.498) = 0.072,  sin(1.498) = 0.997
```

### 4.3 逐对旋转

**对0 旋转:**
```
[x']   [cos(2.0)  -sin(2.0)] [1.0]   [-0.416  -0.909] [1.0]
[y'] = [sin(2.0)   cos(2.0)] [0.5] = [ 0.909  -0.416] [0.5]

x' = 1.0×(-0.416) - 0.5×0.909 = -0.416 - 0.455 = -0.871
y' = 1.0×0.909 + 0.5×(-0.416) = 0.909 - 0.208 = 0.701

对0旋转后: (-0.871, 0.701)
```

**对1 旋转:**
```
[x']   [0.072  -0.997] [0.3]   [0.3×0.072 - 0.8×0.997]   [0.022 - 0.798]   [-0.776]
[y'] = [0.997   0.072] [0.8] = [0.3×0.997 + 0.8×0.072] = [0.299 + 0.058] = [ 0.357]

对1旋转后: (-0.776, 0.357)
```

### 4.4 拼回旋转后的向量

```
q_rotated = [-0.871, 0.701, -0.776, 0.357]
```

向量方向变了，但长度不变（旋转是"刚体变换"）。

## 五、为什么点积只和"相对距离"有关?

### 5.1 两个位置做点积

```
位置2的 q 旋转了 2θ
位置5的 k 旋转了 5θ

q_rotated · k_rotated = |q||k| · cos(2θ - 5θ)
                      = |q||k| · cos(-3θ)
                      = |q||k| · cos(3θ)
```

结果只和 `|2-5|=3` 有关，和"2和5"本身无关。

### 5.2 验证相对位置性质

```
位置0的 q (旋转0θ) 和位置3的 k (旋转3θ):
  点积 ∝ cos(0θ - 3θ) = cos(-3θ) = cos(3θ)

位置5的 q (旋转5θ) 和位置8的 k (旋转8θ):
  点积 ∝ cos(5θ - 8θ) = cos(-3θ) = cos(3θ)

都是"距离3"，点积一样!
```

这就是**相对位置编码**: 模型学到的是"这两个 token 相距多远"，而不是"它们在句子里的绝对位置"。

这更符合语言规律 —— "猫吃鱼"的关系，不因"猫"出现在第1句还是第100句而改变。

## 六、向量化实现: _rotate_half 的原理

### 6.1 问题: 32 对依次旋转太慢

如果写循环:
```python
for j in range(32):  # 32对
    取第 j 对 (x, y)
    计算 cos(m·θ_j), sin(m·θ_j)
    旋转这对
```

Python 循环太慢。需要一次操作旋转所有对。

### 6.2 巧妙的数据布局

**关键观察**: 旋转公式对每对都是独立的。

```
原始向量 (d_k=4):
  q = [a, c, b, d]

布局:
  前半 [a, c] = 各对的"第一个分量" (对0的x, 对1的x)
  后半 [b, d] = 各对的"第二个分量" (对0的y, 对1的y)

即: (a,b) 是一对, (c,d) 是另一对
```

### 6.3 rotate_half 做什么?

```
输入: [a, c, b, d]

half = 2
x1 = [a, c]   (前半)
x2 = [b, d]   (后半)

输出: [-x2, x1] = [-b, -d, a, c]
```

### 6.4 为什么这样就能旋转?

计算: `q * cos + rotate_half(q) * sin`

```
q       = [a,     c,     b,     d    ]
cos     = [cos₀,  cos₁,  cos₀,  cos₁ ]  ← 每对需要同样的 cos
sin     = [sin₀,  sin₁,  sin₀,  sin₁ ]  ← 每对需要同样的 sin
rotate  = [-b,    -d,    a,     c    ]

逐元素相乘再相加:
[0] = a·cos₀ + (-b)·sin₀ = a·cos₀ - b·sin₀  ← 对0旋转后的 x'
[1] = c·cos₁ + (-d)·sin₁ = c·cos₁ - d·sin₁  ← 对1旋转后的 x'
[2] = b·cos₀ + a·sin₀     = b·cos₀ + a·sin₀  ← 对0旋转后的 y'
[3] = d·cos₁ + c·sin₁     = d·cos₁ + c·sin₁  ← 对1旋转后的 y'
```

**完美匹配旋转公式!** 每对都正确旋转了。

### 6.5 为什么这样快?

所有操作都是 PyTorch 的**逐元素运算**，没有循环:
- `q * cos` → GPU 并行乘
- `rotate_half(q) * sin` → GPU 并行乘
- 相加 → GPU 并行加

32 对同时完成，一次操作搞定。

## 七、为什么只旋转 Q 和 K，不旋转 V?

注意力公式:
```
attn_weights = softmax(q_rotated · k_rotated^T / √d_k)
output = attn_weights × v
```

**位置信息在 q·k 的点积中已经注入**:
- q_rotated · k_rotated = |q||k|cos(角度差)
- 角度差 = (位置_m - 位置_n) × θ

V 只是被"加权混合"的内容载体，本身不需要知道位置 —— 权重已经编码了位置关系。

就像:
- Q 和 K 决定"看谁"(位置决定了看多远)
- V 是"看到的内容"

## 八、RoPE vs 正弦位置编码

| | 正弦编码 | RoPE |
|---|---|---|
| **注入方式** | 输入时加一次: `x = x + PE[pos]` | 每层都旋转 Q,K |
| **位置信息** | 绝对: "我在第5位" | 相对: "我离你3位" |
| **深层稀释** | 是 (10层后位置信息变弱) | 否 (每层重新注入) |
| **公式本质** | 加法 | 旋转 (正交变换) |
| **向量长度** | 改变 | 不变 |

---

## 九、总结: RoPE 的三句话

1. **把高维向量拆成很多 2D 对，每对绕原点旋转一个角度**
2. **不同位置旋转不同角度，不同维度对用不同旋转速度**
3. **旋转后的点积只和"角度差"有关 → 编码相对位置**

最精妙的是第三点: 不需要显式计算"位置差"，旋转这个几何操作本身就自动保证了相对位置性质。

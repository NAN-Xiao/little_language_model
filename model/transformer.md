# Transformer: 推理全流程 — 从用户提问到模型回答

以模拟数据为例：**输入 "问：什么是人工智能？答：" → 输出 "人工智能是计算机科学的一个分支..."**

> 本文档以**推理**为主线。训练是"教模型学会回答"，推理是"模型实际回答问题"。
> 先理解推理，再回头看训练会更清晰——训练只是让推理的每一步变得更准。

---

## 1. 总览: 推理时发生了什么?

```
用户提问                              用户看到回答
┌──────────────┐                     ┌──────────────┐
│"问：什么是人 │                     │ "人工智能是  │
│ 工智能？答："│                     │ 计算机科学的 │
│ token_ids    │                     │ 一个分支..." │
│ (1, 8)       │                     │              │
└──────┬───────┘                     └──────▲───────┘
       │                                    │
       │    ┌───────────────────────────────┘
       │    │  自回归循环: 每次生成1个字, 拼回去, 再生成下一个
       │    │
       ▼    ▼
  ┌──────────────────────────────────────────────────┐
  │  单轮循环 (生成一个字):                           │
  │                                                  │
  │  token_ids (1, seq)                               │
  │       │                                          │
  │       ▼  Embedding + 位置编码                     │
  │  (1, seq, 768)                                    │
  │       │                                          │
  │       ▼  10层 DecoderBlock                        │
  │  (1, seq, 768)                                    │
  │       │                                          │
  │       ▼  LayerNorm → Linear(768→68)               │
  │  logits (1, seq, 68)                              │
  │       │                                          │
  │       ▼  取最后位置 → 采样策略 → 1个新token_id   │
  │  新字!                                           │
  │       │                                          │
  │       ▼  拼接回原序列 → 下一轮循环                │
  └──────────────────────────────────────────────────┘
```

## 2. PyTorch 函数原型

Transformer 内部调用的核心 PyTorch API:

```python
# ═══════════════════ 嵌入层 ═══════════════════

nn.Embedding(num_embeddings: int, embedding_dim: int)
    # token_id → 向量查表
    # 例: Embedding(68, 768)  → id=3 查出 (768,) 向量
    # 输入: (B, seq) 的 int   → 输出: (B, seq, 768) 的 float

# ═══════════════════ 线性变换 ═══════════════════

nn.Linear(in_features: int, out_features: int, bias: bool = True)
    # y = x @ W^T + b
    # 例: Linear(768, 768)   → W: (768, 768), b: (768,)
    # 例: Linear(768, 68)    → 输出投影到词表大小
    # 输入: (..., 768)       → 输出: (..., 768) 或 (..., 68)

# ═══════════════════ 归一化 ═══════════════════

nn.LayerNorm(normalized_shape: int, eps: float = 1e-5)
    # 对最后一个维度做归一化: (x - μ) / √(σ² + eps) * γ + β
    # 例: LayerNorm(768)     → 输入输出形状不变, 值域稳定
    # 为什么用 LayerNorm 不用 BatchNorm?
    #   BatchNorm 依赖 batch 统计量, batch 小时不稳定
    #   LayerNorm 每个样本独立归一化, 不受 batch 大小影响

# ═══════════════════ 激活函数 ═══════════════════

nn.ReLU()
    # max(0, x) — 负数变 0, 正数不变
    # FFN 中间层用: 768→3072→ReLU→768

nn.Dropout(p: float = 0.5)
    # 训练时以概率 p 随机将部分元素置 0, 防止过拟合
    # 推理时自动关闭, 所有元素参与计算

# ═══════════════════ 矩阵运算 ═══════════════════

torch.matmul(input: Tensor, other: Tensor) → Tensor
    # 矩阵乘法, 支持批量: (B, m, k) @ (B, k, n) → (B, m, n)
    # 注意力中: q @ k^T → 分数矩阵

torch.tril(input: Tensor, diagonal: int = 0) → Tensor
    # 取下三角, 上三角置 0
    # 构造因果掩码: tril(ones(5,5)) → 下三角全 1, 保证只看过去

Tensor.masked_fill(mask: Tensor, value: float) → Tensor
    # mask 为 True 的位置填 value
    # 例: scores.masked_fill(mask == 0, float("-inf"))
    #   掩码为 0 的位置填 -inf → softmax 后变 0 → 不看未来

# ═══════════════════ 概率与采样 ═══════════════════

torch.nn.functional.softmax(input: Tensor, dim: int) → Tensor
    # softmax(x_i) = exp(x_i) / Σ exp(x_j), 沿 dim 维度
    # 例: softmax([2.1, 0.8, -inf], dim=-1) → [0.79, 0.21, 0.00]
    # 把原始分数变成概率分布 (每行和为 1)

torch.multinomial(input: Tensor, num_samples: int) → Tensor
    # 按概率分布采样
    # 例: multinomial([0.7, 0.2, 0.1], 1) → 70%概率选索引0
    # 生成时: 从 softmax 概率中采样下一个 token

torch.argmax(input: Tensor, dim: int) → Tensor
    # 取最大值的索引 — 贪心解码
    # 例: argmax([0.1, 0.7, 0.2]) → 1

# ═══════════════════ 形状操作 ═══════════════════

Tensor.view(*shape) → Tensor
    # 改变形状, 不改数据 (必须内存连续)
    # 例: (B, seq, 768).view(B, seq, 12, 64) — 拆多头

Tensor.transpose(dim0: int, dim1: int) → Tensor
    # 交换两个维度
    # 例: (B, seq, 12, 64).transpose(1,2) → (B, 12, seq, 64)

Tensor.contiguous() → Tensor
    # 保证内存连续 — transpose 后内存不连续, view 要求连续
    # 例: x.transpose(1,2).contiguous().view(B, seq, 768)

torch.cat(tensors: list[Tensor], dim: int) → Tensor
    # 沿 dim 维度拼接多个张量
    # 例: torch.cat([prompt_ids, new_id], dim=1) — 生成时拼接新 token

# ═══════════════════ 损失函数 ═══════════════════

nn.CrossEntropyLoss(ignore_index: int = -100)
    # 交叉熵损失: -Σ y_i × log(p_i)
    # ignore_index=-100: label 为 -100 的位置不计算 loss
    # Q&A 训练: 问题部分 label=-100, 只有答案参与 loss
```

## 3. 因果掩码: 推理时为什么不会"偷看"未来?

### 3.1 核心规则

```
语言模型的核心规则: 预测下一个字时, 不能偷看后面的字!

  "问：什么是人工智能？答：人工智能是..."
   ↑预测"什"时 只能看"问"            不能看"么是人工智能？答：..."
     ↑预测"么"时 只能看"问什"          不能看"是人工智能？答：..."
       ↑预测"是"时 只能看"问什么"        不能看"人工智能？答：..."
         ...
                                      ↑预测"人"时 只能看"问：什么是人工智能？答："

  → 因果掩码: 下三角矩阵, 保证每个位置只看自己和之前的位置
  → 推理时: 模型只能基于"已经看到的字"预测下一个字, 绝不偷看
```

### 3.2 掩码长什么样?

```
推理时 input_ids: "问：什么是人工智能？答：" → 8个token
因果掩码 (8×8 下三角):

      问  ：  什  么  是  人  工  答  ← key (被看的)
  问 [1,  0,  0,  0,  0,  0,  0,  0]   ← "问"只能看自己
  ： [1,  1,  0,  0,  0,  0,  0,  0]   ← "："能看"问"、自己
  什 [1,  1,  1,  0,  0,  0,  0,  0]
  么 [1,  1,  1,  1,  0,  0,  0,  0]
  是 [1,  1,  1,  1,  1,  0,  0,  0]
  人 [1,  1,  1,  1,  1,  1,  0,  0]
  工 [1,  1,  1,  1,  1,  1,  1,  0]
  答 [1,  1,  1,  1,  1,  1,  1,  1]   ← "答"能看到全部!

  → "答"这个token做注意力时, 能看到完整问题 "问：什么是人工智能？答："
  → 所以"答"的输出最有资格预测下一个字 → 只取最后位置的 logits

  0 的位置 → 填 -inf → softmax 后变 0 → 完全不看
  1 的位置 → 不变     → softmax 后正常参与

  训练时额外屏蔽 pad (详见后文), 推理时通常只有1个样本, 纯因果掩码即可
```

## 4. 推理全流程: 一个字是怎么"走"出来的

> 这是本文的核心章节。跟踪一个 768 维向量从输入到变成一个汉字的全过程。

### 4.1 Phase 1: Embedding — token_id 变成向量

```
用户输入: "问：什么是人工智能？答："
token_ids: (1, 8) = [[3, 12, 45, 7, 41, 52, 6, 34]]

Embedding(68, 768) 查表:
  id=3  "问" → [0.12, -0.34, 0.56, ...]   ← 768个浮点数
  id=12 "："  → [0.78,  0.23, -0.11, ...]
  id=45 "什" → [-0.15, 0.89,  0.33, ...]
  ...
  id=34 "答" → [0.45, -0.67, 0.33, ...]   ← 我们跟踪这个
  → (1, 8, 768)

  此时 "答" 的向量: [0.45, -0.67, 0.33, ...]
    → 只是一个查表得到的"字典编号", 不含任何上下文
    → 模型还不知道前面是"什么是人工智能？"

× √768 ≈ 27.7 缩放:
  "答" → [12.47, -18.55, 9.14, ...]
  → 值域从 ±0.5 变成 ±10, 和位置编码同量级

+ 位置编码 (位置7):
  "答" → [12.47 + PE₇₀, -18.55 + PE₇₁, 9.14 + PE₇₂, ...]
  → 现在 "答" 的向量包含: 语义("答") + 位置(第7个token)
  → 但仍然不知道上下文! 上下文要靠 Block 里的注意力获取
  → (1, 8, 768)
```

### 4.2 Phase 2: 10层 DecoderBlock — 逐步理解上下文

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  Block 1 — "答" 学会了: 谁在我前面                             │
│                                                                 │
│  子层1: Masked Self-Attention                                   │
│    normed = LayerNorm(x₀)                   → (1, 8, 768)      │
│                                                                 │
│    Q = W_q(normed)  → (1, 8, 768) → view → (1, 12, 8, 64)    │
│    K = W_k(normed)  → (1, 8, 768) → view → (1, 12, 8, 64)    │
│    V = W_v(normed)  → (1, 8, 768) → view → (1, 12, 8, 64)    │
│                                                                 │
│    scores = Q @ K^T / √64                   → (1, 12, 8, 8)   │
│                                                                 │
│    "答"(位置7)的注意力权重 (12个头中的一个):                    │
│      [0.05, 0.08, 0.12, 0.15, 0.20, 0.18, 0.10, 0.12]        │
│       ↑问   ↑：   ↑什   ↑么   ↑是   ↑人   ↑工   ↑答          │
│                                                                 │
│    → "答"花20%注意力看"是", 18%看"人"...                      │
│    → 但这只是第1层, 注意力还很"浅"                             │
│                                                                 │
│    attn_out = softmax(scores) @ V            → (1, 12, 8, 64)  │
│    拼回12个头 → W_o投影                      → (1, 8, 768)     │
│    x₁ = x₀ + attn_out                        → (1, 8, 768)    │
│                                                                 │
│  子层2: FFN                                                     │
│    normed = LayerNorm(x₁)                    → (1, 8, 768)     │
│    ffn_out = Linear(768→3072) → ReLU → Linear(3072→768)       │
│                                                → (1, 8, 768)   │
│    x₁ = x₁ + ffn_out                         → (1, 8, 768)   │
│                                                                 │
│  "答"的向量现在: 知道前面有"什么是人工智能？", 但理解很浅     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
     │ x₁: (1, 8, 768)
     ▼
┌─────────────────────────────────────────────────────────────────┐
│  Block 2~3 — "答" 学会了: 词组和问题类型                       │
│    → "人工智能" 是一个专有名词, 不是"人工"+"智能"              │
│    → "什么是...？" 是定义类问题, 需要解释概念                  │
│    → "答" 的注意力开始集中在关键词上                           │
└─────────────────────────────────────────────────────────────────┘
     │
     ▼
┌─────────────────────────────────────────────────────────────────┐
│  Block 4~6 — "答" 学会了: 回答模式和句式                       │
│    → 定义类问题的典型句式: "XXX是YYY的分支"                    │
│    → "答" 的向量开始为生成 "人工智能是..." 做准备              │
│    → 注意力权重: "是"的权重越来越高 → 模型在组织答案结构      │
└─────────────────────────────────────────────────────────────────┘
     │
     ▼
┌─────────────────────────────────────────────────────────────────┐
│  Block 7~10 — "答" 的向量完全语义化                            │
│    → 768维向量里编码了"要说什么"的信息                        │
│    → 虽然还是768个浮点数, 但内容已经从"查表的编号"            │
│      变成了"我对这个问题的理解 + 我准备给出的回答"             │
│    → 模型"准备好"了, 只等投影到词表选出具体哪个字             │
└─────────────────────────────────────────────────────────────────┘
     │ x₁₀: (1, 8, 768)
     ▼
```

### 4.3 Phase 3: 输出投影 — 768维向量 → 68个分数

```
x₁₀: (1, 8, 768)
     │
     ▼  LayerNorm(768)
(1, 8, 768)                     ← 归一化, 值域稳定在合理范围
     │
     ▼  Linear(768 → 68)        ← 投影到词表大小
logits: (1, 8, 68)              ← 每个位置对68个token的原始分数

  我们只关心最后一个位置 (位置7 "答"):

  logits[0, 7, :] = [1.2, -0.3, 0.5, 3.8, -1.1, ..., 0.2]
                     ↑    ↑    ↑    ↑
                    id0  id1  id2  id3 ...

  这68个数是什么意思?
    每个 id 对应词表里的一个 token
    数值越大 → 模型越认为该 token 应该是"下一个字"
    例: id3=3.8 (最高) → 假设 id3 对应 "人" → 模型最想输出 "人"
```

### 4.4 Phase 4: 采样 — 分数变成一个汉字

```
logits[0, 7, :] = [1.2, -0.3, 0.5, 3.8, -1.1, ..., 0.2]
     │
     ▼  temperature=0.7 缩放
[1.71, -0.43, 0.71, 5.43, -1.57, ..., 0.29]   ← 差距被放大
     │
     ▼  top_k=5: 只保留最大的5个, 其余设 -inf
[-inf, -inf, -inf, 5.43, -inf, ..., -inf, 0.29, ...]
只剩5个候选 (假设: "人", "一", "是", "机", "它")
     │
     ▼  softmax
[0.00, ..., 0.72, ..., 0.15, ..., 0.08, ..., 0.05]
 "人" 72%,  "一" 15%,  "是" 8%,  "机" 5%
     │
     ▼  torch.multinomial 按概率采样
72% 概率选到 "人" → id=67

输出第一个字: "人" ✅
```

### 4.5 Phase 5: 回归 — 拼上新字, 重复整个循环

```
第1轮结束, 序列变成:
  [3, 12, 45, 7, 41, 52, 6, 34, 67]  = "问：什么是人工智能？答：人"

═════════════ 第2轮 ═════════════

  输入: [3, 12, 45, 7, 41, 52, 6, 34, 67]  (9个token)
       │
       ▼  Embedding + 位置编码
  (1, 9, 768)
       │
       ▼  10层 DecoderBlock
  (1, 9, 768)
       │
       ▼  LayerNorm → Linear(768→68)
  logits: (1, 9, 68)
       │
       ▼  取位置8 ("人") 的 logits → 采样 → "工" (id=88)

  序列: [..., 67, 88] = "问：什么是人工智能？答：人工"

═════════════ 第3轮 ═════════════

  输入: [..., 67, 88] (10个token)
  → 10层Block → logits → 位置9 → 采样 → "智"

═════════════ 第4~N轮 ═════════════

  "能" → "是" → "计" → "算" → "机" → "科" → "学" → "的" → "分" → "支" → <EOS>

  检测到 <EOS> → 停止!

═════════════ 完整自回归循环 ═════════════

  ┌────────────────────────────────────────────────────────┐
  │                                                        │
  │  while 没遇到 <EOS> 且 未超 max_len:                   │
  │                                                        │
  │    ① Embedding:  token_ids → (1, seq, 768)            │
  │    ② +位置编码:   加上位置信息                         │
  │    ③ 10层Block:   注意力+FFN, 理解上下文               │
  │    ④ LayerNorm:   归一化                               │
  │    ⑤ Linear:      (1, seq, 768) → (1, seq, 68)        │
  │    ⑥ 取最后位置:  logits[-1] → 68个分数                │
  │    ⑦ 采样策略:    temperature → top_k → top_p → softmax│
  │    ⑧ 采样:        multinomial → 得到1个新token_id      │
  │    ⑨ 拼接:        token_ids = cat([token_ids, new_id]) │
  │    ⑩ 判断:        是否 <EOS>? 是否超长? → 继续/停止    │
  │                                                        │
  └────────────────────────────────────────────────────────┘

  最终回答: "人工智能是计算机科学的分支"
```

### 4.6 为什么每轮都要重算整个序列?

```
  第1轮: "问：什么是人工智能？答：" (8个token)
    "答" 的注意力: 看位置0~7

  第2轮: "问：什么是人工智能？答：人" (9个token)
    "人" 是新加的 → 掩码变成 9×9
    → "答" 现在也能看到 "人" 了!
    → 每个 token 的注意力权重都变了
    → 必须 recompute!

  优化: KV-Cache
    前8个token的 K, V 没变 → 缓存起来
    第2轮只需算新token "人" 的 Q, K, V
    用缓存的 K,V + 新的 K,V 算注意力
    → 省掉大量重复计算, 速度翻几倍
    → 本项目当前未实现 KV-Cache, 每步完整重算
```

### 4.7 KV-Cache 详解

```
═════════════ 先搞清楚: Q, K, V 到底是什么 ═════════════

  W_q, W_k, W_v 确实只是 3 个线性层 (权重永远不变)
  但每个 token 都要过这 3 个层, 产出自己的 Q, K, V:

    token0 "问" → Q₀ = W_q(x₀), K₀ = W_k(x₀), V₀ = W_v(x₀)
    token1 "：" → Q₁ = W_q(x₁), K₁ = W_k(x₁), V₁ = W_v(x₁)
    ...
    token7 "答" → Q₇ = W_q(x₇), K₇ = W_k(x₇), V₇ = W_v(x₇)

  → 不是 3 个向量, 是 8×3 = 24 个向量! (8个token × 3个投影)

  当然, 8个token过Linear确实是1次矩阵乘: (1,8,768)@(768,768) → (1,8,768)
  不是9次分别算, GPU上一步搞定。所以线性投影本身不是瓶颈。

═════════════ 那到底优化了什么? 用形状说清楚 ═════════════

  新 token 进 Block 后当然要和老 token 做注意力!
  但关键是: Q 的 seq_len 和 K/V 的 seq_len 可以不同!

  ── 没有 KV-Cache, Block 1 的形状 ──

    输入 x: (1, 9, 768)           ← 9个token全进来

    LayerNorm(x):    (1, 9, 768)
    Q = W_q(normed): (1, 9, 768) → view → (1, 12, 9, 64)  ← seq=9
    K = W_k(normed): (1, 9, 768) → view → (1, 12, 9, 64)  ← seq=9
    V = W_v(normed): (1, 9, 768) → view → (1, 12, 9, 64)  ← seq=9

    scores = Q @ K^T: (1, 12, 9, 9)   ← 9个query × 9个key = 81个分数
    softmax → attn @ V: (1, 12, 9, 64)
    拼回 + W_o: (1, 9, 768)

    FFN: (1, 9, 768) → 768→3072→768 → (1, 9, 768)  ← 9个token过FFN

    → Block 1 输出 (1, 9, 768), 再进 Block 2, 仍是 (1, 9, 768)...

  ── 有 KV-Cache, Block 1 的形状 ──

    输入 x: (1, 1, 768)           ← 只有1个新token进来!

    LayerNorm(x):    (1, 1, 768)
    Q = W_q(normed): (1, 1, 768) → view → (1, 12, 1, 64)  ← seq=1!
    K_new = W_k:     (1, 1, 768) → view → (1, 12, 1, 64)  ← seq=1
    V_new = W_v:     (1, 1, 768) → view → (1, 12, 1, 64)  ← seq=1

    拼缓存:
    K = [K_cache, K_new] = (1, 12, 8, 64) + (1, 12, 1, 64) → (1, 12, 9, 64)
    V = [V_cache, V_new] = (1, 12, 8, 64) + (1, 12, 1, 64) → (1, 12, 9, 64)
                        ↑老8个从缓存来            ↑1个新算的

    scores = Q @ K^T: (1, 12, 1, 9)   ← 1个query × 9个key = 9个分数!
                                            ↑新token  ↑8老+1新

    → 新 token 确实和老 8 个做了注意力! 但只算了1行!
    → 没缓存时算了9行 (8个老token各自也算了一遍 → 白算)

    softmax → attn @ V: (1, 12, 1, 64)
    拼回 + W_o: (1, 1, 768)

    FFN: (1, 1, 768) → 768→3072→768 → (1, 1, 768)  ← 只有1个token过FFN

    → Block 1 输出 (1, 1, 768), 再进 Block 2, 仍是 (1, 1, 768)...

  ── 对比: 每10层Block的形状 ──

    操作           没有缓存              有缓存
    ──────────────────────────────────────────────────────────
    Block 输入     (1, 9, 768)          (1, 1, 768)
    Q 的 seq_len   9                    1
    K 的 seq_len   9                    9 (8缓存+1新)
    V 的 seq_len   9                    9 (8缓存+1新)
    注意力矩阵     (1, 12, 9, 9)        (1, 12, 1, 9)
                  9行×9列=81个分数      1行×9列=9个分数
    Block 输出     (1, 9, 768)          (1, 1, 768)
    FFN 处理       9个token             1个token

    → 注意: K 和 V 的 seq_len 仍然是 9! 新 token 确实看到了所有老 token!
    → 但 Q 的 seq_len 只有 1! 只算了新 token 对所有人的注意力
    → 老那 8 行注意力矩阵根本不需要 (它们的结果上轮已经用过了)

  ── 为什么老 token 的注意力行不需要重算? ──

    第1轮: token0"问"的注意力是 [1.0, 0, 0, ...]  → 只看自己
    第2轮: token0"问"还是只看自己                   → 因果掩码保证!

    因果掩码: token i 只能看到 token 0..i
    第2轮多了 token8"人", 但 token0 看不到 token8
    → token0 的注意力权重完全不变
    → token0 的注意力输出完全不变
    → token0 的 Block 输出完全不变
    → 所以 token0 的那行注意力根本不需要重算!

═════════════ 有缓存时: 只算1个新token ═════════════

  第1轮: 8个token → 算24个投影 → 把K,V存起来
    K_cache = [K₀, K₁, K₂, ..., K₇]  → (1, 12, 8, 64)   ← 存着!
    V_cache = [V₀, V₁, V₂, ..., V₇]  → (1, 12, 8, 64)   ← 存着!

  第2轮: 只输入新token "人" (1个!)
    Q_new = W_q(x_人)    → (1, 12, 1, 64)   ← 只算1个Q
    K_new = W_k(x_人)    → (1, 12, 1, 64)   ← 只算1个K
    V_new = W_v(x_人)    → (1, 12, 1, 64)   ← 只算1个V

    拼缓存:
    K_full = [K₀...K₇, K_new]  → (1, 12, 9, 64)   ← 8个旧的 + 1个新的
    V_full = [V₀...V₇, V_new]  → (1, 12, 9, 64)

    注意力: Q_new @ K_full^T → (1, 12, 1, 9)    ← "人"对9个token的分数
    加权:   attn @ V_full     → (1, 12, 1, 64)   ← 输出1个token

    → 只算了 3 个投影, 而不是 27 个! 省了 89%

  第3轮: 只输入 "工" (1个)
    K_full = [K₀...K₇, K_人, K_工]  → (1, 12, 10, 64)
    → "工"对10个token算注意力 → 采样 → "智"

═════════════ 每层都要缓存! 10层 = 10份独立的K,V缓存 ═════════════

  对, 每一层都有自己的 K,V 缓存! 因为每层的 K,V 来源不同:

    Block 1: K₁ = W_k(LN(x₀))           ← x₀ 是 Embedding 输出
    Block 2: K₂ = W_k(LN(x₁))           ← x₁ 是 Block1 输出
    Block 3: K₃ = W_k(LN(x₂))           ← x₂ 是 Block2 输出
    ...
    Block 10: K₁₀ = W_k(LN(x₉))         ← x₉ 是 Block9 输出

  每层的 W_k 不同, 输入 x 也不同 → K,V 完全独立 → 必须各缓存各的!

═════════════ 缓存的 K,V vs 新 token 的 K,V 怎么计算? ═════════════

  核心公式完全一样: K = W_k(LN(x)), V = W_v(LN(x))
  区别只在于输入 x 从哪来:

  ┌─────────────────────────────────────────────────────────────────────┐
  │                                                                     │
  │  缓存的 K,V (第1轮已经算好, 存在内存里):                         │
  │                                                                     │
  │    Block 1 的缓存: K₁_cache = W_k(LN(x_embedding))               │
  │                   V₁_cache = W_v(LN(x_embedding))               │
  │                   → x 来自 Embedding 层, 第1轮算的, 直接读        │
  │                                                                     │
  │    Block 2 的缓存: K₂_cache = W_k(LN(x_block1_output))           │
  │                   V₂_cache = W_v(LN(x_block1_output))           │
  │                   → x 来自 Block1 的输出, 第1轮算的, 直接读      │
  │                                                                     │
  │    Block L 的缓存: K_L_cache = W_k(LN(x_block(L-1)_output))      │
  │                   → 第1轮逐层算过, 现在从内存读取, 零计算!       │
  │                                                                     │
  │  新 token "人" 的 K,V (第2轮逐层计算):                            │
  │                                                                     │
  │    Block 1: K₁_new = W_k(LN(x_new_embedding))                    │
  │            V₁_new = W_v(LN(x_new_embedding))                    │
  │            → x 来自 Embedding("人") = (1,1,768)                  │
  │            → 这一轮新算的!                                         │
  │                                                                     │
  │    Block 2: K₂_new = W_k(LN(x_new_block1_output))               │
  │            V₂_new = W_v(LN(x_new_block1_output))               │
  │            → x 来自 Block1 对"人"的输出 = (1,1,768)              │
  │            → 依赖 Block1 先算完! 所以必须逐层算                   │
  │                                                                     │
  │    Block L: K_L_new = W_k(LN(x_new_block(L-1)_output))          │
  │            → 依赖上一层先算完, 所以新 token 必须逐层串行         │
  │                                                                     │
  └─────────────────────────────────────────────────────────────────────┘

  逐层对比表:

    层       缓存的 K,V 输入 x          新 token "人"的 K,V 输入 x     计算方式
    ─────────────────────────────────────────────────────────────────────────
    Block1   Embedding("什/么/是/...")   Embedding("人")                新算
    Block2   Block1 的输出(第1轮)        Block1 对"人"的输出(第2轮)    新算
    Block3   Block2 的输出(第1轮)        Block2 对"人"的输出(第2轮)    新算
    ...      ...                         ...                            ...
    Block10  Block9 的输出(第1轮)        Block9 对"人"的输出(第2轮)    新算

  新 K,V 怎么拼进 KV-Cache? — 就是在序列维度上 torch.cat:

    K_cache:  (1, 12, 8, 64)  ← 上一轮存的, 8 个旧 token
    K_new:    (1, 12, 1, 64)  ← 这一轮新算的, 1 个新 token

    K_full = torch.cat([K_cache, K_new], dim=2)   ← 沿 seq_len 维(dim=2)拼接
           → (1, 12, 9, 64)                        ← 9 个 token 的 K

    V 同理:
    V_full = torch.cat([V_cache, V_new], dim=2) → (1, 12, 9, 64)

    拼完之后, 用 K_full 和 V_full 做注意力:
      Q_new:     (1, 12, 1, 64)   ← 只有"人"1个 token 的查询
      K_full^T:  (1, 12, 64, 9)   ← 9 个 token 的键
      Q @ K^T:   (1, 12, 1, 9)    ← "人"对9个token的注意力分数

    每一轮结束后, 把 K_full 和 V_full 存回缓存, 下一轮就能接着拼:
      第3轮: K_cache=(1,12,9,64) + K_new=(1,12,1,64) → (1,12,10,64)
      第4轮: (1,12,10,64) + (1,12,1,64) → (1,12,11,64)
      ...序列越来越长, 缓存也越拼越大

  关键点:
    ① 公式相同: 都是 K=W_k(LN(x)), V=W_v(LN(x)), 只是输入 x 不同
    ② 缓存的 K,V: 从内存读取, 不需要计算 (第1轮已经算过了)
    ③ 新 token 的 K,V: 必须逐层计算, 因为每层依赖上一层的输出
    ④ 拼接方式: torch.cat([旧缓存, 新K], dim=2), 序列维度往后追加

  第2轮, 新 token "人" 逐层走过的完整流程:

    ┌───────────────────────────────────────────────────────────┐
    │  Block 1:                                                │
    │    输入: x_new(1,1,768) 来自 Embedding("人")            │
    │    Q_new = W_q(LN(x_new))   → (1, 12, 1, 64)            │
    │    K_new = W_k(LN(x_new))   → 拼入 K_cache₁              │
    │    V_new = W_v(LN(x_new))   → 拼入 V_cache₁              │
    │    注意力: Q_new @ K_full^T → (1, 12, 1, 9)             │
    │    FFN → 输出 (1,1,768) ← 这是 Block1 对"人"的输出      │
    └───────────────────────────────────────────────────────────┘
         │ 这个输出变成 Block2 的输入
         ▼
    ┌───────────────────────────────────────────────────────────┐
    │  Block 2:                                                │
    │    输入: x_new(1,1,768) 来自 Block1 对"人"的输出        │
    │    Q_new = W_q(LN(x_new))   → (1, 12, 1, 64)            │
    │    K_new = W_k(LN(x_new))   → 拼入 K_cache₂              │
    │    V_new = W_v(LN(x_new))   → 拼入 V_cache₂              │
    │    注意力: Q_new @ K_full^T → (1, 12, 1, 9)             │
    │    FFN → 输出 (1,1,768)                                  │
    └───────────────────────────────────────────────────────────┘
         │
         ▼
       ... 同理 Block3~10, 每层用各自的缓存

  10层缓存总览:

    层     K_cache 形状            V_cache 形状            各自独立!
    ──────────────────────────────────────────────────────────
    1      (1, 12, 9, 64)         (1, 12, 9, 64)         Block1 的 K,V
    2      (1, 12, 9, 64)         (1, 12, 9, 64)         Block2 的 K,V
    3      (1, 12, 9, 64)         (1, 12, 9, 64)         Block3 的 K,V
    ...
    10     (1, 12, 9, 64)         (1, 12, 9, 64)         Block10 的 K,V

    总缓存: 10层 × 2(K,V) = 20 组张量
    每组: 12 × 9 × 64 = 6912 个 float
    总计: 20 × 6912 = 138240 个 float ≈ 0.53 MB (fp32)

  为什么每层的缓存内容不同?

    即使输入同一个句子, 每层看到的"世界"不同:
      Block1 的 K₁: 编码了相邻 token 的浅层关系
      Block5 的 K₅: 编码了句子内部的中层关系
      Block10 的 K₁₀: 编码了全局语义的深层关系
    → 每层的 K,V 是该层独有的特征, 不能跨层复用
    → 所以 10 层必须存 10 份

═════════════ 计算量对比 ═════════════

  生成 N 个新token, 初始 prompt 长度 P:

  无缓存: P + (P+1) + ... + (P+N-1) = N×P + N×(N-1)/2  → O(N²)
  有缓存: P + (N-1)                                          → O(N)

  数值示例: P=8, N=100
    无缓存: 5750 次 token 投影计算
    有缓存: 107 次
    → 加速 54 倍!

═════════════ 内存代价 ═════════════

  缓存大小 = 层数 × 2(K,V) × n_heads × seq_len × d_k

  本项目 (10层, 12头, 64维):
    seq=1000 时: 约 122 MB (fp32)

  大模型 (32层, 32头, 128维):
    seq=4096 时: 约 4 GB (fp16)
    → 长上下文的瓶颈是内存, 不是计算!

═════════════ 常见优化 ═════════════

  ┌──────────────────────────────────────────────────────────┐
  │                                                          │
  │  MQA: 所有头共享1组K,V → 缓存从12组变1组, 内存降12倍   │
  │  GQA: 几个头共享1组K,V → Llama 2 用, 平衡速度和质量    │
  │  量化: fp16→int8→int4, 缓存缩小 2~4 倍                 │
  │  滑动窗口: 只缓存最近W个token的K,V (Mistral)            │
  │                                                          │
  └──────────────────────────────────────────────────────────┘

  本项目当前未实现 KV-Cache, 每步都是完整重算
```

## 5. 采样策略: 怎么从分数变成一个字?

```
假设最后位置的 logits = [2.1, -0.5, 0.8, 3.5, -1.0, ...]

═════════════ temperature — 控制随机性 ═════════════

  logits /= temperature

  temperature = 0.1 (低):
    [21.0, -5.0, 8.0, 35.0, -10.0, ...]  ← 差距被放大
    softmax → [0.00, 0.00, 0.00, 1.00, 0.00, ...]  ← 几乎只选最大
    → 确定性强, 保守, 回答稳定但千篇一律
    → 适合: 事实问答 (必须答对)

  temperature = 1.0 (默认):
    [2.1, -0.5, 0.8, 3.5, -1.0, ...]     ← 原始分布
    softmax → [0.09, 0.01, 0.04, 0.41, 0.01, ...]
    → 正常随机性

  temperature = 2.0 (高):
    [1.05, -0.25, 0.4, 1.75, -0.5, ...]  ← 差距被缩小
    softmax → [0.16, 0.06, 0.11, 0.29, 0.05, ...]  ← 更均匀
    → 更随机, 更有创意, 但可能答非所问

═════════════ top_k — 只保留概率最高的 k 个 ═════════════

  top_k = 5:
    原始: 68 个 token 都有可能
    排序后取前5: 把其余 63 个的 logits 设为 -inf
    → 从 top 5 里采样, 避免选到极低概率的 token

  没有 top_k: "什么是人工智能？答：" 可能采样到 "猫"
  有 top_k=5: 只从最合理的 5 个字里选 (如 "人", "一", "模", "机", "它")

═════════════ top_p (nucleus) — 保留累积概率达到 p 的最少 token ═════════════

  top_p = 0.9:
    排序后概率: [0.45, 0.20, 0.15, 0.08, 0.05, 0.03, 0.02, 0.01, 0.01, ...]
    累积:       [0.45, 0.65, 0.80, 0.88, 0.93, ...]
                                     ↑ 93% > 90%
    保留前 5 个, 其余设 -inf

  自适应:
    确定性高时 (如 "中国的首都是？答："): "北" 占90% → 只保留1个
    不确定性高时: 50 个 token 才占 90% → 保留 50 个

═════════════ repetition_penalty — 惩罚已生成的 token ═════════════

  repetition_penalty = 1.5:
    "人" 原始 logit = 2.0 → 2.0 / 1.5 = 1.33  ← 降低
    "是" 原始 logit = 1.5 → 1.5 (没出现过, 不变)
  → 避免重复输出 "人人人..."

═════════════ no_repeat_ngram_size — 禁止重复 n-gram ═════════════

  no_repeat_ngram_size = 3:
    已生成: "中国的首都是北京中国的首都"
    "首都" 后面曾经跟着 "是" → "是" 被禁止!
  → 避免循环 "中国的首都是北京中国的首都是北京..."

═════════════ min_new_tokens — 最少生成几个 token ═════════════

  min_new_tokens = 10:
    前 10 步: logits[:, eos_token_id] = -inf → 不会过早停止
  → 避免 "问：什么是人工智能？答：技术<EOS>" 敷衍回答

═════════════ 采样策略组合 ═════════════

  事实问答: temperature=0.3, top_k=10          → 确定性强, 准确
  正常问答: temperature=0.7, top_p=0.9          → 平衡
  创意写作: temperature=1.5, top_p=0.95          → 多样

  Q&A 示例:
    "什么是人工智能？答："
      temperature=0.3 → "人工智能是计算机科学的一个分支，致力于研究开发模拟人类智能的系统。"
      temperature=1.5 → "人工智能嘛，简单说就是让机器像人一样思考，有点像教电脑学人话~"

  执行顺序: temperature → min_new_tokens → repetition_penalty
           → no_repeat_ngram → top_k → top_p → softmax → 采样
```

## 6. DecoderBlock × 10: 串行工作的细节

### 6.1 单层 DecoderBlock 内部

```
输入 x: (1, seq, 768)

  子层1: Masked Self-Attention
    normed = LayerNorm(x)                          → (1, seq, 768)
    attn_out = MultiHeadAttention(normed, normed, normed, mask) → (1, seq, 768)
    x = x + attn_out                                → (1, seq, 768)
      ↑ 残差连接: 原始x保留, 注意力输出"加在上面"

  子层2: FFN
    normed = LayerNorm(x)                          → (1, seq, 768)
    ffn_out = Linear(768→3072) → ReLU → Linear(3072→768) → (1, seq, 768)
    x = x + ffn_out                                 → (1, seq, 768)
      ↑ 残差连接: 原始x保留, FFN输出"加在上面"

输出 x: (1, seq, 768) — 形状不变, 但信息更丰富
```

### 6.2 10层怎么串行连?

```
Embedding + 位置编码 → x₀: (1, seq, 768)

  Block 1: x₁ = x₀ + Attn(LN(x₀)) + FFN(LN(x₁))
    → 每个 token 学会"相邻 token 的关系"
    → 如 "人" 知道右边是 "工"

  Block 2: x₂ = x₁ + Attn(LN(x₁)) + FFN(LN(x₂))
    → 学会"词组级关系"
    → 如 "人工智能" 是专有名词

  Block 3:
    → 学会"短语级关系"
    → 如 "什么是人工智能" 是定义类问题

  ...

  Block 8-10:
    → 学会"问题→答案的语义映射"
    → "答：" 之后应该给出概念定义

  LayerNorm → 输出

  关键: 每层形状都是 (1, seq, 768), 但信息从"字面"变成"语义"
```

### 6.3 串行的4个好处

```
┌──────────────────────────────────────────────────────────────────┐
│                                                                  │
│  好处1: 逐层抽象 — 从局部到全局                                 │
│    第1层: "人" 知道右边是 "工"                                   │
│    第3层: "人工智能" 被识别为专有名词                             │
│    第5层: "什么是人工智能？" 被识别为定义类问题                  │
│    第10层: 理解问题语义, 准备生成概念解释                       │
│    → 就像思考: 先认字→理解问题→准备回答→生成答案               │
│                                                                  │
│  好处2: 残差连接保证梯度不消失 — 10层也不会崩                   │
│    x_out = x_in + f(LN(x_in))                                   │
│    ∂x_out/∂x_in = 1 + ∂f/∂x_in ≥ 1                             │
│    → 梯度不会指数衰减, 即使 100 层也能训练                     │
│                                                                  │
│  好处3: 每层学不同的注意力模式 — 12头 × 10层 = 120种           │
│    同层不同头: 语法/关键词/语义 各自关注                        │
│    不同层: 近距离/中距离/远距离 各自擅长                        │
│                                                                  │
│  好处4: 模块化 — 改 n_layers 即可扩展                           │
│    小模型6层~40M  中模型10层~85M  大模型32层~7B                 │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

### 6.4 为什么不能并行?

```
  Block 2 的输入 = Block 1 的输出 → 必须等 Block 1 算完
  如果 Block 2 直接看 x₀ (跳过 Block 1):
    → 看不到 Block 1 提取的特征 → 只能从原始 embedding 重新学
    → 和 Block 1 做重复的事 → 浪费参数

  类比: 回答问题必须先理解问题再回答, 不能跳步
```

## 7. Decoder-Only 架构: 为什么一个问题一条序列就够了?

```
┌──────────────────────────────────────────────────────────────────┐
│                                                                  │
│  Encoder-Decoder (如 T5, BART):                                  │
│    ┌──────────┐          ┌──────────┐                            │
│    │ Encoder  │ ─cross─→ │ Decoder  │                            │
│    │ 双向注意力│  attention│ 因果掩码 │                            │
│    │ 问题全文  │          │ 生成答案  │                            │
│    └──────────┘          └──────────┘                            │
│                                                                  │
│  Decoder-Only (如 GPT, Llama, 本项目):                           │
│    ┌──────────────────────────────────────────┐                  │
│    │  [问题token] [答案token] 全部拼在一起     │                  │
│    │  因果掩码: 答案能看到问题, 问题看不到答案  │                  │
│    └──────────────────────────────────────────┘                  │
│                                                                  │
│  为什么 Decoder-Only 成为主流?                                    │
│  1. 更简单 — 一套注意力机制, 不需要 cross-attention             │
│  2. 更通用 — 续写/问答/翻译/摘要 全是同一种格式                 │
│  3. 更好训 — 所有参数同一方向优化, 没有编码器-解码器对齐问题    │
│  4. 更高效 — KV-Cache 推理时可复用问题的缓存                   │
│                                                                  │
│  核心: 所有任务统一成 "给定前文, 预测下一个token"               │
│    续写: "今天天气" → 预测 "真好"                                │
│    问答: "问：什么是AI？答：" → 预测 "人工智能是..."             │
│    翻译: "翻译为英文：你好" → 预测 "Hello"                      │
│    → 本质都一样, 只是 prompt 格式不同!                           │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

## 8. 训练: 模型怎么学会推理的?

> 推理是"做", 训练是"学"。理解了推理, 训练只是让推理的每一步更准。

### 8.1 训练 vs 推理的区别

```
┌──────────────┬─────────────────────────┬─────────────────────────┐
│              │ 训练 (forward)           │ 推理 (generate)         │
├──────────────┼─────────────────────────┼─────────────────────────┤
│ 输入         │ 完整的问答对             │ 只有问题                │
│              │ "问：XXX？答：YYY"       │ "问：XXX？答："         │
│ 输出         │ 每个位置的 logits        │ 一个新 token_id         │
│ 循环         │ 一次前向, 算所有位置     │ 自回归, 每次生成1个字   │
│ 损失         │ 答案部分算 CrossEntropy  │ 无 (不需要损失)         │
│ 梯度         │ 反向传播, 更新参数       │ 无 (参数不动)           │
│ Dropout      │ 开启                    │ 关闭                    │
│ 目的         │ 让 logits 更准           │ 用学好的 logits 采样    │
└──────────────┴─────────────────────────┴─────────────────────────┘
```

### 8.2 Q&A 训练的关键: Prompt Masking

```
  训练输入: "问：什么是人工智能？答：人工智能是计算机科学的分支"
  完整序列, 一次前向, 但只在答案部分算 loss:

  input_ids = [3, 12, 45, 7, 41, 52, 6, 34, 12, 67, 88, 23]
              "问  ：  什  么  是  人  工  智  能  ？  答  ：  人"
  labels    = [-100,-100,-100,-100,-100,-100,-100,-100,-100,-100,-100, 67]
              ← 问题部分全是 -100, 只有答案参与 loss

  为什么要忽略问题部分的 loss?
    1. 模型不需要学会"复制问题"
    2. 梯度集中在"如何回答"上, 训练更高效
    3. 防止参数浪费在记忆问题上

  CrossEntropyLoss(ignore_index=-100): 遇到 -100 直接跳过
```

### 8.3 训练维度流转 (一次前向)

```
  input_ids (2, 5) = [[3,12,45,7,56], [8,3,12,0,0]]

  → make_causal_mask → (2, 5, 5)  ← 含 pad 屏蔽
  → Embedding(68, 768)            → (2, 5, 768)
  → × √768                        → (2, 5, 768)
  → + 位置编码                     → (2, 5, 768)
  → 10层 DecoderBlock             → (2, 5, 768)
  → LayerNorm                     → (2, 5, 768)
  → Linear(768 → 68)              → (2, 5, 68)

  loss = CrossEntropy(logits, targets, ignore_index=-100)
```

## 9. Xavier 初始化

```
初始化为什么重要?
  全0 → 梯度相同 → 学不到不同特征
  太大 → 输出爆炸 → NaN
  太小 → 输出消失 → 梯度为0

Xavier: 让每层的输出方差 ≈ 输入方差
  W ~ Uniform(-√(6/(fan_in + fan_out)), √(6/(fan_in + fan_out)))

  Linear(768, 768):    W ~ Uniform(-0.063, 0.063)
  Linear(768, 3072):   W ~ Uniform(-0.040, 0.040)  ← 升维, 权重更小
```

## 10. 数值总结: 推理时从 token 到输出字的维度变化

```
步骤                    形状                  说明
──────────────────────────────────────────────────────────────────
输入 token_ids          (1, seq)              用户的问题
    │
    ▼ Embedding(68, 768)
查表                    (1, seq, 768)         每个 token → 768维
    │
    ▼ × √768
缩放                    (1, seq, 768)         和位置编码同量级
    │
    ▼ + 位置编码
加位置信息              (1, seq, 768)         语义 + 位置
    │
    ▼ DecoderBlock × 10
10层理解                (1, seq, 768)         从字面→语义的逐步抽象
    │
    ▼ LayerNorm
归一化                  (1, seq, 768)         值域稳定
    │
    ▼ Linear(768 → 68)
输出 logits             (1, seq, 68)          每个位置对68个token的分数
    │
    ▼ 取最后位置 + softmax + 采样
一个新 token_id         标量 int              拼回去, 下一轮循环
```

# 买量与游戏数据名词汇总

> 集中整理买量投放和游戏运营中的所有核心数据名词，方便快速查阅。

## 一、投放/买量指标

### 1.1 成本类

| 缩写     | 全称              | 中文         | 公式                   | 说明               |
| -------- | ----------------- | ------------ | ---------------------- | ------------------ |
| **CPI**  | Cost Per Install  | 单次安装成本 | 总花费 / 安装数        | 获取一个用户的成本 |
| **CPM**  | Cost Per Mille    | 千次展示成本 | 总花费 / (展示数/1000) | 每1000次展示的价格 |
| **CPC**  | Cost Per Click    | 单次点击成本 | 总花费 / 点击数        | 每次点击的价格     |
| **CPA**  | Cost Per Action   | 单次行动成本 | 总花费 / 行动数        | 按特定行为付费     |
| **CPP**  | Cost Per Purchase | 单次购买成本 | 总花费 / 购买次数      | AEO中衡量付费效率  |
| **oCPM** | Optimized CPM     | 优化千次展示 | —                      | Meta的自动出价模式 |

### 1.2 效果类

| 缩写     | 全称                 | 中文             | 公式                   | 说明               |
| -------- | -------------------- | ---------------- | ---------------------- | ------------------ |

| **CVR**  | Conversion Rate      | 转化率           | 安装数 / 点击数        | 点击后安装的比例   |

## 二、游戏内运营指标

### 2.1 用户规模

| 缩写     | 全称                 | 中文       | 说明                 |
| -------- | -------------------- | ---------- | -------------------- |
| **DAU**  | Daily Active Users   | 日活跃用户 | 当日登录的用户数     |
| **WAU**  | Weekly Active Users  | 周活跃用户 | 7日内登录的用户数    |
| **MAU**  | Monthly Active Users | 月活跃用户 | 30日内登录的用户数   |
| **新增** | New Users            | —          | 当日首次安装的用户   |
| **回流** | Returning Users      | —          | 流失后重新登录的用户 | 见下方回流口径说明 |

**回流（Returning Users）统计口径：**

```
核心问题：多久没登录算"流失"？重新登录后怎么算"回流"？

【常见口径】

口径1：基于连续未登录天数
  - 7日流失：连续7天未登录 → 第8天登录 = 7日回流
  - 14日流失：连续14天未登录 → 第15天登录 = 14日回流
  - 30日流失：连续30天未登录 → 第31天登录 = 30日回流

口径2：基于生命周期阶段
  - 新用户回流：注册7天内流失，之后回来
  - 活跃用户回流：活跃期后流失，之后回来
  - 沉默用户回流：长期未登录（如90天），突然回来

口径3：基于回流后行为（严格口径）
  - 弱回流：登录一次即算（通常不用，噪音大）
  - 标准回流：回流后连续登录≥2天
  - 强回流：回流后7日内有付费行为

【不同产品的流失定义】

| 产品类型 | 流失定义 | 回流观察期 |
|----------|----------|------------|
| 超休闲   | 3天未登录 | 3日回流 |
| 中度游戏 | 7天未登录 | 7日回流 |
| SLG      | 14天未登录 | 14日回流 |
| MMORPG   | 30天未登录 | 30日回流 |

【示例】

用户A登录记录：
  Day1: 登录（新用户）
  Day2: 登录
  Day3-Day10: 未登录（连续8天未登录）
  Day11: 登录 ← 7日回流（按7日口径）

注意：
  - Day11登录后，如果Day12又走了，Day19再回来 → 又是7日回流
  - 同一用户可以多次回流，每次回流独立统计
  - 回流用户通常单独统计"回流留存"（回来后又留了多少天）

【回流的价值】

- 回流成本远低于新客获取成本（CPI的1/5-1/10）
- 回流用户付费率通常比新用户高（了解游戏，更容易付费）
- 回流来源：推送/邮件/召回活动/版本更新/社交邀请
```

### 2.4 付费深度指标

| 指标 | 全称 | 公式 | 说明 | SLG 标准 |
|------|------|------|------|----------|
| **付费率** | Paying User Rate | 付费人数 / 总人数 | 所有用户中付费的比例 | 3-8% |
| **付费渗透率** | Purchase Penetration | 某功能付费人数 / 总人数 | 特定付费点（月卡/战令）的购买比例 | 月卡通常 5-15% |
| **付费留存** | Paying User Retention | DayN登录的付费用户 / 首日付费用户 | **曾经付费的用户**第N天还登录的比例 | D7: 20-40%, D30: 10-25% |
| **付费用户LTV** | Paying User LTV | 付费用户总收入 / 付费用户数 | 只算付费用户的终身价值 | $100-500 |
| **付费频次** | Purchase Frequency | 总付费次数 / 付费人数 | 平均每个付费用户付了多少次 | SLG 通常 3-8次 |
| **复购率** | Repurchase Rate | 二次付费人数 / 首充人数 | 首充后再次付费的比例 | >30%及格，>50%优秀 |
| **付费人次** | Purchase Count | 总付费次数（不去重） | 今日发生多少次付费行为 | — |
| **付费人数** | Paying Users | 付费UID数（去重） | 今日有多少人付过钱 | — |
| **付费次数分布** | Purchase Distribution | — | 付1次/2次/3次...的人数占比 | 见下方说明 |

**付费人次 vs 付费人数：**

```
付费人数（去重）：今天有多少人付过钱（同一个人付3次也算1人）
付费人次（不去重）：今天发生了多少次付费行为（同一个人付3次算3次）

关系：付费人次 ≥ 付费人数
      当付费人次 = 付费人数时，说明所有人都只付了一次

示例：
  用户A：今天付了3次（月卡+礼包+战令）
  用户B：今天付了1次（月卡）
  用户C：今天没有付费

  付费人数 = 2（A和B）
  付费人次 = 4（A的3次 + B的1次）
  人均付费次数 = 4 / 2 = 2次

用途：
  - 付费人数看"有多少人愿意掏钱"
  - 付费人次看"大家总共掏了多少次"
  - 两者差距大 → 说明用户付费频次高，深度好
```

**多次付费在各个指标中的体现：**

| 指标 | 多次付费怎么算 | 说明 |
|------|---------------|------|
| **首充率** | 只算第一次 | 首次付费人数 / 总人数 |
| **复购率** | 算第二次及以上 | 二次付费人数 / 首充人数 |
| **付费频次** | 全部计入 | 总付费次数 / 付费人数 |
| **ARPPU** | 全部计入 | 总收入 / 付费人数（含多次付费累积） |
| **LTV** | 全部计入 | 用户终身累计付费 |
| **付费留存** | 只看出不出席 | 不直接体现付费次数，但付费次数多的用户留存更高 |
| **付费人次** | 全部计入 | 去重前的总付费行为数 |

**付费次数分布：**

```
把付费用户按"总共付了多少次"分组：

  付1次：40% → 首充后流失，需要push复购
  付2-3次：30% → 轻度付费，月卡/战令用户
  付4-10次：20% → 中R，活动驱动消费
  付10次+：10% → 大R/鲸鱼，核心收入来源

分析意义：
  - 付1次的用户太多 → 首充礼包吸引力不够，或后续没有付费点承接
  - 付2-3次是健康基座 → 月卡+战令形成循环
  - 付10次+太少 → 缺少刺激大R的付费内容（排行榜/限时礼包）

SLG健康分布参考：
  1次：30-40%
  2-3次：25-35%
  4-6次：15-25%
  7-10次：8-15%
  10次+：5-10%
```

**付费留存 vs 整体留存：**

```
整体留存 = DayN登录总人数 / 首日新增总人数（含白嫖）
付费留存 = DayN登录的付费用户 / 首日付费用户

关键区别：
  - 整体留存看的是"所有新用户"还留多少
  - 付费留存看的是"付过钱的人"还留多少

为什么付费留存高？
  - 沉没成本：花了钱舍不得走
  - 游戏体验好：付费后成长快，正反馈强
  - 社交绑定：付费玩家通常加入联盟，有社交关系

示例：
  首日新增 10,000 人，其中 300 人付费
  Day7：整体登录 800 人，付费用户登录 90 人
  整体留存 = 800 / 10,000 = 8%
  付费留存 = 90 / 300 = 30%  ← 通常是整体留存的 3-5 倍
```

**付费渗透率细分：**

```
广义：付费渗透率 = 付费率（付费人数 / 总人数）

狭义（按功能拆分）：
  月卡渗透率 = 购买月卡人数 / 总人数
  战令渗透率 = 购买战令人数 / 总人数
  首充渗透率 = 完成首充人数 / 总人数
  礼包渗透率 = 购买某礼包人数 / 总人数

意义：
  - 看哪个付费点最受欢迎
  - 优化低渗透率的付费点

【渗透率计算的三种口径】

```
口径1：累计用户口径（大盘口径，最常用）
  月卡渗透率 = 累计购买月卡人数 / 累计注册人数
  
  示例：服务器累计注册 100,000 人
        累计购买月卡 8,000 人
        月卡渗透率 = 8,000 / 100,000 = 8%

  特点：分母大，数值偏低，反映产品整体变现能力
  适用：汇报高层、评估产品健康度

口径2：活跃用户口径（更真实）
  月卡渗透率 = 某周期内购买月卡人数 / 同期活跃人数
  
  示例：本月活跃用户 20,000 人
        本月购买月卡 3,000 人
        月卡渗透率 = 3,000 / 20,000 = 15%

  特点：分母小，数值偏高，排除沉默用户干扰
  适用：评估付费点吸引力、运营活动效果

口径3：批次用户口径（Cohort口径，最精准）
  月卡渗透率 = 某批次新增中购买月卡人数 / 该批次新增人数
  
  示例：第一周新增 5,000 人
        这 5,000 人中购买月卡 600 人
        月卡渗透率 = 600 / 5,000 = 12%

  特点：按同期新增计算，排除老用户干扰
  适用：对比不同投放渠道/素材的质量
```

**三种口径对比：**

| 口径 | 分母 | 月卡渗透率 | 适用场景 |
|------|------|-----------|----------|
| 累计用户 | 累计注册 | 5-10% | 评估产品整体变现 |
| 活跃用户 | MAU/DAU | 10-20% | 评估付费点吸引力 |
| 批次用户 | 某批次新增 | 8-15% | 渠道质量对比 |

**实战建议：**
- 对外汇报用**累计口径**（显得盘子大）
- 内部优化用**活跃口径**（知道活跃的人有多少付费）
- 渠道评估用**批次口径**（排除老用户干扰，公平对比）

示例（同一组数据，三种口径结果不同）：

  服务器累计注册：100,000 人
  本月活跃用户：20,000 人
  本周新增用户：5,000 人
  本月购买月卡：3,000 人
  
  累计口径：3,000 / 100,000 = 3%
  活跃口径：3,000 / 20,000 = 15%
  批次口径：600（本周新增买月卡）/ 5,000 = 12%

  注意：同一个3,000人里，可能包含老用户续费、活跃用户首购、新用户首购
        活跃口径的3,000人 ≠ 批次口径的600人（有交叉但不同）
```

示例：
  总用户 10,000
  月卡购买 800 人 → 月卡渗透率 8%
  战令购买 500 人 → 战令渗透率 5%
  首充完成 300 人 → 首充渗透率 3%

结论：月卡最受欢迎，战令和首充有提升空间
```

### 2.7 用户行为指标

| 指标 | 全称 | 公式 | 说明 | 参考标准 |
|------|------|------|------|----------|
| **Session** | 启动次数 | — | 用户打开应用的次数 | 日均 2-4 次 |
| **Session Length** | 单次时长 | 总时长 / 启动次数 | 平均每次玩多久 | SLG 10-20分钟 |
| **Play Time** | 人均日时长 | 总时长 / 活跃人数 | 每天玩多久 | SLG 30-60分钟 |
| **流失率** | Churn Rate | 1 - 留存率 | 某批用户流失的比例 | 月流失 60-80% |
| **沉默用户** | Dormant User | — | 近期未登录但尚未完全流失的用户 | 7-30天未登录 |
| **完播率** | Completion Rate | 看完数 / 展示数 | 视频广告看完的比例 | >30%及格 |
| **新手完成率** | Tutorial Completion | 完成新手 / 注册数 | 做完新手引导的比例 | >70%及格 |
| **关卡通过率** | Level Pass Rate | 过关人数 / 尝试人数 | 某关卡的通过比例 | 前10关 >80% |

### 2.8 运营活动名词

| 名词 | 说明 |
|------|------|
| **拉新 / UA** | User Acquisition，获取新用户 |
| **召回** | 对流失用户通过推送/活动/投放拉回来 |
| **版本更新** | 新内容/功能/修复，通常带动回流和付费 |
| **赛季 / Battle Pass** | 周期性通行证，付费解锁奖励 |
| **月卡 / 周卡** | 定时发放奖励的订阅制付费点 |
| **首充礼包** | 首次付费的特惠礼包（通常$0.99-$4.99） |
| **限时活动** | 节日/周末等短期付费活动 |
| **累充活动** | 累计充值达到一定金额送奖励 |
| **转盘/抽卡** | 随机奖励的付费玩法（Gacha） |
| **排行榜** | 按战力/消费排名，刺激大R竞争 |
| **联盟战** | SLG核心社交玩法，驱动联盟协作和付费 |

## 三、商业与财务指标

| 指标 | 全称 | 公式 | 说明 |
|------|------|------|------|
| **CAC** | Customer Acquisition Cost | 总获取成本 / 新增用户数 | 获取一个用户的全部成本（含买量+运营） |
| **回收周期** | Payback Period | 累计收入 ≥ 买量成本所需天数 | Day7回收 = 激进，Day30回收 = 稳健 |
| **毛利** | Gross Profit | 收入 - 渠道分成 - 支付手续费 | 通常为收入的 60-70% |
| **毛利率** | Gross Margin | 毛利 / 收入 | 游戏行业通常 60-70% |
| **净利** | Net Profit | 收入 - 所有成本 | 最终赚的钱 |
| **Burn Rate** | 烧钱速度 | 每月总支出 | 公司每月花多少钱 |
| **Runway** | 资金存活时间 | 现金储备 / 月烧钱速度 | 还能活几个月 |
| **垫资** | Working Capital | — | 买量回收前需要垫付的资金 |
| **GMV** | Gross Merchandise Value | 总交易额 | 用户充值的总额（含未消耗部分） |

**CAC 与 CPI 的区别：**

```
CPI = 买量花费 / 安装数（只看买量）
CAC = (买量 + 运营 + 人力 + 其他) / 新增数（看全部）

示例：
  买量 $100K，运营 $30K，人力 $20K
  新增 10,000 人
  CPI = $100K / 10K = $10
  CAC = $150K / 10K = $15
```

## 五、常见混淆概念

### 5.1 ARPU vs ARPPU

```
ARPU = 总收入 / 总用户数（含白嫖）
ARPPU = 总收入 / 付费用户数（只算付费的）

关系：ARPU = ARPPU × 付费率

示例：
  总用户 10,000 人，付费用户 300 人，总收入 $15,000
  ARPU = $15,000 / 10,000 = $1.5
  ARPPU = $15,000 / 300 = $50
  付费率 = 3%
  验证：$50 × 3% = $1.5 ✅
```

### 5.2 ARPU vs ARPDAU

```
ARPU = 总收入 / 总用户数（累计人均，含已流失用户）
ARPDAU = 日收入 / DAU（当日活跃的活人）

关系：ARPDAU ≈ ARPU × (留存衰减系数) —— 不精确，仅供参考

关键区别：
  ARPU 看的是"这辈子"——用户从安装到现在的累计付费
  ARPDAU 看的是"今天"——当天还在玩的人今天花了多少钱

实战意义：
  ARPDAU 高 + 留存低 = 割韭菜模式（Last War）
  ARPDAU 高 + 留存高 = 健康变现（Whiteout）
  ARPDAU 低 + 留存高 = 有潜力，还没找到付费点

示例：
  游戏 A：ARPDAU = $0.5，D30 留存 8% → 长线健康
  游戏 B：ARPDAU = $1.2，D30 留存 3% → 短期暴利但不可持续
```

### 5.3 ROAS vs ROI

```
ROAS = 收入 / 买量成本（只看买量）
ROI = (收入 - 总成本) / 总成本（看全部）

示例：
  收入 $200K，买量 $100K，运营 $50K
  ROAS = 200K / 100K = 200%（买量翻倍）
  ROI = (200K - 150K) / 150K = 33%（整体赚了33%）
```

### 5.4 CPI vs CPP

```
CPI = 获得一次安装的成本
CPP = 获得一次付费的成本

关系：CPP = CPI / 付费率

示例：
  CPI = $15，付费率 3%
  CPP = $15 / 3% = $50
```

### 5.5 AEO vs VO

```
AEO：优化"会付费的人"（0/1，有付费就行）
VO：优化"付大钱的人"（金额，要付得多）

区别：
  AEO 用户：首充 $5-30 的小R
  VO 用户：首充 $100-500 的大R
```

## 六、数据分析方法

### 6.1 Cohort Analysis（同期群分析）

```
定义：按同一批次（如同一天注册）的用户分组，追踪其随时间的表现

用途：
  - 对比不同批次用户的质量（哪天的留存更好）
  - 评估产品更新/运营活动的效果
  - 预测LTV和回收周期

示例：
  Cohort = 2024-01-01 注册的用户
  
  留存追踪：
    Day0: 10,000 人（100%）
    Day1: 3,500 人（35%）
    Day7: 1,200 人（12%）
    Day30: 600 人（6%）

  付费追踪：
    Day0: 200 人付费，收入 $5,000
    Day7: 累计 350 人付费，收入 $12,000
    Day30: 累计 500 人付费，收入 $25,000
```

### 6.2 Funnel Analysis（漏斗分析）

```
定义：追踪用户从A到B到C的转化路径，看每一步流失多少

常见漏斗：
  安装 → 注册 → 新手完成 → 首次付费 → 复购

示例：
  安装: 10,000
    ↓ 注册: 8,000（80%）← 注册转化率
    ↓ 完成新手: 5,600（70%）← 新手完成率
    ↓ 首次付费: 280（5%）← 首充率
    ↓ 二次付费: 84（30%）← 复购率

优化方向：
  - 注册率低 → 优化注册流程（减少步骤、社交登录）
  - 新手完成率低 → 优化新手引导（缩短、增加奖励）
  - 首充率低 → 优化首充礼包（降价、增加吸引力）
```

### 6.3 AARRR（海盗指标）

| 阶段 | 英文 | 中文 | 关注指标 |
|------|------|------|----------|
| **A** | Acquisition | 获取 | CPI、CTR、CVR、新增数 |
| **A** | Activation | 激活 | 注册率、新手完成率、首次登录时长 |
| **R** | Retention | 留存 | D1/D7/D30留存、回流率 |
| **R** | Revenue | 收入 | ARPU、付费率、LTV、ROAS |
| **R** | Referral | 推荐 | K因子（ viral系数 ）、邀请率、分享率 |

```
K因子 = 邀请率 × 每个邀请带来的新用户数

K > 1：产品自增长，每用户带来超过1个新用户
K = 1：维持不变
K < 1：需要持续买量

示例：
  100个用户中，20个发了邀请 → 邀请率 20%
  每个邀请平均带来 3 个新用户
  K = 20% × 3 = 0.6  ← 还需要买量补充
```

### 6.4 归因模型（Attribution Model）

```
问题：用户看了A广告，又点了B广告，最后安装了。算谁的？

【常见归因模型】

Last Click（最后点击）：
  - 功劳全给最后一个点击的广告
  - 最简单，但忽略前面的影响

First Click（首次点击）：
  - 功劳全给第一个点击的广告
  - 强调"拉新"的价值

Linear（线性分摊）：
  - 所有触点平均分摊功劳
  - 公平但可能过于简化

Time Decay（时间衰减）：
  - 越接近转化的触点，功劳越大
  - 符合直觉

Data-Driven（数据驱动）：
  - 用机器学习计算每个触点的实际贡献
  - 最准确但最复杂

View-Through（展示归因）：
  - 用户没点击但看到了广告，之后安装了也算
  - 通常归因窗口 24小时

【归因窗口】

点击归因窗口：点击后 N 天内安装算该广告（通常 7 天）
展示归因窗口：展示后 N 天内安装算该广告（通常 1 天）

示例：
  Day1: 看到 Meta 广告（展示）
  Day2: 点击 Google 广告
  Day3: 点击 TikTok 广告
  Day5: 安装

  Last Click → TikTok 全功劳
  First Click → Meta 全功劳
  Linear → Meta 33% + Google 33% + TikTok 33%
```

### 6.5 用户分群（Segmentation）

```
定义：把用户分成不同群体，针对性分析/运营

常见分群维度：

1. 按价值分群（RFM模型）
   R = Recency（最近消费时间）
   F = Frequency（消费频次）
   M = Monetary（消费金额）

   高R高F高M → 核心用户（重点维护）
   高R低F低M → 新用户（培养消费习惯）
   低R高F高M → 流失大R（召回优先）

2. 按行为分群
   - 活跃用户：DAU常客
   - 沉默用户：7-30天未登录
   - 流失用户：30天+未登录
   - 付费用户：有付费记录
   - 白嫖用户：从未付费

3. 按渠道分群
   - Meta用户 vs Google用户 vs 自然量用户
   - 不同渠道的留存、付费差异巨大

4. 按版本分群
   - 老版本用户 vs 新版本用户
   - 评估版本更新效果
```

### 6.6 LTV预测方法

```
方法1：历史ARPU × 留存曲线积分
  LTV = ARPDAU × (D1留存 + D2留存 + ... + D∞留存)

  简化版：
    LTV ≈ ARPDAU × 平均生命周期天数
    平均生命周期 = 1 / 日流失率

方法2：Cohort累积收入法
  追踪某批次用户每天的收入，累加到DayN
  Day30 LTV = Day30累计收入 / 该批次人数

方法3：付费用户LTV × 付费率
  LTV = 付费用户LTV × 付费率

方法4：机器学习预测（最准确）
  特征：首日行为（时长、关卡、付费等）
  模型：XGBoost / 神经网络
  输出：预测该用户的终身价值
```

### 6.7 异常检测

```
常见需要监控的异常：

- CPI突然飙升 → 素材疲劳/竞价环境变化
- 留存率断崖下跌 → 版本Bug/服务器问题
- 付费率异常高 → 刷量/作弊
- 广告收入骤降 → Fill Rate下降/eCPM降低
- 某渠道CVR为0 → 归因断开/链接失效

检测方法：
  - 同比环比（和昨天比、和上周同比）
  - 移动平均 + 标准差（超过3σ报警）
  - 同比历史同期（排除季节性因素）
```

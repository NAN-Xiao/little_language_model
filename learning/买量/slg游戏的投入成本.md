# SLG 游戏的投入成本与单服生态

> 回答三个核心问题：SLG 算不算大 DAU 游戏？单服需要多少用户？前期盈利需要投入多少、垫资怎么算？

---

## 一、SLG 算不算大 DAU 游戏？

**不算。SLG 是典型的"低 DAU、高 ARPU"游戏。**

### 1.1 各品类 DAU 对比

| 游戏品类 | 典型 DAU | ARPU | 收入模式 |
|---------|---------|------|---------|
| **超休闲** | 100万-1000万 | $0.01-0.05 | 广告变现 |
| **休闲（三消）** | 50万-500万 | $0.1-0.5 | IAP+IAA |
| **MOBA/竞技** | 100万-1000万 | $1-5 | 皮肤/英雄 |
| **MMORPG** | 10万-100万 | $5-20 | 重度氪金 |
| **SLG** | **5,000-50,000** | **$5-50** | 重度氪金 |
| **棋牌** | 10万-100万 | $2-10 | 房卡/道具 |

### 1.2 为什么 SLG DAU 不高？

```
SLG 的核心是"质量"而非"数量"：

❌ 不需要几百万 DAU
   → 服务器承载有限，人太多反而拥挤
   → 地图资源不够分，新手体验差

✅ 需要几万个高价值用户
   → 每人平均付费 $50-500
   → 5% 的大R贡献 60-80% 收入
   → 1 个大R = 100 个白嫖用户

✅ 需要社交生态
   → 联盟需要 30-100 人
   → 服务器需要 20-50 个活跃联盟
   → 联盟战争需要势均力敌的对手
```

### 1.3 SLG 的收入结构

```
假设一个 SLG 服务器有 10,000 DAU：

- 白嫖用户（70%）：7,000 人，付费 $0
- 小R（20%）：2,000 人，人均 $10 = $20,000
- 中R（8%）：800 人，人均 $100 = $80,000
- 大R（2%）：200 人，人均 $1,000 = $200,000

日收入 ≈ $300,000
月收入 ≈ $9,000,000

→ 80% 收入来自 2% 的大R
→ 这就是 SLG 不需要大 DAU 的原因
```

## 二、单服生态需要多少用户？

### 2.1 单服健康生态的最低人数

| 用户类型 | 最低人数 | 理想人数 | 说明 |
|---------|---------|---------|------|
| **注册玩家** | 5,000 | 20,000+ | 进入服务器的总用户数 |
| **活跃玩家（DAU）** | 1,000 | 3,000-5,000 | 每日登录的用户 |
| **活跃联盟数** | 10 | 20-30 | 有活人的联盟 |
| **每联盟活跃人数** | 20 | 30-50 | 能参与联盟战的最低人数 |
| **付费玩家** | 50 | 200-500 | 至少充过钱的 |
| **大R（>$500）** | 3-5 | 10-20 | 服务器收入的核心来源 |

### 2.2 低于最低人数会怎样？

```
"鬼服"的典型症状：

- 联盟招不满人 → 联盟战打不起来 → 核心玩法失效
- 大R找不到对手 → 没有竞争压力 → 付费动力下降
- 资源采集没人抢 → 发展太快 → 内容消耗完 → 流失
- 聊天频道没人说话 → 社交氛围消失 → 玩家觉得"凉了"

一旦进入鬼服状态：
  → 老玩家加速流失
  → 新玩家进来看到没人就退出
  → 服务器进入死亡螺旋
  → 只能合服或关服
```

### 2.3 合服策略

```
当单服 DAU 跌破 1,000 时，通常需要合服：

合服成本：
  - 技术成本：数据迁移、冲突处理
  - 玩家流失：部分玩家因排名下降而弃游
  - 运营压力：处理玩家纠纷

合服时机：
  - 开服 3-6 个月后，自然流失导致 DAU 下降
  - 保留 2-3 个活跃服务器，其余合并
  - 合服后举办"跨服战"等活动刺激付费
```

### 2.4 开服节奏

```
SLG 的典型开服节奏：

Month 1-2：密集开服（每周 2-3 个新服）
  → 新玩家涌入，每个服都能达到健康人数
  → 大R在新服冲榜，付费意愿强

Month 3-6：减少开服（每周 1 个新服）
  → 老服开始流失
  → 准备第一批合服

Month 6+：稳定开服（每月 2-4 个新服）
  → 主要靠老服运营维持
  → 持续合服，保持活跃
```

## 四、垫资如何计算？

### 4.1 核心公式

```
垫资峰值 ≈ 月总支出 × (T + N) / 2

其中：
  月总支出 = 月买量 + 月运营 = A × (1 + c)
  T = 单用户回本周期（月）
  N = 公司整体盈利时间（月）= T × R / (R - 1 - c)
  R = LTV / CPI
  c = 运营成本系数 = 月运营 / 月买量

简化记忆：
  垫资 ≈ 月总支出 × T × (1.2 ~ 2.0)
  R 越大系数越小，R 越小系数越大
```

**为什么旧公式「日买量 × 回本天数 × 1.5」严重低估？**

```
旧公式只算了"单用户回本前"的投入。
但实际上，即使老用户开始回本了，公司还在买新用户，
新用户还在亏损期。所以累计现金流在 T 之后可能继续下降，
直到垫资峰值（通常在 T ~ N 之间）才触底回升。

注意：如果 R 只略大于 1+c，N 会很大，垫资峰值可能出现在 N 附近；
      如果 R 远大于 1+c，垫资峰值可能出现在 T 附近。
```

### 4.4 现金流表示例（中度 SLG）

```
假设：
  日买量 = $10,000 → 月买量 = $300,000
  月运营 = $120,000（c = 0.4）
  单用户回本 T = 4 个月（120天）
  LTV/CPI R = 5

月份   买量支出   运营支出   老用户贡献   当月净现金流   累计现金流
─────────────────────────────────────────────────────────────────────
M1     $300K      $120K      $0          -$420K        -$420K
M2     $300K      $120K      $0          -$420K        -$840K
M3     $300K      $120K      $0          -$420K        -$1,260K
M4     $300K      $120K      $0          -$420K        -$1,680K
M5     $300K      $120K      $240K       -$180K        -$1,860K  ← 垫资峰值
M6     $300K      $120K      $480K       +$60K         -$1,800K
M7     $300K      $120K      $720K       +$300K        -$1,500K
M8     $300K      $120K      $960K       +$540K        -$960K
M9     $300K      $120K      $1,200K     +$780K        -$180K
M10    $300K      $120K      $1,440K     +$1,020K      +$840K    ← 累计回正
M12    $300K      $120K      $1,920K     +$1,500K      +$3,960K

关键节点：
  - 垫资峰值：M5，约 $186万（累计现金流最负）
  - 当月回正：M6，第6个月
  - 累计回正：M10，第10个月

注意：
  M1-M4 老用户贡献为 $0（还没到 T=4 的回本周期）
  M5 开始老用户贡献收入，但当月仍亏 $180K
  M5 累计 -$186万，比 M4 的 -$168万更负，这才是真正的垫资峰值
```

### 4.5 为什么 SLG 垫资这么高？

```
SLG 垫资高的三大原因：

1. 回本周期长
   休闲游戏：7-30 天回本
   SLG：90-180 天回本
   → 资金占用时间 3-6 倍

2. CPI 高
   休闲游戏：$1-3
   SLG：$10-30
   → 每个用户投入 5-10 倍

3. DAU 增长慢
   休闲游戏：7 天就能看出趋势
   SLG：需要 30-90 天才能判断
   → 决策前已经烧了大量钱

综合结果：
  休闲游戏垫资：$50-200 万
  SLG 垫资：$200-1000 万+
```

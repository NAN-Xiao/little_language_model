## 一、LTV 估算

### 1.1 基于早期 ARPU 的估算

**核心公式：**

```
LTV = ARPU_DayN × 倍数

其中倍数根据游戏类型：
  超休闲：3-5 倍
  休闲：5-8 倍
  中度 SLG：10-15 倍
  重度 SLG：15-25 倍
```

**各阶段精度对比：**

| 可用数据 | 估算公式           | 误差范围 | 适用场景             |
| -------- | ------------------ | -------- | -------------------- |
| Day1     | LTV ≈ ARPU_D1 × 30 | ±60%     | 极早期，仅做量级判断 |
| Day3     | LTV ≈ ARPU_D3 × 12 | ±40%     | 早期筛选             |
| Day7     | LTV ≈ ARPU_D7 × 13 | ±25%     | 决定是否继续投       |
| Day15    | LTV ≈ ARPU_D15 × 6 | ±15%     | 决定是否放量         |
| Day30    | LTV ≈ ARPU_D30 × 4 | ±10%     | 精确测算             |

**示例（同一个游戏，不同天数的估算对比）：**

```
中度 SLG，CPI = $15

Day7 数据（早期估算，误差 ±25%）：
  付费率 = 3%，ARPPU = $45
  ARPU_D7 = $45 × 3% = $1.35
  LTV ≈ $1.35 × 13 = $17.55
  LTV/CPI = $17.55 / $15 = 1.17 ⚠️
  → 大概率不成立，但如果留存/付费趋势极好，可小预算观察到 Day15
  → 如果 Day1 留存 < 25% 或首充率 < 1%，建议止损

Day30 数据（更精确，误差 ±10%）：
  付费率 = 5.5%，ARPPU = $55
  ARPU_D30 = $55 × 5.5% = $3.025
  LTV ≈ $3.025 × 5.8 = $17.55
  LTV/CPI = $17.55 / $15 = 1.17 ❌
  → 和 Day7 估算一致，确认项目不成立，止损

注意：Day30 的倍数不是死板的 4，而是根据 Day7 LTV 反推：
  17.55 / 3.025 ≈ 5.8
  这说明该游戏的付费曲线比典型 SLG 更平缓（长尾更短）
```

**另一个游戏的 Day30 估算（对比）：**

```
中度 SLG，CPI = $15

Day30 数据：
  付费率 = 6%，ARPPU = $60
  ARPU_D30 = $60 × 6% = $3.60
  LTV ≈ $3.60 × 4 = $14.40（用典型倍数 4）
  LTV/CPI = $14.40 / $15 = 0.96 ❌
  → 项目不成立，止损
```

### 1.2 基于留存曲线的估算

**幂函数拟合法：**

```
Retention(t) = a × t^(-b)

其中：
  a = Day1 留存率（小数形式，如 0.25）
  b = 衰减系数
  t = 天数
```

**求 b 值：**

```
已知 Day1 留存 R1，Day7 留存 R7：

R7 / R1 = (7/1)^(-b) = 7^(-b)
ln(R7/R1) = -b × ln(7)
b = -ln(R7/R1) / ln(7)

示例：
  R1 = 25%，R7 = 10%
  b = -ln(0.10/0.25) / ln(7) = -ln(0.4) / 1.946 = 0.916 / 1.946 = 0.47
```

**预测任意天数留存：**

```
Retention(t) = 0.25 × t^(-0.47)

预测：
  Day30 留存 = 0.25 × 30^(-0.47) = 0.25 × 0.20 = 5%
  Day90 留存 = 0.25 × 90^(-0.47) = 0.25 × 0.12 = 3%
  Day180 留存 = 0.25 × 180^(-0.47) = 0.25 × 0.09 = 2.25%
```

### 1.3 LTV 积分法（更精确）

```
LTV = Σ(ARPU_Daily × 留存率)

简化版（按月计算）：
  LTV = ARPPU × 付费率 × Σ(月留存率)

示例：
  ARPPU = $100，付费率 = 5%
  月留存：M1=25%, M2=15%, M3=10%, M4=8%, M5=7%, M6=6%
  
  LTV = $100 × 5% × (0.25+0.15+0.10+0.08+0.07+0.06)
      = $5 × 0.71
      = $3.55（6个月）
      
  → 如果 CPI > $3.55，6个月不回本
```

## 二、CPI 估算

### 2.1 基于 CTR 和 CVR 的估算

```
CPI = CPM / (CTR × CVR × 1000)

或：
CPI = CPC / CVR

其中：
  CPM = 每千次展示成本
  CTR = 点击率（小数）
  CVR = 转化率（点击到安装，小数）
  CPC = 每次点击成本
```

**示例：**

```
Meta 美国市场：
  CPM = $15
  CTR = 1.5% = 0.015
  CVR = 30% = 0.30
  
  CPI = $15 / (0.015 × 0.30 × 1000)
      = $15 / 4.5
      = $3.33
```

### 2.2 各平台 CPI 基准

| 平台       | 休闲游戏 | 中度游戏 | SLG    | RPG    |
| ---------- | -------- | -------- | ------ | ------ |
| **Meta**   | $1-3     | $3-8     | $10-25 | $15-35 |
| **Google** | $0.5-2   | $2-6     | $8-20  | $12-30 |
| **TikTok** | $0.3-1.5 | $1-5     | $5-15  | $8-20  |
| **ASA**    | $2-5     | $5-12    | $15-30 | $20-40 |

### 2.3 CPI 上限计算

```
可接受的最大 CPI = LTV / 目标 ROI 倍数

示例：
  LTV = $45
  目标 LTV/CPI = 3
  最大 CPI = $45 / 3 = $15

→ CPI > $15 的项目不投
```

## 三、ROAS 预测

### 3.1 基于留存曲线的 ROAS 预测

```
ROAS_DayN = (ARPU_DayN × 留存率_DayN) / CPI

或更实用：
ROAS_DayN = DayN 累计收入 / CPI
```

**ROAS 增长曲线估算：**

```
SLG 典型 ROAS 曲线：
  Day7 ROAS  ≈ Day30 ROAS × 0.25
  Day15 ROAS ≈ Day30 ROAS × 0.50
  Day30 ROAS ≈ Day90 ROAS × 0.50
  Day60 ROAS ≈ Day90 ROAS × 0.80
  Day90 ROAS ≈ Day180 ROAS × 0.70
```

**示例：**

```
已知 Day30 ROAS = 60%，预测后续：

Day60 ROAS ≈ 60% / 0.50 × 0.80 = 96%
Day90 ROAS ≈ 60% / 0.50 = 120%
Day180 ROAS ≈ 120% / 0.70 = 171%

→ 预计 90 天回本，180 天 ROI = 71%
```

### 3.2 基于对数回归的预测

```
ROAS(t) = a × ln(t) + b

用 Day7 和 Day30 数据求 a, b：
  Day7:  a × ln(7) + b = ROAS_D7
  Day30: a × ln(30) + b = ROAS_D30

解方程组得 a, b，然后预测任意天数
```

## 四、留存曲线拟合

### 4.1 幂函数模型（最常用）

```
R(t) = a × t^(-b)

拟合方法（最小二乘法）：
  1. 取对数：ln(R) = ln(a) - b × ln(t)
  2. 用线性回归求 ln(a) 和 b
  3. a = e^(ln(a))
```

**Python 示例：**

```python
import numpy as np

# 数据
days = np.array([1, 3, 7, 14, 30])
retention = np.array([0.30, 0.18, 0.12, 0.08, 0.05])

# 取对数
log_days = np.log(days)
log_retention = np.log(retention)

# 线性回归
b, log_a = np.polyfit(log_days, log_retention, 1)
a = np.exp(log_a)

# 预测 Day180
r_180 = a * (180 ** b)
print(f"Day180 留存 ≈ {r_180:.2%}")
```

### 4.2 各游戏类型的典型 b 值

| 游戏类型  | b 值范围 | 留存衰减速度 |
| --------- | -------- | ------------ |
| 超休闲    | 0.8-1.2  | 极快         |
| 休闲      | 0.5-0.8  | 快           |
| 中度 SLG  | 0.4-0.6  | 中等         |
| 重度 SLG  | 0.3-0.5  | 慢           |
| 社交/棋牌 | 0.2-0.4  | 极慢         |

### 4.3 留存衰减系数估算

```
经验公式：
  Day7 留存 ≈ Day1 留存 × 0.40
  Day30 留存 ≈ Day7 留存 × 0.45
  Day90 留存 ≈ Day30 留存 × 0.50
  
即：
  R7/R1 ≈ 0.40
  R30/R7 ≈ 0.45
  R90/R30 ≈ 0.50
```

## 五、现金流与垫资估算

### 5.1 垫资峰值公式

**正确公式：**

```
垫资峰值 ≈ 月总支出 × (T + N) / 2

其中：
  月总支出 = 月买量 + 月运营
  T = 单用户回本周期（月）
  N = 公司整体盈利时间（月）= T×R / (R-1-c)

更精确：做现金流表，找累计现金流最低的点
```

**常见错误（严重低估）：**

```
❌ 垫资 ≈ 日买量 × 回本天数 × 1.5

问题：只算了单用户回本前的投入，
      忽略了持续买量导致公司层面继续失血。
      实际垫资通常是这个数的 2-4 倍。
```

**简化估算：**

```
垫资 ≈ 月总支出 × T × (1.5 ~ 2.5)

示例：
  日买量 $15,000 → 月买量 $450K
  月运营 $180K（c=0.4）
  月总支出 = $630K
  回本周期 4 个月，R=3，N≈13

  垫资 ≈ $630K × (4 + 13) / 2 = $535万
  
  → 建议准备 $500-600万
  → 旧公式算出来 $270万，只有实际的一半！
```

```

  建议信用额度 = $1,000,000 - $1,500,000
  
→ 与代理谈判账期（Net30/Net60）
→ 向 Meta/Google 申请大客户信用额度
```

## 六、盈亏平衡点计算

### 6.1 公司层面盈亏平衡

```
公式：N = T×R / (R-1-c)

参数：
  N = 盈利时间（月）
  T = 单用户回本周期（月）
  R = LTV/CPI
  c = 运营成本/买量成本

前提：R > 1 + c
```

**速算表（简化模型）：**

| R   | c=0   | c=0.2 | c=0.4 | c=0.6 |
| --- | ----- | ----- | ----- | ----- |
| 1.5 | 3T    | 15T   | ∞     | ∞     |
| 2.0 | 2T    | 2.5T  | 3.3T  | 5T    |
| 2.5 | 1.67T | 2T    | 2.3T  | 2.9T  |
| 3.0 | 1.5T  | 1.75T | 2T    | 2.3T  |
| 4.0 | 1.33T | 1.5T  | 1.67T | 1.82T |

### 6.2 当月盈亏平衡

```
当月盈亏平衡条件：
  老用户贡献的收入 ≥ 当月买量成本 + 当月运营成本

即：
  已回本用户群数量 × 每群月净利润 ≥ A(1+c)

示例：
  每群月净利润 = $50,000
  当月买量+运营 = $700,000
  需要已回本群数 = $700,000 / $50,000 = 14 个群
  
  如果 T = 6 个月，需要 Month 20 才能达到 14 个群
```

## 七、素材效果估算

### 7.1 IPM（Install Per Mille）估算

```
IPM = CTR × CVR × 1000

或：
IPM = (安装数 / 展示数) × 1000

基准：
  IPM < 5：差素材，淘汰
  IPM 5-15：一般素材
  IPM 15-30：优质素材
  IPM > 30：爆款素材
```

### 7.2 素材生命周期估算

```
素材有效期 ≈ 预算 / (CPM × Frequency上限)

示例：
  单素材预算 = $10,000
  CPM = $15
  Frequency上限 = 3
  
  展示量 = $10,000 / $15 × 1000 = 666,667
  覆盖用户数 ≈ 666,667 / 3 = 222,222
  
  如果日预算 $5,000，该素材跑 2 天就达到上限
```

### 7.3 素材 ROI 估算

```
素材 ROI = 素材带来的LTV / 素材花费

更实用：
  素材盈亏 = IPM × 展示量/1000 × LTV - 素材制作成本 - 投放花费

示例：
  IPM = 20
  展示量 = 1,000,000
  LTV = $15
  素材制作 = $2,000
  投放花费 = $10,000
  
  安装数 = 20 × 1000 = 20,000
  总收入 = 20,000 × $15 = $300,000
  总成本 = $2,000 + $10,000 = $12,000
  素材ROI = ($300,000 - $12,000) / $12,000 = 24 倍
```

## 八、数值估算速查表

### 8.1 SLG 关键数值基准

| 指标       | 止损线 | 及格 | 良好 | 优秀 |
| ---------- | ------ | ---- | ---- | ---- |
| Day1 留存  | <20%   | 25%  | 30%  | 35%  |
| Day7 留存  | <5%    | 10%  | 15%  | 20%  |
| Day30 留存 | <2%    | 4%   | 6%   | 8%   |
| 首充率     | <0.5%  | 1.5% | 3%   | 5%   |
| Day7 ROAS  | <5%    | 10%  | 15%  | 25%  |
| Day30 ROAS | <40%   | 60%  | 80%  | 120% |
| LTV/CPI    | <1     | 1.5  | 2.5  | 4+   |

### 8.2 常见数值关系

```
CPI = CPM / (CTR × CVR × 1000)
LTV = ARPU × 平均生命周期
ROAS = 累计收入 / 买量成本
ARPPU = LTV / 付费率
IPM = CTR × CVR × 1000

核心判断：
  LTV/CPI > 3 → 优质，放量
  LTV/CPI = 2-3 → 可跑，控制节奏
  LTV/CPI = 1-2 → 微利，优化或止损
  LTV/CPI < 1 → 亏损，立即止损
```

### 8.3 快速心算技巧

```
1. CPI 估算：
   CPM $15，CTR 1%，CVR 30%
   → CPI ≈ $15 / (0.01 × 0.3 × 1000) = $15/3 = $5

2. LTV 估算：
   Day7 ARPU $1，中度 SLG
   → LTV ≈ $1 × 13 = $13

3. ROAS 翻倍时间：
   Day7 ROAS 12% → Day30 ≈ 12% × 4 = 48%
   Day30 ROAS 60% → Day90 ≈ 60% × 2 = 120%

4. 垫资估算：
   日买量 $10K，回本 120 天
   → 垫资 ≈ $10K × 120 × 1.5 = $1.8M

5. 盈亏时间：
   T=6月，R=3，c=0.4
   → N = 6×2/0.6 = 20 个月
```

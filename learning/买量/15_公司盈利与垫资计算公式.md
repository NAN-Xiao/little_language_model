> 汇总买量投放中所有涉及公司层面盈利、现金流、垫资的核心公式，附完整推导、示例和速查表。

---

## 参数说明

文档中涉及三个核心参数，理解它们是使用所有公式的前提：

| 参数  | 全称             | 含义                                      | 怎么获得                                                                         |
| ----- | ---------------- | ----------------------------------------- | -------------------------------------------------------------------------------- |
| **T** | 单用户回本周期   | 一个用户从安装到累计收入 ≥ CPI 所需的时间 | 看 DayN ROAS 达到 100% 需要多少天，再换算成月。比如 Day120 回本 → **T = 4 个月** |
| **R** | LTV/CPI 收入倍数 | 一个用户终身价值是获客成本的几倍          | 估算 LTV ÷ CPI。比如 LTV = $220，CPI = $11 → **R = 20**                          |
| **c** | 运营成本系数     | 运营成本占买量成本的比例                  | 月运营成本 ÷ 月买量成本。比如月运营 $200K，月买量 $500K → **c = 0.4**            |

**实际意义：**

- **T 越大** → 回本越慢 → 垫资越久 → 资金风险越高
- **R 越大** → 用户越值钱 → 盈利越快 → 利润空间越大
- **c 越大** → 公司越"重" → 盈亏平衡线越高 → 盈利越难

**三游戏的参数对比：**

| 游戏              | T    | R   | c   | 公司盈利时间 N             |
| ----------------- | ---- | --- | --- | -------------------------- |
| Whiteout Survival | 4月  | 20  | 0.4 | **4.3月** ✅                |
| Age of Origins    | 5月  | 11  | 0.4 | **5.8月** ✅                |
| Last War          | 6月+ | 3.3 | 0.4 | **15.3月** ❌（实际做不到） |

Last War 的 R = 3.3 大于 1+c = 1.4，理论上能回本。但按旧版错误公式（N=T(R-1)/(R-2-c)）算出来15个月；按修正公式（N=TR/(R-1-c)）约 7.4 个月。问题在于 CPI 还在涨、R 还在跌，动态恶化下实际回不了本。

## 二、公式1：公司整体盈利时间

### 2.1 公式

```
          T × R
N = ─────────────
      R - 1 - c

参数：
  N = 公司整体盈利时间（月）
  T = 单用户回本周期（月）
  R = LTV / CPI（收入倍数）
  c = 运营成本系数 = 月运营成本 / 月买量成本

前提条件：R > 1 + c，否则永远不回本

⚠️ 旧版修正说明：此前文档写为 N=T(R-1)/(R-2-c)，临界条件 R>2+c。
  该公式有重复扣除成本的问题，临界条件偏高。修正后临界条件为 R>1+c。
```

### 2.2 公式推导

**用一个具体例子来理解：**

```
假设：
  - 每月固定买量成本 A = $300,000
  - 运营成本系数 c = 0.4，每月运营成本 = $120,000
  - 单用户回本周期 T = 4 个月（用户安装后第4个月末回本）
  - 每个用户最终价值 LTV = 3 × CPI，即 R = 3
```

**Month 7 时（N = 7），各用户群的状态：**

| 用户群 | 购买时间 | 到第7个月已运营 | 状态 | 原因 |
|--------|---------|---------------|------|------|
| 群1 | 第1个月 | 6个月 | ✅ 已回本且有利润 | 6 > T=4，花$300K，赚回$900K，净利$600K |
| 群2 | 第2个月 | 5个月 | ✅ 已回本且有利润 | 5 > T=4，净利$600K |
| 群3 | 第3个月 | 4个月 | ⚠️ 刚好回本 | 4 = T=4，刚好回本，净利润≈$0 |
| 群4 | 第4个月 | 3个月 | ❌ 未回本 | 3 < T=4，还在亏损 |
| 群5 | 第5个月 | 2个月 | ❌ 未回本 | 2 < T=4 |
| 群6 | 第6个月 | 1个月 | ❌ 未回本 | 1 < T=4 |
| 群7 | 第7个月 | 0个月 | ❌ 未回本 | 刚买的 |

**所以：**
- "有利润"的用户群：群1、群2 = **2个**（严格来说）
- "已回本"的用户群：群1、群2、群3 = **3个**
- 公式中用 N - T = 7 - 4 = **3个**，是近似（把刚好回本的群3也算进去了）

```
已回本用户群累计净利润 ≈ (N - T) × (R - 1) × A
  = 3 × (3 - 1) × $300K
  = 3 × 2 × $300K
  = $1,800,000

每个用户群赚回 (R-1) × A：
  花 A = $300K，赚回 R × A = $900K，净利润 = (3-1) × $300K = $600K

累计总支出 = N × A × (1 + c)
  = 7 × $300K × 1.4
  = $2,940,000
  └─ 7个月买量 = 7 × $300K = $2,100K
  └─ 7个月运营 = 7 × $120K = $840K
```

**盈亏平衡时（累计净利润 = 累计总支出）：**

```
(N - T)(R - 1)A = N × A × (1 + c)

代入例子：
  左边 = $1,800,000（已回本群的利润）
  右边 = $2,940,000（总支出）
  $1,800,000 < $2,940,000 → 第7个月还没回本
```

**设盈亏平衡时间为 N，解方程：**

```
(N - 4)(3 - 1) × 300K = N × 300K × 1.4

两边消去 300K：
  (N - 4) × 2 = N × 1.4

展开：
  2N - 8 = 1.4N
  2N - 1.4N = 8
  0.6N = 8
  N = 13.3 个月

→ 这个例子需要约13个月才能公司层面回本

用修正公式验证：
  N = T×R / (R-1-c)
    = 4 × 3 / (3-1-0.4)
    = 12 / 1.6
    = 7.5 个月

（旧版错误公式算出来 13.3 个月，偏保守）
```

**一般化的代数推导（修正版）：**

```
简化模型假设：
  - 已回本用户群累计收入 = (N-T) × R × A
  - 累计总支出 = N × A × (1 + c)

盈亏平衡（累计净利润 = 0）：
  (N - T) × R × A = N × A × (1 + c)

两边消去 A：
  (N - T) × R = N(1 + c)

展开：
  NR - TR = N + Nc
  N(R - 1 - c) = TR

最终结果：
          T × R
  N = ─────────────
        R - 1 - c
```

### 2.3 为什么 R > 1 + c 才能回本？

```
从公式 N = TR/(R-1-c) 可以看出：

当 R = 1 + c 时，分母 R - 1 - c = 0：
  N = TR / 0 → 无穷大（永远不回本）

为什么？
  稳态当月收入 = R × A（所有用户群同时贡献）
  稳态当月支出 = A × (1+c)

  R = 1+c 时：
    稳态当月收入 = 稳态当月支出
    → 当月刚好打平，没有盈余去填历史亏损坑

只有 R > 1 + c 时：
  稳态当月收入 > 每月新增支出
  → 有多余的利润去填前面 T 个月的亏损坑
  → 最终能回本

注意：这是简化模型的结论。实际中回本后用户仍持续贡献，
      未回本用户也有部分收入，精确计算需要做现金流表。
```

## 三、公式2：盈亏平衡 ROAS

### 3.1 公式

```
单用户回本：ROAS_breakeven = 100%

公司层面盈利：ROAS_breakeven = 100% × (1 + c)

其中 c = 运营成本 / 买量成本
```

### 3.2 速查表

| 运营成本比例 c | 单用户回本所需 ROAS | 公司盈利所需 ROAS |
| -------------- | ------------------- | ----------------- |
| 0%             | 100%                | 100%              |
| 20%            | 100%                | 120%              |
| 40%            | 100%                | 140%              |
| 60%            | 100%                | 160%              |
| 80%            | 100%                | 180%              |
| 100%           | 100%                | 200%              |

### 3.3 示例

```
月买量 $500K，月运营 $200K
c = 200/500 = 0.4

公司盈利需要：ROAS > 100% × (1+0.4) = 140%

Whiteout：预测 Day180 ROAS = 210% > 140% → 盈利 ✅
Age of Origins：预测 Day180 ROAS = 160% > 140% → 微利 ✅
Last War：预测 Day180 ROAS = 95% < 140% → 亏损 ❌
```

## 四、公式3：盈利空间

### 4.1 公式

```
盈利空间 = (预测最终 ROAS - 盈亏平衡 ROAS) / 盈亏平衡 ROAS

或：
盈利空间 = (R - (1+c)) / (1+c)

其中 R 是最终 LTV/CPI
```

### 4.2 示例

```
Whiteout：
  R = 20，c = 0.4
  盈利空间 = (20 - 1.4) / 1.4 = 18.6 / 1.4 = 13.3 = 1330%
  
  每投入 $1，最终赚回 $14.3（净利润 $13.3）

Age of Origins：
  R = 11，c = 0.4
  盈利空间 = (11 - 1.4) / 1.4 = 9.6 / 1.4 = 6.9 = 690%
  
  每投入 $1，最终赚回 $7.9（净利润 $6.9）

Last War：
  R = 3.3，c = 0.4
  盈利空间 = (3.3 - 1.4) / 1.4 = 1.9 / 1.4 = 1.36 = 136%
  
  每投入 $1，最终赚回 $2.36（净利润 $1.36）
  → 但这是理想情况，实际上 CPI 还在涨，R 还在跌
```

## 五、公式4：垫资计算

### 5.1 原公式的问题

```
❌ 错误公式（常见但严重低估）：
  垫资 = 日买量 × 回本天数 × 1.5

问题在哪？
  这个公式只算了"单用户回本前"的投入。
  但实际上，即使单用户开始回本了，公司层面还在亏钱——
  因为每天都在买新用户，新用户还在亏损期。

  垫资峰值不是"单用户回本"的时候，
  而是"累计现金流最负"的时候，通常更晚。
```

### 5.2 正确理解：垫资峰值 = 累计现金流最低点

```
持续买量的情况下，现金流走势：

累计现金流
   ↑
 0 ┤                              ●─────── 回正
   │                        ●     │
   │                  ●     │     │
-1M ┤            ●     │     │     │
   │      ●     │     │     │     │
-2M ┤●────┼─────┼─────┼─────┼─────┤
   │      │     │     │     │     │
-3M ┤──────┼─────┼─────┼─────┼─────┤
   │      │     │     │     │     │
-4M ┤──────┼─────┼─────┼─────┼─────┤
   │      │     │     │     │     │
-5M ┤──────┼─────┼─────┼─────┼─────┤ ← 垫资峰值
   │      │     │     │     │     │
   └──────┴─────┴─────┴─────┴─────┴──────→ 时间
       M1    M3    M5    M7    M9   M11

Month 1-T：纯烧钱，没有收入
Month T+1：老用户开始回本，但收入 < 支出，累计继续下降
Month N：当月净现金流 = 0，累计现金流达到最低点（垫资峰值）
Month N+1：当月开始盈利，累计现金流回升
```

### 5.3 精确计算方法：现金流表

**核心原则：做现金流表，找到累计现金流最低的点。**

**示例：每月买量 $50万，持续买量**

```
假设：
  月买量 A = $500,000
  月运营 = $200,000（c = 0.4）
  月总支出 = $700,000
  单用户回本周期 T = 4 个月
  LTV/CPI R = 3
  用户回本后每月贡献利润（平均）= $100,000/群

月份   月支出     老用户贡献   当月净现金流   累计现金流
─────────────────────────────────────────────────────────
M1     $700K      $0          -$700K        -$700K
M2     $700K      $0          -$700K        -$1,400K
M3     $700K      $0          -$700K        -$2,100K
M4     $700K      $0          -$700K        -$2,800K
M5     $700K      $100K       -$600K        -$3,400K
M6     $700K      $200K       -$500K        -$3,900K
M7     $700K      $300K       -$400K        -$4,300K
M8     $700K      $400K       -$300K        -$4,600K
M9     $700K      $500K       -$200K        -$4,800K
M10    $700K      $600K       -$100K        -$4,900K
M11    $700K      $700K       $0            -$4,900K  ← 垫资峰值！
M12    $700K      $800K       +$100K        -$4,800K  ← 开始回升
M13    $700K      $900K       +$200K        -$4,600K

结论：
  - 垫资峰值 = $490万（发生在第11个月）
  - 原公式"日买量×回本天数×1.5" = $168万 → 严重低估！
  - 实际垫资是原公式的 2.9 倍
```

### 5.4 简化公式

**改进后的简化公式：**

```
垫资峰值 ≈ 月总支出 × (回本周期 + 公司盈利时间) / 2

其中：
  月总支出 = 月买量 + 月运营 = A × (1 + c)
  回本周期 T = 单用户回本周期（月）
  公司盈利时间 N = T×R / (R-1-c)

代入上面的例子验证：
  月总支出 = $700K
  T = 4
  N = 4 × 3 / (3-1-0.4) = 7.5

  垫资 = $700K × (4 + 7.5) / 2
       = $700K × 5.75
       = $4,025,000

实际峰值 $490万，简化公式算出来 $402万，略低估约18%。
建议在此基础上加 20-30% 安全边际。
```

**更保守的快速估算：**

```
垫资 ≈ 月总支出 × 公司盈利时间 N × 0.5

或：
垫资 ≈ 月总支出 × T × (1.5 ~ 2.5)

R 越大（用户越值钱），系数越小
R 越小（回本越慢），系数越大
```

### 5.5 示例

**示例1：Whiteout Survival（R很大，垫资少）**

```
日买量 = $15,000 → 月买量 = $450K
月运营 = $180K（c=0.4）
月总支出 = $630K
T = 4个月，R = 20，N = 4.3

精确法（现金流表）：
  Month 1-4：每月亏 $630K，累计 -$2,520K
  Month 5：群1回本，假设月利润 $712K
           当月净现金流 = $712K - $630K = +$82K
  累计在 M4 达到峰值：$2,520K

简化公式：
  垫资 = $630K × (4 + 4.3) / 2 = $2,614K

→ 需要准备 $260万+
→ 因为 R=20 很大，回本快，垫资相对少
```

**示例2：中度 SLG（R一般，垫资中等）**

```
日买量 = $10,000 → 月买量 = $300K
月运营 = $120K（c=0.4）
月总支出 = $420K
T = 4个月，R = 3，N = 13.3

简化公式：
  垫资 = $420K × (4 + 13.3) / 2
       = $420K × 8.65
       = $3,633K

→ 需要准备 $360万+
→ R=3 属于可跑但盈利慢，垫资较大
```

**示例3：Last War 后期（R危险，垫资巨大）**

```
日买量 = $50,000 → 月买量 = $1,500K
月运营 = $600K（c=0.4）
月总支出 = $2,100K
T = 6个月，R = 3.3，N = 15.3

简化公式：
  垫资 = $2,100K × (6 + 15.3) / 2
       = $2,100K × 10.65
       = $22,365K

→ 需要准备 $2200万+ 的垫资
→ 这就是为什么停投——根本垫不起
```

### 5.6 垫资速查表（修正版）

**基于改进公式：垫资 = 月总支出 × (T + N) / 2**

假设 c = 0.4：

| 月买量 | 月总支出 | T=3月 R=3 | T=4月 R=3 | T=6月 R=3 |
|--------|---------|----------|----------|----------|
| **$100K** | $140K | $245万 | $364万 | $546万 |
| **$300K** | $420K | $735万 | $1,091万 | $1,638万 |

### 5.7 为什么 SLG 垫资这么高？

```
SLG 垫资高的三大原因：

1. 回本周期长
   休闲游戏：7-30 天回本
   SLG：90-180 天回本
   → 资金占用时间 3-6 倍

2. CPI 高
   休闲游戏：$1-3
   SLG：$10-30
   → 每个用户投入 5-10 倍

3. 公司盈利时间远晚于单用户回本
   单用户回本后，公司还在亏钱
   因为新用户还在源源不断地进来
   → 实际垫资 = 单用户回本期的 2-5 倍

综合结果：
  休闲游戏垫资：$50-200 万
  SLG 垫资：$200-2000 万+
```

## 六、公式5：月现金流净额

### 6.1 公式

```
月现金流 = 老用户贡献收入 - 当月买量支出 - 当月运营成本

或展开为：
月现金流 = Σ(各月买量用户群 × 当月该群ARPU) - 当月买量 - 当月运营
```

### 6.2 简化模型

```
假设：
  - 每月固定买量成本 A
  - 每月固定运营成本 cA
  - 用户回本周期 T 个月
  - 回本后每群每月净利润 = (R-1)A / 剩余生命周期

简化计算（回本后每群每月稳定贡献利润 P）：
  P = (R-1)A / L
  其中 L = 用户活跃生命周期（月）

月现金流：
  Month N：
    已回本群数 = N - T
    老用户贡献 = (N-T) × P
    当月支出 = A(1+c)
    净现金流 = (N-T) × P - A(1+c)
```

### 6.3 现金流表示例

**Whiteout Survival 现金流预测**

```

  月买量 A = $300,000
  运营成本 cA = $120,000（c=0.4）

  用户活跃生命周期 L = 24 个月

回本后每群月利润：
  P = (20-1) × $300K / 24 = $5,700,000 / 24 = $237,500/月/群
```

## 七、综合决策框架

### 7.1 五步判断法

```
Step 1：拿到核心数据
  - Day30 ROAS
  - CPI
  - LTV（估算）
  - 月运营成本

Step 2：计算 R = LTV / CPI
  - R < 1+c → 永远不回本，止损
  - R = 1+c → 刚好平衡，没有利润空间
  - R > 1+c → 计算具体回本时间

Step 3：计算公司盈利时间
  N = T×R / (R-1-c)

Step 4：计算垫资需求
  垫资 = 日买量 × 回本天数 × 1.5

Step 5：决策
  N < 12月 + 垫资可承受 → 放量
  N = 12-24月 + 垫资可承受 → 谨慎跑
  N > 24月 或 垫资不可承受 → 止损
```

## 八、常见误区

```
❌ 误区1：只看单用户回本，不看公司层面
   → 单用户180天回本 ≠ 公司6个月盈利
   → 公司盈利时间 = 单用户回本 × R/(R-1-c)

❌ 误区2：低估运营成本
   → 月运营 $200K，看起来不多
   → 但累积起来，让盈亏平衡 ROAS 从100%涨到140%

❌ 误区3：垫资算少了
   → 只算"买量花多少钱"，不算运营成本
   → 正确：垫资 = 日买量 × 回本天数 × (1+c)

❌ 误区4：R 刚好等于 1+c 就以为能跑
   → R = 1+c 时 N = 无穷大，永远不回本
   → 需要 R 显著大于 1+c（建议 R > 2+c 才有安全边际）

❌ 误区5：用流水判断盈利
   → Last War 月流水15亿，但公司层面不盈利
   → 用公式算 N 和垫资，才是真判断
```

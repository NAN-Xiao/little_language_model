## SLG 策略游戏买量专题

SLG（Strategy Game，策略游戏）的买量和休闲游戏有本质区别。SLG 更关注"用户质量"而非"用户数量"，因为 SLG 的盈利高度依赖少数大 R 用户。

---

### SLG 3. 冷启动详解

**冷启动（Cold Start）**指的是一款**全新的游戏**在没有任何用户基础、没有历史数据、没有口碑的情况下，刚开始买量投放的阶段。

**冷启动的典型特征：**

```
刚上架的 SLG 游戏：
  ❌ 没有用户数据（算法不知道谁喜欢玩）
  ❌ 没有评分/评论（商店页没说服力）
  ❌ 没有付费数据（不知道 LTV 是多少）
  ❌ 没有素材数据（不知道什么视频能吸量）
  ❌ 没有预算积累（不敢大量花钱）
```

**冷启动期（0-30 天）的目标：**

| 目标 | 说明 |
|------|------|
| **验证产品** | 核心玩法是否吸引人？留存是否正常？ |
| **积累数据** | 让 Meta/Google 的算法学习高价值用户 |
| **测试素材** | 找到 3-5 个能跑正的素材方向 |
| **测算 LTV** | 拿到第一批真实付费数据，预测长期回收 |
| **控制亏损** | 小预算测试，不盲目烧钱 |

**冷启动期的典型预算节奏：**

```
Day 1-7：日预算 $500-1,000
  - 目标：跑出安装，看首日留存
  - 如果 Day1 留存 < 25% → 产品有问题，先改产品

Day 8-14：日预算 $1,000-2,000
  - 目标：积累付费数据，看首充率
  - 如果首充率 < 1% → 付费引导有问题

Day 15-30：日预算 $2,000-3,000
  - 目标：测试 AEO，让平台学习付费用户
  - 如果 AEO 的 CPP 太高 → 继续用 CPI 养数据
```

**为什么冷启动不能用 VO：**

| 要求 | 冷启动期状态 |
|------|------------|
| 每天 50+ Purchase 事件 | ❌ 没有 |
| 稳定日安装量 100+ | ❌ 刚开始 |
| 回传数据积累 1-2 周 | ❌ 刚接入 SDK |

```
冷启动的正确路径：
  CPI/AEO 跑起来 → 每天产生 Purchase 事件 → 回传给 Meta
  ↓
  数据积累 2-4 周
  ↓
  切换 VO，开始找高价值用户
```

**冷启动期最容易犯的错误：**

| 错误 | 后果 |
|------|------|
| 一上来就用 VO | 数据不够，算法学不出来，CPI 飙高 |
| 预算给太猛 | 数据还没验证就烧 $10,000/天，亏光 |
| 素材只做一种 | 测不出方向，不知道用户喜欢什么 |
| 只看 CPI 不看留存 | CPI $5 但留存 5%，来的全是垃圾用户 |
| 7 天没数据就放弃 | 算法学习需要时间，7 天根本不够 |

**冷启动成功的标准：**

```
SLG 冷启动通过的标准：
  ✅ Day1 留存 > 25%
  ✅ Day7 留存 > 8%
  ✅ 首充率 > 1.5%
  ✅ 有 2-3 个素材 CPI <$20
  ✅ AEO 的 CPP 在可接受范围（如 <$50）
  ✅ 日付费事件 > 30 个（准备切 VO）
```

### SLG 3.1 四层漏斗决策法

冷启动期间，根据 Day 3/7/15/30 的数据分阶段决策，层层过滤。

```
Day 1-3 → 产品是否合格？（不合格直接停）
    ↓
Day 7 → 付费模型是否成立？（能初步预测 LTV）
    ↓
Day 15 → 趋势是否向上？（验证早期预测）
    ↓
Day 30 → 能否放量？（最终决定）
```

**第一层：Day 1-3（产品验证）**

| 指标 | 止损线 | 及格 | 良好 | 优秀 |
|------|--------|------|------|------|
| **Day1 留存** | < 20% | 20% | 25% | 30% |
| **Day3 留存** | < 10% | 12% | 15% | 20% |
| **平均游戏时长** | < 10min | 15min | 25min | 40min |
| **教程完成率** | < 50% | 60% | 70% | 80% |

```
Day1 留存 < 20%：
  → 立即停投，改产品（核心玩法/加载/引导有问题）

Day1 留存 20-25%：
  → 小预算继续（$500-1000/天），优化新手引导

Day1 留存 > 25%：
  → 产品合格，进入第二层
```

**第二层：Day 7（付费模型验证）**

| 指标 | 止损线 | 及格 | 良好 | 优秀 |
|------|--------|------|------|------|
| **Day7 留存** | < 5% | 8% | 12% | 17% |
| **首充率** | < 0.5% | 1% | 1.5% | 2.5% |
| **Day7 付费率** | < 1.5% | 2% | 3% | 5% |
| **Day7 ARPPU** | < $20 | $30 | $50 | $80 |

```
Day7 估算 LTV（快速法）：
  Day7 ARPU = Day7 ARPPU × Day7 付费率
  LTV ≈ Day7 ARPU × 倍数（轻度10-12 / 中度12-15 / 重度15-20）

  例子：Day7 ARPPU=$45，付费率3%，中度SLG
  LTV ≈ $45×3%×13 = $17.55

  若 CPI=$15：LTV/CPI=1.17
    → 大概率不成立，但如果留存/付费趋势极好，可小预算观察到 Day15
    → 如果 Day1 留存 < 25% 或首充率 < 1%，建议止损 ⚠️
  若 CPI=$8：LTV/CPI=2.2 → 可以跑 ✅
```

| Day7 ROAS | 结论 | 动作 |
|-----------|------|------|
| < 5% | 大概率亏损 | 停投或极小预算观察 |
| 5-10% | 偏悲观 | 小预算（$1000/天），必须等 Day15 确认 |
| 10-15% | 有希望 | 正常投放（$3000/天），观察趋势 |
| > 15% | 优质 | 准备放量 |

**Day7 决策要结合趋势，不能只看绝对值：**

```
LTV/CPI = 1.17 但趋势极好：
  Day1 留存 = 35%（优秀）
  Day7 留存 = 18%（优秀）
  首充率 = 2.5%（优秀）
  Day7 ROAS = 12%，但 Day3 才 4% → 付费在加速
  → 小预算继续，等 Day15

LTV/CPI = 1.17 且趋势一般：
  Day1 留存 = 22%（及格）
  Day7 留存 = 8%（及格）
  首充率 = 0.8%（差）
  付费曲线平缓，没有加速迹象
  → 建议止损，继续投大概率亏更多
```

**第三层：Day 15（趋势验证）**

| 指标 | 止损线 | 及格 | 良好 | 优秀 |
|------|--------|------|------|------|
| **Day15 留存** | < 2% | 3% | 5% | 7% |
| **Day15 ROAS** | < 15% | 25% | 35% | 50% |

```
关键判断：趋势是否向上

✅ 健康趋势：
  Day7 ROAS=12%，Day15 ROAS=28% → 付费加速 ✅

❌ 危险趋势：
  Day7 ROAS=12%，Day15 ROAS=15% → 增长极慢 ⚠️

❌❌ 死亡趋势：
  Day7 ROAS=12%，Day15 ROAS=13% → 用户付费见顶 ❌
```

**第四层：Day 30（最终决策）**

| 指标 | 止损 | 观察 | 可继续 | 放量 |
|------|------|------|--------|------|
| **Day30 留存** | < 2% | 2-3% | 3-5% | > 5% |
| **Day30 ROAS** | < 40% | 40-60% | 60-80% | > 80% |
| **月付费率** | < 3% | 3-5% | 5-8% | > 8% |

```
Day30 决策：

ROAS < 40%：
  → 止损。即使后期回本，周期过长，资金效率低

ROAS 40-60%：
  → 观察。预计 Day90 ROAS 70-90%，接近回本

ROAS 60-80%：
  → 可继续。预计 Day90 ROAS > 100%

ROAS > 80%：
  → 放量。预计 Day180 ROAS > 150%
```

**完整决策流程图：**

```
Day 3：
  ├─ Day1 留存 < 20% → 停投，改产品
  └─ Day1 留存 ≥ 20% → 继续

Day 7：
  ├─ Day7 ROAS < 5% → 停投
  ├─ Day7 ROAS 5-10% → 小预算观察
  ├─ Day7 ROAS 10-15% → 正常投放
  └─ Day7 ROAS > 15% → 准备放量

Day 15：
  ├─ Day15 ROAS < Day7×2 → 趋势不好，准备撤
  ├─ Day15 ROAS 20-35% → 符合预期，等 Day30
  └─ Day15 ROAS > 35% → 超预期，准备放量

Day 30：
  ├─ ROAS < 40% → 止损
  ├─ ROAS 40-60% → 观察，等 Day60
  ├─ ROAS 60-80% → 持续投放
  └─ ROAS > 80% → 放量
```

### SLG 1. 核心差异

| 维度 | 休闲游戏 | SLG 策略 |
|------|---------|---------|
| **买量规模** | 10,000~50,000/天 | 1,000~5,000/天 |
| **CPI** | $1~$3 | $10~$30 |
| **LTV** | $3~$10 | $100~$500 |
| **回本周期** | 7~30 天 | 90~180 天 |
| **现金流需求** | $500K~$1M | $2M~$5M |
| **关键指标** | 留存、IPM | 付费率、ARPPU、联盟加入率 |
| **素材风格** | 轻快、消除、过关 | 战争、建造、复仇、联盟 |
| **优化目标** | Install | Purchase / Value |
| **主要渠道** | Meta/Google/TikTok | Meta/Google（TikTok 效果一般）|

### SLG 2. SLG 特有的关键指标

| 指标 | 公式 | 行业标准 | 说明 |
|------|------|---------|------|
| **首次付费时间** | 用户从安装到第一次付费的天数 | 3~7 天 | 太长说明付费引导有问题 |
| **7日付费率** | Day7 付费人数 / 安装数 | 2~5% | SLG 付费转化慢，要看到 7 天 |
| **30日付费率** | Day30 付费人数 / 安装数 | 4~8% | 核心指标 |
| **ARPPU** | 总收入 / 付费人数 | $200~$1,000 | SLG 的 ARPPU 极高 |
| **联盟加入率** | 加入联盟人数 / 注册数 | 40~70% | 联盟是 SLG 留存的核心 |
| **城堡等级达标率** | 达到指定城堡等级的用户比例 | Day7: 30~50% | 反映新手引导效果 |
| **大R占比** | 付费>$500 的用户 / 付费用户 | 5~15% | 大R贡献 60~80% 收入 |

### SLG 4. SLG 出价策略

SLG 不应以 CPI 为优化目标，应以**付费事件**或**价值**为优化目标。

```
不推荐：
  - 优化 Install → 来很多白嫖用户，不付费

推荐：
  - 优化 Purchase（AEO）→ 平台找更可能付费的用户
  - 优化 Value（VO）→ 平台找高付费用户

**Meta 所有出价类型对比：**

| 出价类型 | 全称 | 优化目标 | 付费方式 | CPI | 适合阶段 | 适合游戏 |
|---------|------|---------|---------|-----|---------|---------|
| **CPM** | Cost Per Mille | 展示量 | 每千次展示付费 | 低 | 品牌曝光 | 大IP首发 |
| **CPC** | Cost Per Click | 点击量 | 每次点击付费 | 中 | 测试素材CTR | 所有 |
| **CPI** | Cost Per Install | 安装量 | 每次安装付费 | 低 | 冲量/冲榜 | 超休闲 |
| **CPA** | Cost Per Action | 特定行为 | 每次行为付费 | 中 | 获取注册/留存 | 中重度 |
| **AEO** | App Event Optimization | 应用事件 | 按展示付费（oCPM）| 中高 | 获取付费用户 | **SLG/RPG** |
| **VO** | Value Optimization | 用户价值 | 按展示付费（oCPM）| 高 | 获取高价值用户 | **SLG/RPG** |
| **A+AC** | Advantage+ App Campaign | 全自动 | 按展示付费 | 系统定 | 自动化投放 | 所有 |

**关键区别：**

```
CPM/CPC/CPI/CPA = 你告诉平台"按这个结果收费"
  → 平台只负责把这个成本控制在目标范围内
  → 不保证用户质量

AEO/VO = 你告诉平台"帮我找这类人"，平台按展示收费
  → 平台用算法学习高价值用户特征
  → 自动优化受众和版位
  → CPI 会波动，但用户质量更高
```

**SLG 推荐路径：**

```
冷启动期（0-30天）：
  → CPI 或 AEO（Purchase）
  → 积累数据和模型

成长期（30-90天）：
  → AEO（Purchase）为主
  → 测试 VO

放量期（90天+）：
  → VO 为主
  → 配合 AEO 做补充
```

Meta AEO 设置：
  1. 确保 Purchase 事件回传正常
  2. Campaign → Ad Set → Optimization & Delivery
  3. Optimization for Ad Delivery: "App Events"
  4. 选择 "Purchase" 事件
  5. 设置 Cost Per Result Goal（目标 CPP）

Meta VO 设置：
  1. 需要每天至少 100 个 Purchase 事件
  2. Optimization for Ad Delivery: "Value"
  3. 设置 Minimum ROAS（如 150%）
  4. 平台会自动优化高价值用户

Google UAC 设置：
  1. Campaign → Bidding
  2. 选择 "In-app actions"
  3. 选择 "Purchase" 事件
  4. 设置 Target CPA（目标 CPP）
```

### SLG 5. SLG 素材策略

**SLG 素材的核心元素：**

| 元素 | 作用 | 示例 |
|------|------|------|
| **冲突/危机** | 前3秒抓住注意力 | "我的城堡被攻打了！" |
| **成长/建造** | 展示成就感 | 从小村庄到宏伟帝国 |
| **联盟/社交** | 展示团队力量 | "加入我们，一起征服世界" |
| ** revenge** | 情感驱动 | "三年后，我回来了..." |
| **英雄/角色** | 增加代入感 | 展示英雄技能和故事 |

**SLG 素材类型效果排序（Meta 渠道）：**

```
1. UE4 渲染 CG（效果最好，成本最高）
   - 战争场面、英雄技能、城堡建造
   - CPI 通常比平均水平低 20~30%

2. 真人剧情（效果好，成本中等）
   -  revenge 剧情、联盟战争、主播推荐
   - 适合 TikTok 和 Meta Reels

3. 玩法录屏（效果中等，成本低）
   - 展示真实游戏画面
   - 需要配史诗音乐

4. 图文卡片（效果一般，成本低）
   - 适合 Google Display
```

**为什么不只投 UE4？**

UE4 素材虽好，但三大局限决定了必须混合投放：

| 局限 | 说明 | 后果 |
|------|------|------|
| **量跑不动** | UE4 CTR 低（0.5-1%），算法探索受限 | 日预算花不出去 |
| **素材疲劳** | 一条 UE4 CG 成本 $5,000-20,000，一个月只能做 2-3 条 | Frequency 快速上升，CPI 飙升 |
| **人群覆盖窄** | 只吸引核心 SLG 玩家，天花板低 | 量太小，无法支撑回收周期 |

**各素材类型的投放定位：**

| 素材类型 | 定位 | 作用 | 预算占比 |
|---------|------|------|---------|
| **UE4/玩法** | 质量担当 | 找核心玩家，保留存和付费 | 30-40% |
| **真人剧情** | 量担当 | 拉新、冲量、测方向 | 40-50% |
| **游戏录屏** | 真实性担当 | 减少"货不对板"差评 | 10-20% |
| **图片** | 补充担当 | 低成本覆盖、再营销 | 5-10% |

**"真人测、UE4 收"策略：**

```
Step 1：真人素材快速测试方向
  - 这周做"复仇剧情"
  - 下周做"联盟战争"
  - 再下周做"从零建帝国"
  - 成本低、周期短、数据反馈快

Step 2：数据验证哪个方向 CTR 高
  - 复仇剧情 CTR 2.5% → 方向好！

Step 3：让 UE4 团队做对应主题的 CG
  - 制作"复仇主题"的高质量 CG
  - 用 UE4 收割核心玩家
```

**实际 Campaign 架构示例：**

```
Campaign A（UE4 核心素材）：
  - 预算：$3,000/天
  - 素材：5 条 UE4 战争/建造 CG
  - 目标：AEO/VO，找高质量用户
  - KPI：D7 ROAS > 15%

Campaign B（真人剧情素材）：
  - 预算：$5,000/天
  - 素材：20 条真人短视频
  - 目标：CPI/AEO，大量拉新
  - KPI：CPI <$15，D1 留存 > 20%

Campaign C（录屏/图片）：
  - 预算：$1,000/天
  - 素材：10 张商店图 + 5 条录屏
  - 目标：补充量、再营销
  - KPI：CPI <$10
```

### SLG 6. SLG 地区策略

| 地区 | CPI | ARPPU | 特点 | 推荐度 |
|------|-----|-------|------|--------|
| **美国** | $15~$30 | $500~$1,500 | 付费能力强，竞争激烈 | ★★★★★ |
| **日本** | $20~$40 | $800~$2,000 | ARPPU 最高，SLG 接受度高 | ★★★★★ |
| **韩国** | $10~$25 | $400~$1,000 | SLG 文化成熟 | ★★★★★ |
| **西欧** | $8~$20 | $300~$800 | 德法英付费好 | ★★★★ |
| **东南亚** | $2~$8 | $50~$200 | 量大利薄 | ★★★ |
| **中东** | $5~$15 | $200~$600 | 新兴地区，增长快 | ★★★★ |
| **拉美** | $3~$10 | $80~$250 | 潜力市场 | ★★★ |

**SLG 地区拓展顺序：**

```
阶段1：美国（验证产品）
  → 美国市场最大，验证产品是否成立

阶段2：日本/韩国（高价值市场）
  → 东亚 SLG 用户质量极高
  → 需要本地化（语言、文化、客服）

阶段3：西欧/中东（扩展市场）
  → 西欧付费稳定
  → 中东增长快，竞争相对小

阶段4：东南亚/拉美（量大利薄）
  → 需要调整商业模式（更多 IAA）
  → 或作为"养服"用户
```

### 2. ARPU、ARPPU 与 LTV 估算

**ARPU 和 ARPPU 的区别：**

| | **ARPU** | **ARPPU** |
|--|---------|-----------|
| **全称** | Average Revenue Per User | Average Revenue Per Paying User |
| **中文** | 每用户平均收入 | 每付费用户平均收入 |
| **公式** | 总收入 ÷ 总用户数 | 总收入 ÷ 付费用户数 |
| **包含用户** | 所有人（付费+免费） | 只算付费的人 |
| **作用** | 看整体变现效率 | 看付费用户有多值钱 |

**两者的关系：**

```
ARPU = ARPPU × 付费率

例子：
  总用户 10,000 人，付费用户 300 人，总收入 $15,000

  ARPU = $15,000 / 10,000 = $1.5（每个人平均贡献 $1.5）
  ARPPU = $15,000 / 300 = $50（付费的人平均花 $50）
  付费率 = 300 / 10,000 = 3%

  验证：$50 × 3% = $1.5
```

**用 Day7 ARPPU 估算 LTV：**

单独用 Day7 ARPPU 不能直接估算 LTV，因为 ARPPU 只告诉你"付费的人花了多少钱"，没告诉你"有多少比例的人付费"。

**正确做法：先把 ARPPU 转成 ARPU，再估算 LTV。**

```
Step 1：算 Day7 ARPU
  Day7 ARPU = Day7 ARPPU × Day7 付费率

  例子（参考 Last War / Whiteout Survival 水平）：
    Day7 ARPPU = $55
    Day7 付费率 = 4.5%
    Day7 ARPU = $55 × 4.5% = $2.48

Step 2：用经验倍数估算 LTV
  LTV ≈ Day7 ARPU × 倍数

  倍数根据 SLG 类型：
    轻度 SLG（合成/放置，如 Top War）：10-12 倍
    中度 SLG（休闲 SLG，如 Last War）：12-15 倍
    重度 SLG（率土-like，如三国志战略版）：15-20 倍

  例子（中度 SLG）：
    LTV ≈ $2.48 × 12 = $29.8

Step 3：判断
  CPI = $15（Meta US，2024-2025 水平）
  LTV/CPI = $29.8 / $15 = 1.99 ≈ 2

  → 项目可以跑，但盈利空间一般
  → 如果 CPI 能降到 $12，LTV/CPI = 2.5，更舒服
```

**快速估算表（参考当前主流 SLG 美国市场）：**

| Day7 ARPPU | Day7 付费率 | Day7 ARPU | 轻度 LTV | 中度 LTV | 重度 LTV |
|-----------|------------|-----------|---------|---------|---------|
| $40 | 3% | $1.20 | $12-14 | $14-18 | $18-24 |
| $55 | 4.5% | $2.48 | $25-30 | $30-37 | $37-50 |
| $70 | 5.5% | $3.85 | $38-46 | $46-58 | $58-77 |
| $100 | 7% | $7.00 | $70-84 | $84-105 | $105-140 |

**判断标准（LTV/CPI）：**

```
LTV/CPI < 1：项目不成立，止损
LTV/CPI = 1-2：勉强回本，无利可图
LTV/CPI = 2-3：可以跑，但盈利薄
LTV/CPI > 3：优质项目，可以放量
```

**常见错误：**

```
❌ 错误1：直接用 Day7 ARPPU 当 LTV
  Day7 ARPPU = $50，以为 LTV = $50
  → 错！ARPU 才是基于所有用户的

❌ 错误2：混淆 ARPU 和 ARPPU
  汇报时说"ARPU 是 $50"，实际说的是 ARPPU
  → ARPU $50 意味着每人平均花 $50，极其罕见

❌ 错误3：倍数用得不对
  轻度 SLG 用了重度 SLG 的 20 倍
  → 高估 LTV，导致错误放量决策
```

### 3. SLG 盈利周期详解

**真正赚钱的周期：180-365 天。** 这是 SLG 的残酷现实。

**单用户回本时间线：**

| 阶段 | 时间 | DayN ROAS | 状态 |
|------|------|-----------|------|
| 大亏期 | Day 1-30 | 20-40% | 花 $15，回 $3-6 |
| 回本准备 | Day 31-90 | 60-90% | 花 $15，回 $9-13.5 |
| 临界点 | Day 91-180 | 100-140% | 花 $15，回 $15-21 ✅ |
| 盈利期 | Day 181-365 | 150-250% | 花 $15，回 $22-37 |
| 稳定期 | Day 365+ | 250-400% | 老用户持续印钞 |

**为什么回本周期这么长？**

SLG 的收入高度依赖**长尾付费**：

```
一个 SLG 用户的付费曲线：

Day 1-7：   付费 $0（还在体验，观望）
Day 8-30：  付费 $5-20（首充礼包，小额试探）
Day 31-90： 付费 $30-100（开始参与联盟战，需要加速）
Day 91-180：付费 $100-500（赛季竞争，必须氪金才能赢）
Day 181+：  付费 $500-2000（成为服务器核心玩家，持续投入）

LTV = $635-2620，但 80% 发生在第 90 天之后
```

**三个容易被混淆的概念：**

| 概念 | 含义 | 时间点 |
|------|------|--------|
| **单用户回本** | 一个用户的 LTV > CPI | Day 180 |
| **现金流打平** | 公司月收入 > 月支出 | Month 12-18 |
| **真正盈利** | 累计利润覆盖所有历史亏损 | Month 18-24 |

**"18 个月"是什么？**

18 个月不是单用户回本的时间（单用户 180 天就能回本），而是**公司层面从投入到累计盈利的时间**。

```
为什么公司层面需要 18 个月？

因为你每天都在买新用户：
  Month 1 买的用户 → Day180 回本（Month 6）
  Month 2 买的用户 → Day180 回本（Month 7）
  Month 3 买的用户 → Day180 回本（Month 8）
  ...
  Month 12 买的用户 → Day180 回本（Month 17）

在 Month 18 之前：
  - 每天都有新用户在"亏损"（还没回本）
  - 只有早期买的老用户在"赚钱"
  - 老用户赚的钱 < 新用户亏的钱
  - 公司整体还在烧钱

Month 18 之后：
  - 老用户积累足够多
  - 老用户贡献的收入 > 新用户的买量成本
  - 公司整体开始真正盈利
```

**实际案例的现金流测算：**

```
一款 SLG，月买量预算 $500,000，运营成本 $200,000/月

Month 1-6：
  买量花费：$3,000,000
  游戏收入：$800,000
  运营成本：$1,200,000
  累计净亏损：-$3,400,000 💸

Month 7-12：
  买量花费：$3,000,000
  游戏收入：$3,500,000（老用户开始大量付费）
  运营成本：$1,200,000
  累计净亏损：-$4,100,000（还在亏，但增速放缓）

Month 13-18：
  买量花费：$3,000,000
  游戏收入：$6,500,000
  运营成本：$1,200,000
  累计净利润：+$200,000 ✅ 终于扭亏为盈

Month 19+：
  累计老用户持续付费
  进入"印钞机"模式
```

**SLG 的"死亡谷"：**

```
Month 3-6 是最危险的时期：

  已花了大量钱买量，但收入还没起来
  投资人开始质疑
  团队开始焦虑
  有人想砍预算止损

但如果砍预算：
  → 老用户流失后没有新用户补充
  → 服务器变鬼服，留存用户也流失
  → 前功尽弃

SLG 买量的铁律：
  一旦开始就不能停，停了就是死。
```

**盈利时间的数学推导：**

假设每月固定买量，公司整体累计盈亏平衡的时间公式：

```
        T × R
N = ───────────
    R - 1 - c

参数：
  N = 公司整体累计回正时间（月）
  T = 单用户回本周期（月）
  R = LTV / CPI（收入倍数）
  c = 运营成本系数 = 月运营成本 / 月买量成本

前提：R > 1 + c，否则永远不回本
```

> **旧版公式错误说明：** 此前文档写为 `N = T(R-1)/(R-2-c)`，临界条件是 `R > 2+c`。**这是错的。** 错误根源在于把"已回本用户净利润"与"累计总支出"直接对比时，右边重复扣除了已回本用户的买量成本。修正后的临界条件是 `R > 1+c`，回正时间也更短。

**公式推导过程（修正版）：**

```
假设：
  - 每月买量成本 = A
  - 单用户回本周期 = T 个月
  - 每个用户群最终 LTV = R × A
  - 运营成本 = c × A

简化模型（保守估计）：
  - 未回本用户收入 ≈ 0（实际不为0，此假设高估回正时间）
  - 已回本用户收入 = R × A（一次性到账，实际细水长流）

Month N 时：
  已回本用户群数量 = N - T
  累计总收入 = (N-T) × R × A
  累计总支出 = N × A × (1+c)

盈亏平衡（累计净利润 = 0）：
  (N-T) × R × A = N × A × (1+c)

  两边消去 A：
  (N-T) × R = N(1+c)

  展开：
  NR - TR = N + Nc
  N(R - 1 - c) = TR

  最终结果：
        T × R
  N = ─────────
      R-1-c
```

**为什么 LTV/CPI 必须大于 1 + c？**

```
从公式 N = TR/(R-1-c) 可以看出：

当 R = 1 + c 时：
  分母 R-1-c = 0，N = ∞
  → 公司永远累计回不了本

  R < 1+c：永远亏损（累计现金流一直往下）
  R = 1+c：当月勉强打平，但前期亏损填不满
  R > 1+c：稳态利润 > 新用户成本+运营成本，有多余填历史坑

【情况1：忽略运营成本 c=0】
  R > 1 就能回本（显然，LTV > CPI 即可）

【情况2：考虑运营成本 c=0.4（更现实）】
  盈亏平衡线：R > 1.4

  例：R = 2.4, c = 0.4, T = 3个月
    N = 3 × 2.4 / (2.4 - 1 - 0.4) = 7.2 / 1.0 = 7.2 个月
    → 公司约 7-8 个月累计回正

  例：R = 2.4, c = 0.4, T = 4个月
    N = 4 × 2.4 / 1.0 = 9.6 个月

  例：R = 3.0, c = 0.4, T = 4个月
    N = 4 × 3.0 / 1.6 = 7.5 个月

⚠️ 重要提醒：
  上述是"简化模型"，假设未回本用户收入=0、已回本用户收入一次性到账。
  实际情况中：
    - 未回本用户也有部分收入（加快回正）
    - 已回本用户收入是细水长流（延缓回正）
  两个因素方向相反，简化模型的误差通常在 ±20-30%。
  更精确的模型需要考虑具体收入曲线，见下文。
```

**更精确的理解：回本后用户仍在贡献**

你提到的关键问题——"第4、5个月还有利润去填新用户的成本"——完全正确。

简化模型假设已回本用户的利润"一次性到账"，实际上利润是持续产生的。更现实的现金流如下（中度 SLG 示例）：

```
假设：T=3个月回本，R=2.4，c=0.4，每批用户买量成本 A=1

用户群收入曲线（累计收入倍数 r(t)）：
  Month 1: 0.10  |  Month 4: 1.20（回本后继续涨）
  Month 2: 0.30  |  Month 5: 1.40
  Month 3: 1.00  |  Month 6: 1.60
                 |  Month 7+: 每季度 +0.2，直到 2.4

Month   当月总收入   当月支出(1.4)   当月净现金流   累计净利润
─────────────────────────────────────────────────────────────
M1      0.1         1.4            -1.3          -1.3
M2      0.3         1.4            -1.1          -2.4
M3      1.0         1.4            -0.4          -2.8
M4      1.2         1.4            -0.2          -3.0   ← 垫资峰值
M5      1.4         1.4             0.0          -3.0
M6      1.6         1.4            +0.2          -2.8
M7      1.8         1.4            +0.4          -2.4
M8      2.0         1.4            +0.6          -1.8
M9      2.2         1.4            +0.8          -1.0
M10     2.4         1.4            +1.0           0.0   ← 累计回正！
M11+    2.4         1.4            +1.0          +1.0/月（稳态利润）
```

关键发现：
1. R=2.4, c=0.4 **不是**永远不回本，约 10 个月累计回正
2. 垫资峰值在 M4（约 3 个月支出），不是简化模型算的 7 个月
3. 稳态当月利润 = R×A - (1+c)×A = 2.4 - 1.4 = 1.0/月 ✓

这说明：只要 R > 1+c，长期一定能回本。但"多久回本"取决于收入曲线形状。
```

**查表速算（简化模型，c=0）：**

公式：`N = TR/(R-1)`

| LTV/CPI (R) | N/T | T=6月 | T=3月 |
|-------------|-----|-------|-------|
| **1.5** | 1.5/0.5=3 | **18 个月** | **9 个月** |
| **2.0** | 2/1=2 | **12 个月** | **6 个月** |
| **2.5** | 2.5/1.5≈1.67 | **10 个月** | **5 个月** |
| **3.0** | 3/2=1.5 | **9 个月** | **4.5 个月** |
| **4.0** | 4/3≈1.33 | **8 个月** | **4 个月** |

**考虑运营成本（c=0.4）的查表：**

公式：`N = TR/(R-1.4)`

| LTV/CPI (R) | N/T | T=6月 | T=3月 |
|-------------|-----|-------|-------|
| **1.5** | 1.5/0.1=15 | **90 个月** | **45 个月** |
| **2.0** | 2/0.6≈3.33 | **20 个月** | **10 个月** |
| **2.5** | 2.5/1.1≈2.27 | **14 个月** | **7 个月** |
| **3.0** | 3/1.6≈1.88 | **11 个月** | **5.5 个月** |
| **4.0** | 4/2.6≈1.54 | **9 个月** | **4.5 个月** |

> 对比旧版（错误公式 `N=T(R-1)/(R-2-c)`）：旧版算出的时间偏长，临界线偏高（`R>2+c`），会给决策者造成"项目不成立"的误判。修正后的模型显示：**只要 `R > 1+c`，项目长期可盈利**，关键是回正时间和垫资峰值能否承受。

### 4. SLG 现金流管理要点

SLG 单用户回本周期长（90~180 天），公司层面累计回正约 7~18 个月（取决于 R 和 c），现金流管理比休闲游戏更重要。

```
1. 分阶段投放
   - 测试期：$500~$2,000/天，验证 CPI 和早期留存
   - 观察期：$2,000~$5,000/天，观察 Day30 ROAS
   - 放量期：$5,000+/天，确认盈利后逐步加大

2. 严格控制亏损 Campaign
   - Day30 ROAS < 50%：立即暂停
   - Day30 ROAS 50~80%：观察，优化素材和定向
   - Day30 ROAS > 80%：可以持续投放
   - Day90 ROAS > 100%：回本，继续
   - Day180 ROAS > 150%：盈利，加大投放

3. 准备充足现金流
   - 假设日买量 $20,000（中度 SLG 放量期），回本周期 120 天
   - 需要垫付：$20,000 × 120 = $2,400,000
   - 加上运营成本，建议准备 $3,500,000+

4. 建立信用额度
   - 与代理建立月结/账期合作
   - 与平台申请信用额度（Meta/Google 对大客户有账期）
```

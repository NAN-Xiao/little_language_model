# LTV 估算实操手册

> 只有早期数据时，如何一步步算出 LT 和 LTV，并做出买量决策。

---

## 一、名词说明

| 名词     | 含义                | 单位      |
| -------- | ------------------- | --------- |
| **LT**   | 用户平均生命周期    | 天数      |
| **LTV**  | 用户终身价值        | 金额（$） |
| **ARPU** | 每用户平均收入      | 金额（$） |
| **ROAS** | 广告支出回报率      | 百分比    |
| **CPI**  | 单次安装成本        | 金额（$） |
| **R**    | LTV / CPI，收入倍数 | 无量纲    |

**核心关系：**
```
LTV = ARPU × LT
```

---

## 二、LT 估算方法

### 2.1 什么是 LT

```
LT（Lifetime）= 用户从安装到流失的平均时间

LTV = ARPU × LT

LT 是"时间"（天数）
LTV 是"金额"（美元）
```

**核心思路：**

```
只有前7天或30天的留存数据，怎么预测 LT？

解法：用幂函数拟合留存曲线，外推预测长期留存，再累加求 LT
```

**步骤：**

```
Step 1：用已知留存数据拟合留存曲线
  R(t) = a × t^(-b)

Step 2：外推预测任意天数留存
  代入 t = 30, 60, 90, 180 得到预测留存率

Step 3：按月累加求 LT
  LT ≈ 30 × (1 + M1 + M2 + M3 + ...)

Step 4：算 LTV
  LTV = 日均_ARPU × LT
```

---

### 2.2 方法1：两点法（D1 + D7）

**适用场景：** 最常见。只要有 D1 和 D7 留存，就能算出 b。

**公式推导：**

```
留存曲线：R(t) = a × t^(-b)

已知：
  R(1) = a × 1^(-b) = a = D1
  R(7) = a × 7^(-b) = D7

两式相除：
  R7 / R1 = 7^(-b)
  ln(R7/R1) = -b × ln(7)
  b = -ln(R7/R1) / ln(7)
```

**完整示例：**

```
已知：D1 = 35%，D7 = 12%

Step 1：求 b
  b = -ln(0.12 / 0.35) / ln(7)
    = -ln(0.343) / 1.946
    = 1.070 / 1.946
    = 0.55

Step 2：得到留存曲线
  a = D1 = 0.35
  R(t) = 0.35 × t^(-0.55)

Step 3：预测长期留存
  D30:  0.35 × 30^(-0.55) = 0.35 × 0.154 = 5.4%
  D60:  0.35 × 60^(-0.55) = 0.35 × 0.105 = 3.7%
  D90:  0.35 × 90^(-0.55) = 0.35 × 0.084 = 2.9%
  D180: 0.35 × 180^(-0.55) = 0.35 × 0.057 = 2.0%

Step 4：按月累加求 LT
  M1 = D1 = 35%        （第1个月末留存）
  M2 = (D7~D30 平均) ≈ 17%   （第2个月）
  M3 = (D30~D60 平均) ≈ 10%  （第3个月）
  M4 = (D60~D90 平均) ≈ 7%   （第4个月）
  M5 = 5.3%            （外推）
  M6 = 4.2%            （外推）
  M7~12 ≈ 3~2%         （外推）

  LT ≈ 30 × (1 + 0.35 + 0.17 + 0.10 + 0.07 + 0.053 + 0.042 + 0.03×6)
     ≈ 30 × 2.0
     ≈ 60 天

→ 中度 SLG，LT 约 60 天
```

**计算每一步的 t^(-0.55)：**

```
30^(-0.55) = e^(-0.55 × ln(30)) = e^(-0.55 × 3.401) = e^(-1.871) = 0.154
60^(-0.55) = e^(-0.55 × ln(60)) = e^(-0.55 × 4.094) = e^(-2.252) = 0.105
90^(-0.55) = e^(-0.55 × ln(90)) = e^(-0.55 × 4.500) = e^(-2.475) = 0.084
```
轻度游戏：截断到 D90 或 M3
中度 SLG：截断到 M12（360 天）
重度 SLG：截断到 M18 或 D365

### 2.3 方法2：三点法（D1 + D7 + D30）——最小二乘拟合

**适用场景：** 有 D1、D7、D30 三个数据点，拟合更精确。

**最小二乘法完整推导：**

```
Step 1：取对数，把幂函数变直线

  幂函数：R(t) = a × t^(-b)
  两边取 ln：ln(R) = ln(a) - b × ln(t)

  设：y = ln(R)，x = ln(t)，A = ln(a)
  得到：y = A - b × x  （一条直线）

Step 2：把数据点变成 (x, y)

  已知：D1=42%, D7=17%, D30=8%

  t=1:   x=ln(1)=0.000,   y=ln(0.42)=-0.868
  t=7:   x=ln(7)=1.946,   y=ln(0.17)=-1.772
  t=30:  x=ln(30)=3.401,  y=ln(0.08)=-2.526

Step 3：最小二乘法求 A 和 b

  公式：
    斜率 = -b = Σ((x - x̄)(y - ȳ)) / Σ((x - x̄)²)
    截距 = A = ȳ - (-b) × x̄

  算平均值：
    x̄ = (0 + 1.946 + 3.401) / 3 = 1.782
    ȳ = (-0.868 - 1.772 - 2.526) / 3 = -1.722

  算斜率 -b：
    分子 = (0-1.782)(-0.868+1.722) + (1.946-1.782)(-1.772+1.722)
         + (3.401-1.782)(-2.526+1.722)
         = -1.521 + (-0.008) + (-1.302)
         = -2.831

    分母 = (0-1.782)² + (1.946-1.782)² + (3.401-1.782)²
         = 3.176 + 0.027 + 2.621
         = 5.824

    斜率 = -2.831 / 5.824 = -0.486
    所以 b = 0.486 ≈ 0.49

  算截距 A：
    A = -1.722 - (-0.486) × 1.782
      = -1.722 + 0.866
      = -0.856

**斜率和截距代表什么？**

```
取对数后，幂函数变成一条直线：y = A - b × x

  x = ln(t)    ← 天数的对数（横轴）
  y = ln(R)    ← 留存率的对数（纵轴）

截距 A = ln(a)：
  → 直线和 y 轴的交点
  → A 越大，a 越大，D1 留存越高
  → 例：A = -0.856 → a = e^(-0.856) = 0.42 → D1 = 42%

斜率 = -b：
  → 直线的倾斜程度
  → 斜率越负（越陡），b 越大，留存掉得越快
  → 例：斜率 = -0.49 → b = 0.49（重度 SLG 级别，衰减慢）

对比：
  b = 0.9（超休闲）：斜率很陡，留存掉得极快
  b = 0.3（社交）：斜率平缓，留存掉得极慢
```

**图示：**

```
ln(R)  ↑
  0    ┤● D1=42%                    斜率 = -b = -0.49
       │  \                         （直线向下倾斜）
 -1    │    ● D7=17%
       │      \
 -2    │        ● D30=8%
       │
 -3    └────────┬────────┬────────→ ln(t)
               0        2        3.4
              ln(1)   ln(7)   ln(30)

直线方程：y = -0.856 - 0.49 × x
截距 A = -0.856（直线和 y 轴交点）
```

Step 4：从 A 反推 a

  a = e^A = e^(-0.856) = 0.425 ≈ 0.43

→ 得到留存曲线：R(t) = 0.43 × t^(-0.49)
```

**验证：**

```
t=1:   0.43 × 1^(-0.49)     = 0.43 = 43%   ≈ 42% ✓
t=7:   0.43 × 7^(-0.49)     = 0.43 / 2.62 = 0.164 ≈ 16%  ≈ 17% ✓
t=30:  0.43 × 30^(-0.49)    = 0.43 / 5.42 = 0.079 ≈ 7.9% ≈ 8%  ✓
```

**预测长期留存：**

```
D60:  0.43 × 60^(-0.49) = 0.43 / 7.55 = 5.7%
D90:  0.43 × 90^(-0.49) = 0.43 / 9.22 = 4.7%
D180: 0.43 × 180^(-0.49) = 0.43 / 13.0 = 3.3%
D365: 0.43 × 365^(-0.49) = 0.43 / 18.5 = 2.3%
```

**按月累加求 LT：**

```
M1 = 42%, M2 = 25%, M3 = 15%, M4 = 11%
M5 = 8%, M6 = 6.5%, M7~12 ≈ 5-3%
M13~24 ≈ 3-2%

LT ≈ 30 × (1 + 0.42 + 0.25 + 0.15 + 0.11 + 0.08 + 0.065 + 0.05×6 + 0.03×12)
   ≈ 30 × 2.8
   ≈ 84 天

→ Whiteout 级别，LT 约 84 天
```

### 2.4 方法3：四点法（D1 + D7 + D15 + D30）——最小二乘拟合

**适用场景：** 有 Day15 数据时，四点拟合比三点更稳，误差 ±10%。

**完整示例：**

```
已知：D1 = 38%，D7 = 15%，D15 = 9%，D30 = 5.5%

Step 1：取对数，变成 (x, y)

  t=1:   x=ln(1)=0.000,   y=ln(0.38)=-0.968
  t=7:   x=ln(7)=1.946,   y=ln(0.15)=-1.897
  t=15:  x=ln(15)=2.708,  y=ln(0.09)=-2.408
  t=30:  x=ln(30)=3.401,  y=ln(0.055)=-2.900

Step 2：最小二乘（n = 4）

  x̄ = (0 + 1.946 + 2.708 + 3.401) / 4 = 2.014 
  ȳ = (-0.968 - 1.897 - 2.408 - 2.900) / 4 = -2.043

  分子 = (0-2.014)(-0.968+2.043)
       + (1.946-2.014)(-1.897+2.043)
       + (2.708-2.014)(-2.408+2.043)
       + (3.401-2.014)(-2.900+2.043)
       = -2.165 - 0.010 - 0.253 - 1.189
       = -3.617

  分母 = (0-2.014)² + (1.946-2.014)²
       + (2.708-2.014)² + (3.401-2.014)²
       = 4.056 + 0.005 + 0.482 + 1.924
       = 6.467

  斜率 = -3.617 / 6.467 = -0.559
  → b = 0.559 ≈ 0.56

  A = -2.043 - (-0.559) × 2.014 = -0.917
  a = e^(-0.917) = 0.400

Step 3：得到留存曲线
  R(t) = 0.40 × t^(-0.56)

Step 4：验证拟合
  t=1:   0.40 × 1      = 40%  ≈ 38%  ✓
  t=7:   0.40 × 7⁻⁰·⁵⁶ = 13.4% ≈ 15%  （接近）
  t=15:  0.40 × 15⁻⁰·⁵⁶ = 8.8%  ≈ 9%   ✓
  t=30:  0.40 × 30⁻⁰·⁵⁶ = 6.0%  ≈ 5.5% （接近）

Step 5：预测长期留存
  D60:  0.40 × 60⁻⁰·⁵⁶ = 4.0%
  D90:  0.40 × 90⁻⁰·⁵⁶ = 3.2%
  D180: 0.40 × 180⁻⁰·⁵⁶ = 2.2%
  D365: 0.40 × 365⁻⁰·⁵⁶ = 1.5%

Step 6：按月累加求 LT（硬截断 M12）
  M1 = 38%, M2 ≈ 10%, M3 ≈ 4.8%, M4 ≈ 3.6%
  M5 ≈ 2.8%, M6 ≈ 2.2%, M7~12 ≈ 2~1%

  LT ≈ 30 × (1 + 0.38 + 0.10 + 0.048 + 0.036 + 0.028
              + 0.022 + 0.018 + 0.015 + 0.013 + 0.012
              + 0.011 + 0.010)
     ≈ 30 × 1.69
     ≈ 51 天

→ 中度 SLG，LT 约 51 天
```

**和两点法、三点法对比：**

| 方法 | 数据点 | 算出 b | 误差 | D15 验证 |
|------|--------|--------|------|---------|
| 两点法 | D1, D7 | 0.55 | ±25% | — |
| 三点法 | D1, D7, D30 | 0.52 | ±15% | 预测 D15=11%，实际 9%（偏高） |
| **四点法** | **D1, D7, D15, D30** | **0.56** | **±10%** | **预测 D15=8.8%，实际 9% ✓** |

四点法的关键价值：D15 检验了幂函数在 D7→D30 之间是否偏离。如果三点法预测的 D15 和实际差距大，说明留存曲线不是纯幂函数，需要警惕。

### 2.5 方法4：查 b 值表（只有 D7 留存时）

**适用场景：** 你只有 D7 留存，没有 D1。根据游戏类型查表估算。

```
已知：D7 = 12%，游戏类型 = 中度 SLG

查表得：中度 SLG 的 b ≈ 0.5

反推 D1：
  R7 = R1 × 7^(-b)
  0.12 = R1 × 7^(-0.5)
  R1 = 0.12 / 0.378 = 0.317 ≈ 32%

然后用方法1（两点法）继续算 LT...
```

**各游戏类型的典型 b 值：**

| 游戏类型  | b 值 | 典型 D1 | 典型 D7 | 预测 LT    | 说明     |
| --------- | ---- | ------- | ------- | ---------- | -------- |
| 超休闲    | 0.9  | 35%     | 8%      | 3-7 天     | 衰减极快 |
| 休闲      | 0.6  | 35%     | 12%     | 15-25 天   | 衰减快   |
| 中度 SLG  | 0.5  | 35%     | 15%     | 30-60 天   | 衰减中等 |
| 重度 SLG  | 0.45 | 40%     | 18%     | 60-120 天  | 衰减慢   |
| 社交/棋牌 | 0.3  | 45%     | 25%     | 150-300 天 | 衰减极慢 |

### 2.6 按月累加求 LT

**公式：**

```
LT ≈ 30 × (1 + M1 + M2 + M3 + ...)

M1 = D1        （第1个月末留存）
M2 = D30       （第2个月末留存）
M3 = D60       （第3个月末留存）
...
```

**注意：**
```
1 = 第0个月（安装当月）
这种累加是行业简化做法，假设安装当月全部存活
实际精确值会略低，但在买量决策精度内可接受
```

**为什么只加到 M12？幂函数的截断问题**

```
⚠️ 幂函数的数学缺陷：R(t) = a × t^(-b) 在 b < 1 时，
   积分 ∫R(t)dt 从 1 到 ∞ 是发散的——理论上可以无限加下去。

   M12 留存 ≈ 2%  → 贡献 0.02
   M24 留存 ≈ 1%  → 贡献 0.01
   M36 留存 ≈ 0.7% → 贡献 0.007
   M48 留存 ≈ 0.5% → 贡献 0.005
   ...永远减不到 0

   按幂函数无脑累加，LT 会越来越大，预估会偏乐观。
```

**现实中的处理：幂函数 + 硬截断**

```
幂函数只在前 60-180 天拟合得好，之后实际的留存曲线会偏离：

1. 游戏内容消耗完 → 留存会加速下降
2. 核心玩家形成稳定社群 → 留存趋近于 1-3% 平台期

无论哪种情况，都不会按幂函数无限拖下去。

实操截断规则：
  ✓ 累加到 M12（约 360 天），或
  ✓ 累加到留存率 < 1% 的月份，或
  ✓ 直接封顶 LT = 365 天（最常见）

尾部修正：
  从 M12 到 M24 的额外贡献 ≈ 0.3-0.5 个月
  超过 M24 的贡献 < 0.2 个月，可忽略
```

**修正后的 LT 计算（硬截断到 M12）：**

```
LT ≈ 30 × (1 + 0.35 + 0.17 + 0.10 + 0.07 + 0.053 + 0.042 + 0.03 + 0.025 + 0.022 + 0.02 + 0.018 + 0.015)
   ≈ 30 × 1.87
   ≈ 56 天

对比无限累加的"虚假值"：
  无限累加 → ≈ 72 天（偏高 28%，不可用）
  硬截断 M12 → ≈ 56 天（更保守、更现实）
```

→ **估算 LT 时，必须声明截断点**。不声明截断点的 LT 没有意义。

## 三、LTV 估算方法

### 3.1 什么是 LTV

```
LTV（Lifetime Value）= 一个用户从安装到流失前，总共贡献的收入

简化理解：LTV = ARPU × LT

为什么需要估算 LTV？
- Day7 的数据是最先拿到的，但决策需要看长期
- 等 Day365 的数据出来再决策，黄花菜都凉了
- 所以必须用早期数据预测 LTV，指导买量决策
```

---

### 3.2 方法1：倍数法（最常用）

**核心思路：**
```
LTV ≈ DayN_ARPU × 经验倍数

不同天数的数据，对应不同的倍数
```

**倍数表：**

| 可用数据  | 超休闲 | 休闲   | 中度SLG  | 重度SLG  | 误差 |
| --------- | ------ | ------ | -------- | -------- | ---- |
| **Day3**  | 10-12x | 8-10x  | 8-10x    | 6-8x     | ±40% |
| **Day7**  | 5-6x   | 4-5x   | 8-10x    | 6-8x     | ±25% |
| **Day15** | 3-4x   | 3-4x   | 3.5-4.5x | 3-4x     | ±15% |
| **Day30** | 2-2.5x | 2-2.5x | 2.5-3.5x | 2.5-3.5x | ±10% |
| **Day60** | 1.5-2x | 1.5-2x | 1.8-2.2x | 1.8-2.2x | ±5%  |

**示例1：第7天估算（中度 SLG）**

```
Day7 ARPU = $1.35

LTV ≈ $1.35 × 8 = $10.8
  或 $1.35 × 10 = $13.5

→ 取中值：LTV ≈ $12（误差 ±$2）
```

**示例2：第30天估算（中度 SLG）**

```
Day30 ARPU = $3.60

LTV ≈ $3.60 × 3 = $10.8
  或 $3.60 × 3.5 = $12.6

→ 取中值：LTV ≈ $11.7（误差 ±$1）
```

**为什么 Day30 倍数比 Day7 小？**

```
不是倍数变小了，而是分母变大了。

Day7：ARPU = $1.35（用户只付了$1.35）
  LTV ≈ $1.35 × 8 = $10.8

Day30：ARPU = $3.60（用户已经付了$3.60）
  LTV ≈ $3.60 × 3 = $10.8

结果一样，但 Day30 的 ARPU 已经包含了更多收入，
所以需要的倍数更小。
```

---

### 3.3 方法2：LT + ARPU 法

**适用场景：** 你已经用留存曲线算出了 LT，有 ARPU 数据，直接相乘。

**公式：**
```
LTV = 日均_ARPU × LT

日均_ARPU = DayN_ARPU / N
```

**示例：**
```
已知：
  Day7 ARPU = $1.35（每天 $0.19）
  通过留存曲线预测 LT = 60 天

LTV = $0.19/天 × 60 天 = $11.4

→ 和倍数法结果对比：
  Day7 ARPU $1.35 × 8 倍 = $10.8
  → 两种方法接近 ✅
```

---

### 3.4 方法3：积分法（有留存 + 付费数据时）

**公式：**
```
LTV = ARPPU × 付费率 × Σ(各月留存率)

或
LTV = 日均_ARPU × LT
```

**示例：**
```
已知：
  ARPPU = $100
  付费率 = 5%
  月留存：M1=35%, M2=17%, M3=10%, M4=7%, M5=5%, M6=4%

LT = 30 × (1 + 0.35 + 0.17 + 0.10 + 0.07 + 0.05 + 0.04)
   = 30 × 1.78
   = 53 天

LTV = $100 × 5% × (0.35+0.17+0.10+0.07+0.05+0.04)
    = $5 × 0.78
    = $3.90（半年）

→ 如果 CPI > $3.90，半年不回本
```

---

### 3.5 方法4：ROAS 反推法

**适用场景：** 你有某个时间点的 ROAS，想反推最终 LTV，判断项目能否盈利。

**核心规律：**
```
Day7  / Day30  ≈ 0.20-0.25  →  Day30 ≈ Day7 × (4-5)
Day30 / Day180 ≈ 0.30-0.40  →  Day180 ≈ Day30 × (2.5-3.5)
```

---

**方法一：倍数法（30 秒出结论）**

**Step 1：Day7 → 预测 Day30**

```
已知：Day7 ROAS = 12%

Day30 ROAS ≈ Day7 × 4 = 12% × 4 = 48%
（保守用 ×4，乐观用 ×5）

→ 预测 Day30 ROAS = 48-60%
```

**Step 2：Day30 → 预测 Day180**

```
已知：Day30 ROAS = 48%

Day180 ROAS ≈ Day30 × 2.5 = 48% × 2.5 = 120%
（保守用 ×2.5，乐观用 ×3.5）

→ 预测 Day180 ROAS = 120-168%
```

**Step 3：算 LTV 和判断**

```
CPI = $15

保守：LTV = $15 × 1.20 = $18  →  LTV/CPI = 1.2  → 微利，观察
乐观：LTV = $15 × 1.68 = $25.2 →  LTV/CPI = 1.68 → 可跑
```

**ROAS 预测速查表：**

| 已知 Day30 ROAS | 预测 Day90 | 预测 Day180 | 预测 Day365 |
| --------------- | ---------- | ----------- | ----------- |
| **30%**         | 55-65%     | 75-105%     | 105-150%    |
| **40%**         | 70-85%     | 100-140%    | 140-200%    |
| **50%**         | 90-110%    | 125-175%    | 175-250%    |
| **60%**         | 110-130%   | 150-210%    | 210-300%    |
| **80%**         | 145-175%   | 200-280%    | 280-400%    |

---

**方法二：对数回归法（更准）**

**假设 ROAS 增长符合对数曲线：**
```
ROAS(t) = a × ln(t) + b
```

**Step 1：用两个已知点求 a, b**

```
已知：Day7 ROAS = 12%，Day30 ROAS = 45%

t=7:   12 = a × ln(7) + b   →  12 = 1.946a + b
t=30:  45 = a × ln(30) + b  →  45 = 3.401a + b

两式相减：
33 = 1.455a
a = 22.68

代入第一式：
b = 12 - 1.946 × 22.68 = -32.13

→ ROAS(t) = 22.68 × ln(t) - 32.13
```

**Step 2：预测 Day180**

```
ROAS(180) = 22.68 × ln(180) - 32.13
          = 22.68 × 5.193 - 32.13
          = 117.8 - 32.13
          = 85.7%
```

**Step 3：判断**

```
LTV = $15 × 0.857 = $12.9
LTV/CPI = 12.9/15 = 0.86 → 亏损，优化或止损
```

---

**两种方法对比：**

| 场景 | 倍数法 | 对数回归法 | 差异原因 |
|------|--------|-----------|---------|
| Day7=12%, Day30=45% | Day180=112-157% | Day180=86% | 倍数法假设线性增长，对数法更保守 |
| Day7=15%, Day30=60% | Day180=150-210% | Day180=120% | 高基数下对数增长放缓 |

**实操建议：**
  ✓ 快速决策用倍数法
  ✓ 涉及大额放量用对数法
  ✓ 两种都算，取中间值

**公式：**
```
LTV = CPI × 预测最终 ROAS
```

**完整示例：**
```
已知：Day30 ROAS = 60%，CPI = $15

倍数法：
  预测 Day180 ROAS = 60% × 3 = 180%
  LTV = $15 × 1.8 = $27
  → LTV/CPI = 1.8 → 勉强回本

对数回归法（需 Day7 数据辅助）：
  假设 Day7 ROAS = 15%，Day30 ROAS = 60%
  a = (60-15)/(ln30-ln7) = 45/1.455 = 30.9
  b = 15 - 30.9×1.946 = -45.1
  ROAS(180) = 30.9×5.193 - 45.1 = 115.3%
  LTV = $15 × 1.15 = $17.25
  → LTV/CPI = 1.15 → 微利
```

## 五、决策标准

### 5.1 基于 LTV/CPI 的判断

**前提：LTV 是截断到 M12 的保守估计**

```
截断后的 LTV 漏掉了 M12 之后的尾部收入，所以是"下限"。
真实 LTV ≥ 保守 LTV。

这意味着：
  ✓ 保守 LTV/CPI > 2 → 真实值更高，放心跑
  ⚠ 保守 LTV/CPI = 1-2 → 加上尾部可能刚好回本，微利
  ✗ 保守 LTV/CPI < 1 → 连下限都覆盖不了 CPI，肯定亏损
```

| 保守 LTV/CPI | 真实 LTV/CPI（估计） | 结论        | 动作               |
| ------------ | ------------------- | ----------- | ------------------ |
| **< 1**      | < 1.3               | 亏损        | 立即止损           |
| **1-2**      | 1.3-2.5             | 微利/不盈利 | 优化或止损         |
| **2-3**      | 2.5-4               | 可跑        | 控制节奏，持续优化 |
| **> 3**      | > 4                 | 优质        | 放量               |

### 5.2 只看 R 不够，还要看回本速度 T

**核心问题：同样的 R=3，不同的回本速度，命运完全不同。**

```
游戏 A：R=3，T=3 个月，Day30 ROAS = 80%
  → 早期现金流好，垫资 6 个月，活下来 ✓

游戏 B：R=3，T=12 个月，Day30 ROAS = 25%
  → 早期现金流差，垫资 18 个月，可能撑不到回本 ✗
```

**→ R 决定"最终能不能赚"，T 和早期 ROAS 决定"能不能活到赚钱那天"。**

**付费曲线形状的影响：**

| 维度 | 游戏 A（前高后低） | 游戏 B（前低后高） | 结论 |
|------|-------------------|-------------------|------|
| R（最终） | 3.0 | 3.0 | 一样 |
| T（回本） | 3 个月 | 12 个月 | 差 4 倍 |
| Day30 ROAS | 80% | 25% | 差距巨大 |
| 垫资峰值 | 约 $200 万 | 约 $800 万 | 差 4 倍 |
| 存活概率 | 高 | 低 | 关键差异 |

**修正后的三维判断标准：**

| 条件 | 最低要求 | 说明 |
|------|---------|------|
| **R > 2.4** | 必须满足 | 最终能盈利（c=0.4 时） |
| **Day30 ROAS > 40%** | 必须满足 | 早期现金流健康 |
| **T < 6 个月** | 建议满足 | 单用户回本别太慢 |

```
三个条件同时满足 → 项目能活，可以跑
只满足 R，不满足 Day30 ROAS 和 T → 可能死在现金流上
```

### 5.3 基于不同天数数据的决策

```
只有 Day3 数据：
  → 误差 ±40%，只做方向性判断
  → D3 留存 < 8% 或 Day3 ROAS < 2%：大概率有问题

只有 Day7 数据：
  → 误差 ±25%，决定是否继续投
  → Day7 ROAS < 5%：大概率亏损
  → Day7 ROAS 5-10%：偏悲观，小预算观察
  → Day7 ROAS 10-15%：有希望，正常投放
  → Day7 ROAS > 15%：优质，准备放量

只有 Day15 数据：
  → 误差 ±15%，决定是否放量
  → Day15 ROAS < 15%：趋势不好，准备撤
  → Day15 ROAS 20-35%：符合预期，等 Day30
  → Day15 ROAS > 35%：超预期，准备放量

有 Day30 数据：
  → 误差 ±10%，精确测算
  → Day30 ROAS < 40%：止损
  → Day30 ROAS 40-60%：观察，等 Day60
  → Day30 ROAS 60-80%：持续投放
  → Day30 ROAS > 80%：放量
```

## 六、常见错误

```
❌ 错误1：LT 累加单位混淆
  按月累加时，"1"代表第0个月（30天），不是第0天
  → 不要试图把"1"拆成30个1.0来按天累加

❌ 错误2：用 ARPPU 直接当 LTV
  ARPPU = $50，以为 LTV = $50
  → 错！ARPU = ARPPU × 付费率，LTV 是基于所有用户的

❌ 错误3：倍数用错游戏类型
  休闲游戏用了重度 SLG 的倍数
  → 高估 LTV，导致错误放量

❌ 错误4：只看单点数据，不看趋势
  Day7 ROAS = 10%，但 Day3 才 3% → 付费在加速 ✅
  Day7 ROAS = 10%，但 Day3 就 8% → 增长极慢 ⚠️

❌ 错误5：忽略 CPI 变化
  用早期 CPI $8 算 LTV/CPI = 2.5
  实际 CPI 涨到 $15，LTV/CPI = 1.3
  → 模型已经崩了，还在按旧数据决策

❌ 错误6：等数据完美再决策
  等 Day90 数据 → 已经烧了 3 个月钱
  → Day30 数据 ±10% 误差足够决策了
```

# 第四部分：数据分析与优化

## 10. 日常数据监控

### 10.1 每日必看 Dashboard

```
创建自己的 Dashboard（用 Excel/Google Sheets 或 BI 工具）：

渠道维度：
  ├─ 渠道名称
  ├─ 日花费
  ├─ 日安装数
  ├─ CPI
  ├─ Day1 ROAS
  ├─ Day7 ROAS（如果已回传）
  ├─ Day1 留存
  ├─ 素材数量
  └─ 状态（投放中/暂停/待优化）

素材维度：
  ├─ 素材名称
  ├─ 渠道
  ├─ 曝光数
  ├─ 点击数
  ├─ CTR
  ├─ 安装数
  ├─ IPM
  ├─ CPI
  ├─ Frequency
  └─ 状态
```

### 10.2 数据异常排查

```
场景1：CPI 突然上涨
  排查步骤：
    1. 看 CTR 是否下降 → 素材疲劳？
    2. 看 Frequency 是否上升 → 需要换素材
    3. 看竞争对手是否在大推 → 竞价变激烈
    4. 看平台是否有算法更新 → 等待或调整
    5. 看定向是否太窄 → 扩大受众

场景2：安装多但留存差
  排查步骤：
    1. 看素材是否过度承诺 → 用户期望与实际不符
    2. 看目标受众是否太泛 → 引入非目标用户
    3. 看游戏新手引导是否有问题
    4. 对比不同渠道的留存 → 某个渠道用户质量差

场景3：ROAS 不达标
  排查步骤：
    1. 看 LTV 曲线是否下降 → 产品问题
    2. 看 CPI 是否上涨 → 买量问题
    3. 看付费率是否下降 → 付费点设计问题
    4. 看 ARPPU 是否下降 → 大R用户流失
```

### 10.3 Cohort 分析

```
Cohort 表 = 按安装日期分组，追踪每组用户的表现

示例：

安装日期   Day0  Day1  Day3  Day7  Day14 Day30 LTV
2024-01-01 100%  45%   30%   22%   18%   15%  $2.5
2024-01-02 100%  43%   28%   20%   16%   14%  $2.3
2024-01-03 100%  42%   27%   19%   15%   13%  $2.1
2024-01-04 100%  44%   29%   21%   17%   14%  $2.4
2024-01-05 100%  40%   25%   18%   14%   11%  $1.8 ← 异常！

分析：
  - 1月5日的用户质量明显差（留存和LTV都低）
  - 排查：那天上了什么新素材？新渠道？新定向？
  - 结论：可能是某个新素材引入了低质量用户
```

### 10.4 优化清单

```
每周优化清单：

□ 素材优化
  □ 检查所有素材的 Frequency
    - >3 的素材准备替换
    - >5 的素材立即替换
  □ 分析 Top 3 和 Bottom 3 素材
    - Top 3：制作类似变体
    - Bottom 3：分析原因，改进或淘汰
  □ 制作新素材（每周至少 3~5 个）

□ 定向优化
  □ 按年龄拆分数据
    - 哪个年龄段 CPI 最低？LTV 最高？
  □ 按性别拆分
  □ 按地区拆分
    - 哪些国家 ROI 为正？加大投放
    - 哪些国家 ROI 为负？减少或暂停
  □ 检查 Lookalike 效果
    - 1% vs 3% vs 5%，哪个最好？

□ 出价优化
  □ 检查 Cost Cap 是否合理
    - 如果一直达不到 Cap，说明可以降 Cap
    - 如果经常超过 Cap，说明需要升 Cap 或优化素材
  □ 测试新的 Bid Strategy

□ 渠道优化
  □ 对比各渠道 ROI
  □ 测试新渠道（每月至少测试1个）
  □ 暂停 ROI 为负的渠道

□ 产品反馈
  □ 把买量数据反馈给产品团队
    - 留存曲线变化
    - 用户行为数据
    - 付费点效果
```

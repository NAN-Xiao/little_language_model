# 矩阵乘法详解 —— LLM 中的核心运算

## 核心规则

矩阵乘法 `@` 只操作**最后两个维度**，前面的维度（batch、seq）原封不动保留。

---

## 矩阵运算规律总结

### 基础矩阵运算

| 运算 | 输入 A | 输入 B | 输出 | 规则 |
|------|--------|--------|------|------|
| 加法 | `(M, N)` | `(M, N)` | `(M, N)` | 对应位置相加 |
| 数乘 | `(M, N)` | 标量 `c` | `(M, N)` | 每个元素乘以 c |
| 转置 | `(M, N)` | — | `(N, M)` | 行列互换 |
| 矩阵乘法 | `(M, K)` | `(K, N)` | `(M, N)` | 行乘列求和 |
| 点积（内积） | `(N,)` | `(N,)` | 标量 | 对应相乘再相加 |
| 外积 | `(M,)` | `(N,)` | `(M, N)` | 列向量乘行向量 |
| 逐元素乘法 | `(M, N)` | `(M, N)` | `(M, N)` | 对应位置相乘（Hadamard积）|

---

### 基础矩阵运算 —— 具体数字例子

#### 1. 加法（对应位置相加）

```
A = [[1, 2, 3],      B = [[10, 20, 30],
     [4, 5, 6]]           [40, 50, 60]]

A + B = [[1+10,  2+20,  3+30],     [[11, 22, 33],
         [4+40,  5+50,  6+60]]  =  [44, 55, 66]]

(2, 3) + (2, 3) = (2, 3)
```

**规则：形状必须完全相同，对应位置相加。**

---

#### 2. 数乘（每个元素乘以标量）

```
A = [[1, 2, 3],
     [4, 5, 6]]

c = 2

c * A = [[2×1, 2×2, 2×3],     [[2,  4,  6],
         [2×4, 2×5, 2×6]]  =  [8, 10, 12]]
```

**规则：标量与矩阵每个元素分别相乘。**

---

#### 3. 转置（行列互换）

```
A = [[1, 2, 3],
     [4, 5, 6]]

A.T = [[1, 4],
       [2, 5],
       [3, 6]]

(2, 3) → (3, 2)
```

**规则：第 i 行第 j 列的元素 → 第 j 行第 i 列。**

---

#### 4. 点积（内积，对应相乘再相加）

```
a = [1, 2, 3]
b = [4, 5, 6]

a · b = 1×4 + 2×5 + 3×6 = 4 + 10 + 18 = 32

(3,) · (3,) = 标量
```

**规则：两个向量对应位置相乘，然后全部加起来，结果是一个数。**

---

#### 5. 外积（列向量乘行向量）

```
a = [1, 2, 3]          (3,)
b = [10, 20, 30, 40]   (4,)

a 看作列向量 (3,1)，b 看作行向量 (1,4)

       [10, 20, 30, 40]
[1]    [1×10, 1×20, 1×30, 1×40]   [10,  20,  30,  40]
[2] ×  [2×10, 2×20, 2×30, 2×40] = [20,  40,  60,  80]
[3]    [3×10, 3×20, 3×30, 3×40]   [30,  60,  90, 120]

(3, 1) @ (1, 4) = (3, 4)
```

**外积 vs 矩阵乘法：**

外积在数学上记作 `⊗`，但在代码中它**就是矩阵乘法 `@`** 的一个特例：

```python
import torch

a = torch.tensor([1, 2, 3])        # (3,)
b = torch.tensor([10, 20, 30, 40]) # (4,)

# 方式1: torch.outer (直接算)
result = torch.outer(a, b)
# result = [[10, 20, 30, 40],
#           [20, 40, 60, 80],
#           [30, 60, 90, 120]]

# 方式2: reshape 后做 @ (原理相同)
a_col = a.view(3, 1)  # (3,) → (3, 1) 列向量
b_row = b.view(1, 4)  # (4,) → (1, 4) 行向量
result = a_col @ b_row  # (3,1) @ (1,4) = (3,4)
```

本质上：外积 = 把两个向量先变成列和行，再做矩阵乘法。所以数学上用 `⊗`，代码里用 `@` 也能算。

---

#### 6. 逐元素乘法（Hadamard积）

```
A = [[1, 2, 3],      B = [[2, 0, 4],
     [4, 5, 6]]           [1, 3, 2]]

A ⊙ B = [[1×2, 2×0, 3×4],     [[2,  0, 12],
         [4×1, 5×3, 6×2]]  =  [4, 15, 12]]

(2, 3) ⊙ (2, 3) = (2, 3)
```

**规则：形状必须相同，对应位置相乘。注意：这和矩阵乘法 `@` 完全不同！**

---

### 广播（Broadcasting）规则

广播是 PyTorch 的自动机制：**形状对齐后，把维度为1的方向复制扩展，然后做运算。**

#### 例1：A(2,3) + B(3,)

```
A = [[1, 2, 3],     B = [10, 20, 30]
    [4, 5, 6]]

A.shape = (2, 3)
B.shape = (3,) → 补齐为 (1, 3) → 扩展为 (2, 3)

B 广播后 = [[10, 20, 30],
            [10, 20, 30]]

A + B = [[1+10,  2+20,  3+30],   =  [[11, 22, 33],
         [4+10,  5+20,  6+30]]       [14, 25, 36]]
```

**每个值怎么算：**
- `result[0,0] = A[0,0] + B[0] = 1 + 10 = 11`
- `result[0,1] = A[0,1] + B[1] = 2 + 20 = 22`
- `result[1,0] = A[1,0] + B[0] = 4 + 10 = 14`

#### 例2：A(3,1) + B(1,4)（两个方向都广播）

```
A = [[1],           B = [[10, 20, 30, 40]]
    [2],
    [3]]

A.shape = (3, 1) → 扩展为 (3, 4)
B.shape = (1, 4) → 扩展为 (3, 4)

A 广播后 = [[1, 1, 1, 1],      B 广播后 = [[10, 20, 30, 40],
            [2, 2, 2, 2],                 [10, 20, 30, 40],
            [3, 3, 3, 3]]                 [10, 20, 30, 40]]

A + B = [[11, 21, 31, 41],
         [12, 22, 32, 42],
         [13, 23, 33, 43]]
```

#### 例3：标量广播

```
A = [[1, 2, 3],     B = 10
    [4, 5, 6]]

B 广播后 = [[10, 10, 10],
            [10, 10, 10]]

A + B = [[11, 12, 13],
         [14, 15, 16]]
```

#### 例4：不能广播的情况

```
A = [[1, 2, 3],     B = [10, 20]
    [4, 5, 6]]

A.shape = (2, 3)
B.shape = (2,) → 补齐为 (1, 2)

对齐后: A=(2, 3), B=(1, 2)
        最后一维: 3 ≠ 2，而且都不是1！

→ 不能广播！报错！
```

#### 广播规则

| A | B | 能否广播 | 结果 | 原因 |
|---|-----|---------|------|------|
| `(3, 4)` | `(4,)` | ✓ | `(3, 4)` | B补齐(1,4)，1扩成3 |
| `(3, 4)` | `(1, 4)` | ✓ | `(3, 4)` | 1扩成3 |
| `(3, 1)` | `(1, 4)` | ✓ | `(3, 4)` | 两个1各自扩展 |
| `(3, 4)` | `(3,)` | ✗ | 报错 | B补齐(1,3)，3≠4 |
| `(3, 4)` | `(2, 4)` | ✗ | 报错 | 3≠2，都不是1 |

**判断规则：从后往前对齐，先看最后一维！对应维度要么相等，要么其中一个为1。**

```
A = (3, 4)     ← 3行4列
B = (4,)       ← 补齐为 (1, 4)

对齐后:
  A: (3, 4)
  B: (1, 4)
      ↑  ↑
      |  └── 最后一维: 4 = 4 (相等 ✓)
      └───── 第一维:  3 和 1 (其中一个是1 ✓)
→ 能广播！

A = (3, 4)
B = (3,)       ← 补齐为 (1, 3)

对齐后:
  A: (3, 4)
  B: (1, 3)
      ↑  ↑
      |  └── 最后一维: 4 ≠ 3 (不相等！都不是1！✗)
      └───── 第一维:  3 和 1 (其中一个是1，但最后一维已经报错了)
→ 不能广播！
```

### 高维矩阵乘法

| 场景 | 输入 A | 输入 B | 输出 | 说明 |
|------|--------|--------|------|------|
| 基础二维 | `(M, K)` | `(K, N)` | `(M, N)` | 普通矩阵乘法 |
| 带 batch | `(B, M, K)` | `(K, N)` | `(B, M, N)` | B 个独立二维乘法，B 保留 |
| 三维 @ 三维 | `(B, M, K)` | `(B, K, N)` | `(B, M, N)` | B 个独立二维乘法，B 保留 |
| 更多维 | `(..., M, K)` | `(..., K, N)` | `(..., M, N)` | 前面的 `...` 维度原封不动 |

**核心规律：**
1. 最后两维做矩阵乘法：`(M, K) @ (K, N) = (M, N)`
2. 前面所有维度原封不动保留
3. 中间维度 K 必须相同（才能相乘）
4. 结果去掉 K，保留 M 和 N

### LLM 中的应用

| 运算 | 维度变化 | 作用 |
|------|---------|------|
| `x @ W_q` | `(B, seq, d) @ (d, d) = (B, seq, d)` | Q/K/V 投影 |
| `q @ k.T` | `(B, seq, d) @ (d, seq) = (B, seq, seq)` | Attention 分数 |
| `attn @ v` | `(B, seq, seq) @ (B, seq, d) = (B, seq, d)` | 加权混合 value |
| `ffn @ W_proj` | `(B, seq, d) @ (d, vocab) = (B, seq, vocab)` | 输出 logits |

## 例1：二维矩阵乘法 (基础)

```
[1, 2, 3]       [1, 0]       [1×1+2×0+3×1,  1×0+2×1+3×0]       [4, 2]
[4, 5, 6]   @   [0, 1]   =   [4×1+5×0+6×1,  4×0+5×1+6×0]   =   [10, 5]v
                [1, 0]

(2, 3)        @   (3, 2)     =           (2, 2)
```

**计算过程：**

结果第0行第0列 = `1×1 + 2×0 + 3×1 = 1 + 0 + 3 = 4`

结果第0行第1列 = `1×0 + 2×1 + 3×0 = 0 + 2 + 0 = 2`

结果第1行第0列 = `4×1 + 5×0 + 6×1 = 4 + 0 + 6 = 10`

结果第1行第1列 = `4×0 + 5×1 + 6×0 = 0 + 5 + 0 = 5`

**规则：`(M, K) @ (K, N) = (M, N)`**

中间维度 K 必须相同，结果去掉 K。

---

## 例2：带 batch 维度 (最常用)

```
[[[1, 2, 3],         [[1, 0],
  [4, 5, 6],     @    [0, 1],         =
  [7, 8, 9]]]         [1, 0]]]

(1, 3, 3)        @       (3, 2)
```

**结果：**

```
[[[1×1+2×0+3×1,  1×0+2×1+3×0],       [[[4, 2],
  [4×1+5×0+6×1,  4×0+5×1+6×0],   =    [10, 5],
  [7×1+8×0+9×1,  7×0+8×1+9×0]]]       [16, 8]]]

(1, 3, 3)        @       (3, 2)       =       (1, 3, 2)
```

**规则：`(B, M, K) @ (K, N) = (B, M, N)`**

batch 维度 `B=1` 保留，后面两维做矩阵乘法。

## 例3：两个三维矩阵相乘 (Attention 中的 Q@K.T)

用 batch=2 展示 batch 维度怎么对应：

```
A = [                    B = [
  # batch 0                # batch 0
  [[1, 2],                  [[1, 0],
   [3, 4]],                  [0, 1]],
  # batch 1                # batch 1
  [[5, 6],                  [[2, 0],
   [7, 8]]]                   [0, 2]]
]                        ]

(2, 2, 2)        @       (2, 2, 2)
```

**计算过程（batch 维度一一对应）：**

```
batch 0 的计算:
[[1, 2],     @   [[1, 0],     =   [[1×1+2×0, 1×0+2×1],    =   [[1, 2],
 [3, 4]]         [0, 1]]           [3×1+4×0, 3×0+4×1]]          [3, 4]]

batch 1 的计算:
[[5, 6],     @   [[2, 0],     =   [[5×2+6×0, 5×0+6×2],    =   [[10, 12],
 [7, 8]]         [0, 2]]           [7×2+8×0, 7×0+8×2]]          [14, 16]]
```

**最终结果：**

```
[[[1, 2],
  [3, 4]],
 [[10, 12],
  [14, 16]]]

(2, 2, 2)        @       (2, 2, 2)      =      (2, 2, 2)
```

**核心洞察：**

> **三维矩阵乘法 = B 个独立的二维矩阵乘法，结果按 batch 顺序拼起来。**

```
(B, M, K) @ (B, K, N) = (B, M, N)
        ↓
batch 0: (M, K) @ (K, N) = (M, N)  ← 第 0 个 batch 独立算
batch 1: (M, K) @ (K, N) = (M, N)  ← 第 1 个 batch 独立算
        ↓
把 B 个结果拼在一起 → (B, M, N)
```

**规律：**
- batch 0 的结果放在输出 batch 0 的位置
- batch 1 的结果放在输出 batch 1 的位置
- 两个矩阵的 batch 维度必须相同（都是 2）
- 各自最后两维做矩阵乘法

**规则：`(B, M, K) @ (B, K, N) = (B, M, N)`**

## 例4：转置后相乘 (Attention scores)

```
        [[1, 2, 3],                  [[1, 4],
 q =     [4, 5, 6],     k.T =         [2, 5],
         [7, 8, 9]]                   [3, 6]]

(3, 3)              @              (3, 2)
```

等等，维度不匹配！转置后：

```
        [[1, 2, 3],                  [[1, 4, 7],
 q =     [4, 5, 6],     k.T =         [2, 5, 8],
         [7, 8, 9]]                   [3, 6, 9]]

(3, 3)              @              (3, 3)
```

**结果：**

```
[1×1+2×2+3×3,  1×4+2×5+3×6,  1×7+2×8+3×9]     [14, 32, 50]
[4×1+5×2+6×3,  4×4+5×5+6×6,  4×7+5×8+6×9]  =  [32, 77, 122]
[7×1+8×2+9×3,  7×4+8×5+9×6,  7×7+8×8+9×9]     [50, 122, 194]
```

**(3, 3) @ (3, 3) = (3, 3)**

这就是 Attention 中的 `scores = q @ k.T`，结果是一个 `seq × seq` 的方阵。

## 例5：Transformer 中的应用串起来

输入 3 个 token，d_model=2：

```
# Embedding 后 (3 个位置，每个 2 维):
x = [[0.5, 0.3],
     [0.2, 0.8],
     [0.1, 0.4]]     (3, 2)

# W_q 投影矩阵 (2, 2):
W_q = [[1, 0],
       [0, 1]]
```

**Step 1: q = x @ W_q.T**

```
[0.5, 0.3]       [1, 0]       [0.5×1+0.3×0,  0.5×0+0.3×1]     [0.5, 0.3]
[0.2, 0.8]   @   [0, 1]   =   [0.2×1+0.8×0,  0.2×0+0.8×1]  =  [0.2, 0.8]
[0.1, 0.4]                       [0.1×1+0.4×0,  0.1×0+0.4×1]     [0.1, 0.4]

(3, 2)        @    (2, 2)     =            (3, 2)
```

**Step 2: scores = q @ k.T**

```
q = k (简化，假设 W_k = W_q):

[0.5, 0.3]                  [0.5, 0.2, 0.1]
[0.2, 0.8]   @   k.T =      [0.3, 0.8, 0.4]
[0.1, 0.4]

(3, 2)        @     (2, 3)     =     (3, 3)
```

**结果：**

```
[0.5×0.5+0.3×0.3,  0.5×0.2+0.3×0.8,  0.5×0.1+0.3×0.4]     [0.34, 0.34, 0.17]
[0.2×0.5+0.8×0.3,  0.2×0.2+0.8×0.8,  0.2×0.1+0.8×0.4]  =  [0.34, 0.68, 0.34]
[0.1×0.5+0.4×0.3,  0.1×0.2+0.4×0.8,  0.1×0.1+0.4×0.4]     [0.17, 0.34, 0.17]
```

**Step 3: out = attn @ v**

假设 softmax 后：

```
attn = [[1.0, 0.0, 0.0],
        [0.3, 0.7, 0.0],
        [0.2, 0.3, 0.5]]    (3, 3)

v = [[0.5, 0.3],
     [0.2, 0.8],
     [0.1, 0.4]]            (3, 2)
```

**结果：**

```
[1.0×0.5+0.0×0.2+0.0×0.1,  1.0×0.3+0.0×0.8+0.0×0.4]     [0.50, 0.30]
[0.3×0.5+0.7×0.2+0.0×0.1,  0.3×0.3+0.7×0.8+0.0×0.4]  =  [0.29, 0.65]
[0.2×0.5+0.3×0.2+0.5×0.1,  0.2×0.3+0.3×0.8+0.5×0.4]     [0.21, 0.56]

(3, 3)        @     (3, 2)     =     (3, 2)
```

**Step 4: logits = ffn @ W_proj**

```
ffn = [[0.5, 0.3],           W_proj = [[1, 0, 1],
       [0.29, 0.65],                   [0, 1, 0]]
       [0.21, 0.56]]

(3, 2)        @     (2, 3)     =     (3, 3)
```

**结果：**

```
[0.5×1+0.3×0,  0.5×0+0.3×1,  0.5×1+0.3×0]     [0.50, 0.30, 0.50]
[0.29×1+0.65×0,  0.29×0+0.65×1,  0.29×1+0.65×0]  =  [0.29, 0.65, 0.29]
[0.21×1+0.56×0,  0.21×0+0.56×1,  0.21×1+0.56×0]     [0.21, 0.56, 0.21]
```

**每一步的 seq=3 始终都在！3 个位置同时算！**

## 总结

| 运算 | 维度变化 | LLM 中的位置 |
|------|---------|-------------|
| `x @ W` | `(B, seq, d) @ (d, d) = (B, seq, d)` | Q/K/V 投影 |
| `q @ k.T` | `(B, seq, d) @ (d, seq) = (B, seq, seq)` | Attention scores |
| `attn @ v` | `(B, seq, seq) @ (B, seq, d) = (B, seq, d)` | 加权混合 |
| `ffn @ W_proj` | `(B, seq, d) @ (d, vocab) = (B, seq, vocab)` | 输出 logits |

**核心规律：`@` 只操作最后两维，batch 和 seq 原封不动保留。**

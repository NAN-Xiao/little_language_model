# Optimizer 优化器详解

## 一句话理解

**优化器 = 根据梯度调整参数的工具**。梯度告诉你"往哪走"，优化器决定"走多快、怎么走"。

---

## 核心流程（4步）

```
1. zero_grad()     清空旧梯度
2. loss.backward() 计算新梯度
3. step()          用梯度更新参数
4. (可选) 调整学习率
```

---

## 两种常用优化器对比

| | SGD | Adam |
|--|-----|------|
| 中文 | 随机梯度下降 | 自适应矩估计 |
| 特点 | 简单，只走梯度方向 | 聪明，自动调整步长 |
| 学习率 | 固定，需手动调 | 自适应，每个参数不同 |
| 动量 | 可选 | 内置 |
| 适用 | 大模型、大规模数据 | 大多数情况默认选择 |

---

## SGD 更新公式

```
参数 = 参数 - 学习率 × 梯度

w = w - lr * grad
```

## Adam 更新公式（简化）

```
1. 计算梯度的一阶动量（平均值）: m = 0.9*m + 0.1*grad
2. 计算梯度的二阶动量（平方平均值）: v = 0.999*v + 0.001*grad²
3. 用动量更新参数: w = w - lr * m / sqrt(v)
```

**Adam 聪明在哪？**
- 梯度大的参数，步长自动变小（防止震荡）
- 梯度小的参数，步长自动变大（加速收敛）

---

## 学习率 (Learning Rate)

| 学习率 | 效果 |
|--------|------|
| 太大 | 步子太大，跳过最优解，loss 不下降甚至发散 |
| 太小 | 步子太小，收敛极慢，训练时间太长 |
| 合适 | loss 稳定下降，最终收敛 |

**常用策略：**
- `warmup`：前 N 步从小逐渐增大（防止一开始梯度爆炸）
- `decay`：训练后期逐渐减小（精细调整）

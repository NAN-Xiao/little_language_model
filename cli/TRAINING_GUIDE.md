# Transformer 训练指南 —— 参数、日志与调参

> 对应文件: `cli/train.py`, `config.py`
> 训练命令: `python __main__.py train`

---

## 一、训练参数速查表

### 1.1 数据参数

| 参数 | 默认值 | 作用 | 调节建议 |
|------|--------|------|---------|
| `data_path` | Wikipedia JSONL | 训练数据文件路径 | 文本用 `.txt`，QA用 `.jsonl` |
| `seq_len` | 256 | 输入序列最大长度 | 显存够可提到 512/1024 |
| `val_split` | 0.1 | 验证集比例(10%) | 数据少可调到 0.05 |

### 1.2 批次与显存参数

| 参数 | 默认值 | 作用 | 调节建议 |
|------|--------|------|---------|
| `batch_size` | 4 | 每步处理样本数 | 显存 OOM → 减小 |
| `grad_accum_steps` | 8 | 梯度累积步数 | 等效 batch = 4×8 = 32 |
| `mixed_precision` | auto | 混合精度 | bf16 最稳，fp16 需 GradScaler |
| `safe_mode` | True | 稳定 CUDA 策略 | 报错时开启 |

**等效 batch_size 计算：**
```
实际 batch_size = batch_size × grad_accum_steps = 4 × 8 = 32
```

### 1.3 学习率参数

| 参数 | 默认值 | 作用 | 调节建议 |
|------|--------|------|---------|
| `learning_rate` | 3e-4 | 初始学习率 | loss 震荡→调小; loss 不降→调大 |
| `lr_schedule` | warmup_const | 学习率调度策略 | 推荐 warmup_const |
| `warmup_steps` | 800 | 预热步数 | 小数据调小，大数据调大 |
| `min_lr` | 0.0 | 学习率下限 | 长训练防止 lr 归零 |

**三种调度策略：**
```
const:        固定不变 ──────────────→
              适合微调/短训

warmup_const: 小步走 ↗ 大步走 ───────→
              推荐！最稳

transformer:  小步走 ↗ 峰值 ↘ 衰减 ──→
              适合大规模训练
```

### 1.4 模型与训练控制

| 参数 | 默认值 | 作用 |
|------|--------|------|
| `num_epochs` | 3 | 训练总轮数 |
| `max_grad_norm` | 1.0 | 梯度裁剪阈值，防梯度爆炸 |
| `log_interval` | 50 | 多少步打印一次日志 |
| `save_interval` | 1 | 多少轮保存一次 checkpoint |
| `seed` | 42 | 随机种子，保证可复现 |

## 二、训练日志解读

### 2.1 每轮结束时的汇总日志

```
第3/10轮 | 训练损失=2.3456(PPL=10.4) | 验证损失=2.6789(PPL=14.6) | 用时=45.2s
```

| 字段 | 含义 | 正常范围 |
|------|------|---------|
| `第3/10轮` | 当前第3轮，共10轮 | - |
| `训练损失=2.3456` | 训练集平均 loss | 预训练: 2~4; 微调: 1~3 |
| `PPL=10.4` | 困惑度 = e^loss | 越小越好 |
| `验证损失=2.6789` | 验证集平均 loss | 应接近训练损失 |
| `用时=45.2s` | 本轮训练耗时 | - |

### 2.2 中间详细日志（每 log_interval 步）

```
第3轮 | step=150 | batch=500/2500 | loss=2.1234 | 平均损失=2.4567 | 学习率=3.00e-04
```

| 字段 | 含义 |
|------|------|
| `step=150` | 全局累计步数（跨 epoch） |
| `batch=500/2500` | 当前 epoch 第500个 batch，共2500个 |
| `loss=2.1234` | 当前 batch 的 loss |
| `平均损失=2.4567` | 本 epoch 到目前为止的平均 loss |
| `学习率=3.00e-04` | 当前学习率 = 0.0003 |

## 三、核心名词解释

### Epoch（轮）
把整个训练数据集完整过一遍 = 1 个 epoch。

```
数据集 10000 条, batch_size=4
→ 每轮 10000/4 = 2500 个 batch
→ 训练 10 轮 = 模型看了 10 遍全部数据
```

### Batch Size（批次大小）
模型一次处理多少条数据。

| batch_size | 效果 |
|-----------|------|
| 1 | 更新快但噪声大 |
| 4 | 推荐起步，梯度较稳 |
| 32 | 梯度稳定但显存占用大 |

### Gradient Accumulation（梯度累积）
显存不够放 batch_size=32？可以 batch_size=4，累积 8 次后再更新。

```
等效 batch_size = 4 × 8 = 32
好处: 小显存模拟大 batch
代价: 每 8 步才更新一次，训练稍慢
```

### Loss（损失）
衡量模型"错得多离谱"。公式: `loss = -log(模型给正确答案的概率)`

| loss 值 | 含义 |
|---------|------|
| 0.1 | 很有信心 (~90%) → 很好 |
| 1.0 | 有点拿不准 (~37%) → 一般 |
| 3.0 | 基本瞎猜 (~5%) → 很差 |
| 4.16 | 完全随机猜（64词表均匀分布） |

**注意: loss 越低越好！**

### Train Loss vs Val Loss
- **Train Loss**: 模型在"练习题"上的表现
- **Val Loss**: 模型在"模拟考"上的表现（没见过的新数据）

### Perplexity（困惑度, PPL）
`PPL = exp(loss)`，表示模型"平均面对多少种等概率选择"。

| PPL | 含义 |
|-----|------|
| 64 | 像从64个词里猜 → 一般 |
| 20 | 像从20个词里猜 → 不错 |
| 5 | 像从5个词里猜 → 很好 |

### Learning Rate（学习率）
模型更新参数的步长。太大→震荡发散，太小→收敛慢。

### Mixed Precision（混合精度）
- `fp32`: 正常计算（4字节/数）
- `fp16/bf16`: 半精度（2字节/数），显存减半+计算更快
- `bf16` 推荐：范围和 fp32 一样，更稳

## 四、判断训练好坏

### ✅ 好现象
- 训练损失和验证损失都稳定下降
- 两者差距不大（< 0.5）
- 每轮验证损失刷新最低记录
- 生成的文本越来越通顺

### ⚠️ 过拟合（立刻停！）
- 训练损失继续下降，验证损失开始上升
- 两者差距越来越大
- 生成文本像复读机，直接复制训练数据

**应对**: 减小模型、加 dropout、早停

### ⚠️ 欠拟合
- 训练损失和验证损失都很高（> 3.0）
- 多轮后下降非常缓慢
- 生成文本完全胡言乱语

**应对**: 增大模型、延长训练、调高学习率

## 五、常用命令

```bash
# 基础训练
python __main__.py train

# 指定数据、调大 epoch
python __main__.py train --data-path data/my_corpus.txt --num-epochs 10

# 小显存方案：减小 batch，增大累积步数
python __main__.py train --batch-size 2 --grad-accum-steps 16

# 断点续训
python __main__.py train --resume-from checkpoints/pretrain/best_model.pt

# 全精度训练（bf16 出问题时的兜底）
python __main__.py train --mixed-precision off
```
